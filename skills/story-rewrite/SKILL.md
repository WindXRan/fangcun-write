# story-rewrite：结构仿写

**核心信念：只仿"骨相"，不仿"皮相"。**

**架构原则：主线程预读一切，agent 拿到 prompt 就写，不读任何文件。**

---

## ⚠️ 核心约束

### 反调色盘（防止"扒骨肉"）

仿写必须在以下3个维度上与源文本**完全不同**：

| 维度 | 说明 |
|------|------|
| **开篇方式** | 如源文本用心理治疗/催眠开篇，新书必须用其他方式（商战/遗物/急救等） |
| **破冰方式** | 如源文本用送物被拒，新书必须用其他方式（替解围/共同任务/意外接触等） |
| **节奏安排** | 如源文本一章写重生前，新书必须压缩或改变节奏 |

### 时间线连续

Phase 1 框架只提供「情绪弧线+叙事功能」，不提供具体场景。Phase 2 生成章纲锁定关键事件。Phase 3 agent 在章纲内自由发挥。

---

## 流程总览

### 试水模式（3章）

```
Phase 1（主线程，~2min）  Phase 2（1 agent，~2min）  Phase 3（3 agent 并行，~3min）  Phase 4（主线程，~30s）
读源文本 + 提取框架        生成章纲（每章情节点）        仿写3章（prompt即写，不读文件）   合并TXT + 字数统计 + 自检
```

**总耗时：~7-8分钟（试水3章）**

### 批量模式（全书）

```
Phase 1（一次性）: 全书章纲生成 → 存储到文件
Phase 2（循环）: 4章并行写 → 质量检查 → 不达标重写 → 合并 → 摘要压缩 → 滚动
Phase 3（结尾）: 全局一致性检查 + 禁用词扫描
```

**20章耗时：约15-20分钟，一次完成**

---

## 批量模式入口

用户输入：`/story-rewrite 批量 {源文本路径} {目标章数}`

示例：`/story-rewrite 批量 仿写试水库/源文本.txt 20`

**主流程**：

```
1. Phase 1: 读取源文本 + 生成仿写框架 + 全书章纲（一次性）
2. Phase 2: 分批写作循环（每批4章）
   loop:
     读取下4章章纲
     并行启动4个agent写作
     每章质量检查（禁用词+字数+钩子）
     不达标则重写（最多2次）
     合并到正文目录
     压缩前文摘要（近6章详记+更早概要）
     if 已完成目标章数 → 结束循环
     else → 继续循环
3. Phase 3: 全书质量检查
```

## Phase 1：结构提取 + 预加载

**用户只提供源文本路径，剩下全部自主完成。**

主线程收到源文本后，**同步完成以下操作**（不启动 agent，直接在主线程做）：

### Step 1：读取源文本

读取前3章 + 最后1章，记录：
- 源书名、题材、章数
- 开篇方式（具体场景 → 抽象为叙事功能）
- 破冰方式（具体事件 → 抽象为叙事功能）
- 节奏安排（各章情绪基调）
- 人设标签（女主/男主/配角/反派）
- 结局走向
- **叙述者风格**（关键，决定仿写是否"像人"）：
  1. 叙述者是否跳出来评论（有→吐槽型/冷幽默型/故事人型；无→沉浸型/冷峻型）
  2. 是否有现代人视角/网感语言（强→吐槽型；中→冷幽默/沉浸型；弱→故事人/冷峻型）
  3. 句子平均长度（<15字→吐槽型/冷峻型；15-25字→冷幽默/沉浸型；>25字→故事人型）
  4. 是否有第四面墙碎裂（有→吐槽型；偶尔→沉浸型；无→其他）
  5. 情绪外放程度（高→吐槽型；中→沉浸型；低→冷幽默/冷峻型）
  6. 口语化程度（极高→吐槽型/故事人型；中→冷幽默/沉浸型；低→冷峻型）
  7. **从源文本前3章提取 3-5 段最能代表叙述者风格的原文片段**（每段50-100字），作为仿写的"语感锚点"

### Step 2：预加载参考文件

同步读取以下三个文件，提取关键内容嵌入 Phase 2/3 prompt（agent 不再读文件）：

1. `narrative-writer.md`（位于 `.opencode/agents/`）→ 提取**三层展开法**（发生层+感知层+反应层）和**情绪弧线规则**
2. `banned-words.md`（位于 `skills/story-deslop/references/`）→ 提取**一级禁用词**（情态/动作/表情/心理/判断/形容/过渡类，**控制在 20 词以内**，避免挤占 Agent 推理空间）
3. `narrator-persona.md`（位于 `skills/story-rewrite/references/`）→ 根据 Step 1 的叙述者风格分析，**匹配最接近的人格类型**，提取该人格的定义和语言特征。如果源文本风格介于两种人格之间，允许混合（如 A+C）

⚠️ 主线程读取时需拼接项目绝对路径，不能用 `~/.opencode/`（那是用户全局目录，文件不存在）。

### Step 3：生成仿写框架

输出以下框架（只保留抽象模式，禁止具体场景）：

```
- 新书名：
- 新主角：（名字+核心标签，必须与源文本不同）
- 新背景：（时代/世界观，必须与源文本不同时代/不同地域）
- 核心冲突：（1句话）
- 开篇叙事功能：（抽象，如「铺垫→创伤回忆→重生契机」）
- 破冰叙事功能：（抽象，如「主动靠近→遭遇冷遇→坚持」）
- 开篇差异点：（必须与源文本不同）
- 破冰差异点：（必须与源文本不同）
- 节奏对齐表（3章）：
  - 第1章：情绪基调 / 叙事功能 / 弧线走向
  - 第2章：情绪基调 / 叙事功能 / 弧线走向
  - 第3章：情绪基调 / 叙事功能 / 弧线走向
- 人物关系模式（抽象）：
- 信息差设计：
- 核心差异对照表（时代/地域/职业/身份层面）：
- 角色名表（主线程必须统一，禁止 Agent 自行改名）：
  - 女主：{名字}
  - 男主：{名字}
  - 配角A：{名字+功能}
  - 配角B：{名字+功能}
- 题材标签：
- **叙述者人格**：（从 narrator-persona.md 匹配的人格类型，如"吐槽型网文读者"）
- **叙述者语感锚点**：（从源文本提取的 3-5 段原文片段，每段50-100字，作为仿写的风格参考）

## 关键事件锚点表（Phase 1 主线程填写，Phase 2 章纲展开，Phase 3 agent执行）
- 第1章锚点：{必须发生的事件A}，{必须发生的事件B}，本章以{悬念/情绪点}收尾
- 第2章锚点：时间线{紧接第1章/次日/数日后}，{必须发生的事件A}，{必须发生的事件B}
- 第3章锚点：时间线{紧接第2章/次日/数日后}，{必须发生的事件A}，{必须发生的事件B}
```

**自检**：框架给没读过源文本的人，能据此写出完全不同但结构相似的故事吗？不能 → 重新抽象化。**叙述者人格和语感锚点是否已填入框架？** 没有 → 回到 Step 1 补充提取。

```
# 写作规则（优先级：人格 > 规则）

## 1. 人味（最重要）

**核心原则：你不是在"写小说"，你是在"讲故事"。** 这个人格会怎么讲这个故事？用那个人格的嘴来说。

**具体要求**：
- 叙述者要有"声音"——不是隐形的旁白，是一个有性格的人在讲述
- 允许废话、允许吐槽、允许语法不完美、允许句子长短不一——这些是人类写作的特征，不是bug
- 对话要像真人说话——有打断、有沉默、有答非所问、有口语（"嗯""操""行吧"）
- 不怕"不完美"——人类写作有重复、有犹豫、有突然的跳跃

**⚠️ 情绪表达禁止重复**：
以下表达**全章只能出现 1 次**，出现第 2 次就是 AI 指纹：
- "脑子里一片空白" / "大脑空白了一瞬"
- "整个人僵住了" / "僵在原地"
- "悬着的心终于死了"
- "心跳加速" / "心跳漏了一拍" / "心跳飙到一百八"
- "两眼一黑" / "眼冒金星"
- "手心冒汗" / "后背发凉"
**替代方式**：用具体动作代替情绪词。"心跳加速"→"她握着手机的手紧了紧"；"整个人僵住了"→"她端着碗没动，筷子悬在半空"；"大脑空白"→直接写她的行为（"她张了张嘴，一个字没说出来"）

**⚠️ 网络梗降频**：
- 每章最多 1-2 个网络梗（"好家伙""薛定谔的XX"等），不能连续出现
- 用具体判断/动作代替网络梗。"好家伙，原主准备得挺齐全"→"冰箱里菜和肉都有，连饭盒都洗好了"
- "悬着的心终于死了"→"她盯着那个数字看了三秒，把手机扣在床上"

**⚠️ 超短句不能连续 3 个**：
- "黑的。焦的。冒烟的。"→刻意，一眼 AI
- 正确做法：超短句最多 2 个连续，第 3 句必须变长或换节奏
- "黑的。焦的。冒烟的。"→"黑的，焦的，已经看不出原来是什么了。"

**绝对禁止的词**（这些词几乎100%是AI指纹）：
仿佛、犹如、宛若、如同、一丝、一抹、深吸一口气、缓缓、不禁、微微、轻轻、淡淡、眼中闪过、嘴角勾起、眉头微皱、心中一动、心头一震、心下了然、不由得、闪烁着光芒、脸色一变、目光如炬、沉声道、只见、不由自主、瞳孔微缩

**绝对禁止的句式**：
- "不是A，而是B"（如"他不是冷漠，而是绝望"）→ 直接写B
- "，带着……"（万能状语）→ 拆短句
- "声音不大，却带着……" → 直接写声音
- "他/她知道……" → 用行为展示
- "仿佛/犹如……一般" → 删掉或白描
- "眼中闪过一丝……""嘴角勾起一抹……" → 用动作替代
- "心中涌起一股……""心头一震" → 用身体反应
- "他不知道的是……"式预告 → 删掉
- 章尾升华/总结/哲理 → 用动作或对话收尾

## 2. 结构（防止AIGC检测）

**核心原则：人类写作的结构是"混乱"的，不是"优化"的。**

- 句长要乱：不要刻意控制，让句子自然长短不一
- 段落要乱：不要每段都差不多长，有的段就一句话，有的段五六句
- 句首要换：不要连续用同一个主语开头
- 动作要解耦：不是每个动作都要补身体反应，有时候就写"他站起来"就够了
- 信息要跳跃：允许读者不知道的信息存在，不所有背景都交代清楚
- **用具体细节代替抽象情绪**：不要只写"穷"，写"超市临期酸奶三块五，她拿起来又放回去了"；不要只写"热"，写"风扇转了一晚上，床单还是潮的"

## 3. 反抄袭
- 禁止读取任何文件
- 禁止复制源文本情节/句式
- 角色名必须使用框架中的名字

## 写完后
1. 正文第一行必须是章名头，格式：`第{N}章 {章名}`（独占一行，前后各一个空行）
2. 用 write 工具写入 `temp_{书名}_ch{N}.txt`
3. 用 bash 统计字数：`(Get-Content 'temp_{书名}_ch{N}.txt' -Raw -Encoding UTF8) -replace '[\s]','' | Measure-Object -Character | Select-Object -ExpandProperty Characters`
4. 字数必须在 1800-2400 字之间。超出则删减，不足则补充，直到达标
5. 字数达标后，只返回：`第{N}章 {章名} | 字数：{X}字 | 达标`
```

### ⚠️ 关键事件锚定规则（防止多agent各自为政）

**问题根因**：多agent并行时，如果prompt没有锁定关键事件，每个agent会自行设计，导致章节间时间线断裂、事件版本冲突、人物行为矛盾。

**解法**：
1. Phase 1 框架包含「关键事件锚点表」，主线程在生成框架时把每章必须发生的1-2个关键事件锁死
2. Phase 2 生成章纲，将锚点展开为具体情节点（时间线+事件+钩子）
3. Phase 3 agent prompt 注入章纲内容，agent在章纲内自由发挥

锚点是「叙事功能+关键事件」层面的抽象描述，章纲是具体的情节点，agent在章纲框架内自行设计场景。

### 主线程构建 prompt 时的变量填充

| 变量 | 说明 |
|------|------|
| `{N}` | 章节号（1/2/3...） |
| `{总章数}` | 目标章数 |
| `{前情摘要}` | 批量模式：近6章详记+更早概要；试水模式：逐章递增 |
| `{框架内容}` | Phase 1 输出的完整框架（含角色名表+关键事件锚点表+叙述者人格+语感锚点） |
| `{章纲内容}` | Phase 2 输出的当前章节章纲（章名+核心事件+情节点序列+目标情绪+章首钩子+爽点+章尾钩子+字数目标） |
| `{禁用词列表}` | 从 `banned-words.md` 提取的一级禁用词（情态/动作/表情/心理类，控制在 20 词以内，避免挤占推理空间） |
| `{叙述者人格}` | Phase 1 匹配的人格类型名称+定义（从 narrator-persona.md 提取） |
| `{语感锚点}` | Phase 1 从源文本提取的 3-5 段原文片段（每段50-100字），作为仿写的风格参考 |

## Phase 4：合并输出

Phase 3 的 agent 返回后，主线程执行以下步骤：

### Step 0：字数校验（主线程全权负责）

**agent 不校验字数，主线程兜底。** 超标截断，不足警告。

```powershell
# 字数统计（汉字+标点+英文字母，排除空格换行）
(Get-Content '文件路径' -Raw -Encoding UTF8) -replace '[\s]','' | Measure-Object -Character | Select-Object -ExpandProperty Characters
```

对每章正文执行：
1. 统计字数
2. 如果 > 2400：按段落边界从末尾截断，保留前 2400 字（截断到最近的完整段落）
3. 如果 < 1600：在终端警告「第N章字数不足，需补充」
4. 截断后的正文才写入最终文件

### Step 1：读取临时文件

从 `temp_{书名}_ch{N}.txt` 读取各章正文。

⚠️ 临时文件命名规则：`temp_{书名}_ch{1,2,3}.txt`，书名从框架中取，去掉特殊字符（空格保留为下划线）。合并完成后删除临时文件。

### 试水模式：合并为TXT

写入 `仿写试水库/试水_{新书名}.txt`，格式：

```
========================================
{新书名}
========================================
作者：AI仿写
题材：{题材}

【简介】
{简介}

========================================
【仿写框架】（供 /story-long-write 开书时提取）
========================================
{Phase 1 输出的完整框架}

========================================
第1章 {章名}
========================================
{正文}

----------------------------------------
第2章 {章名}
----------------------------------------
{正文}

----------------------------------------
第3章 {章名}
----------------------------------------
{正文}

========================================
【后续方向】
- 第4章预告：[简述]
========================================
```

### 批量模式：逐章写入正文目录

每批4章写完后，主线程执行：

1. **读取临时文件**：从 `temp_{书名}_ch{N}.txt` 读取每章正文
2. **质量检查**：
   - 字数校验：用 `python skills/story-rewrite/tools/wordcount.py` 统计
   - 禁用词扫描：对照 banned-words.md
   - 钩子检查：确认章尾有钩子
3. **写入正文目录**：`{书名}/正文/第{NNN}章_{章名}.md`
4. **删除临时文件**
5. **更新追踪文件**：
   - 追踪/伏笔.md（新增/回收伏笔）
   - 追踪/时间线.md（记录事件时序）
   - 追踪/角色状态.md（角色状态变化）
   - 追踪/上下文.md（进度摘要）

### 摘要压缩机制（批量模式专用）

**目的**：防止上下文溢出，保持连贯性。

**规则**：
- 近6章：逐章详记（事件+状态变化+伏笔）
- 更早章节：压缩为概要（每10章一段，含核心事件+角色状态变化）
- 卷级总览：每卷一句话总结

**压缩格式**：

```markdown
## 近6章详记（第{N-5}-{N}章）
第{N-5}章：{事件摘要}
第{N-4}章：{事件摘要}
...
第{N}章：{事件摘要}

## 十章概要
第1-10章 | {核心事件} | {角色状态变化}
第11-20章 | {核心事件} | {角色状态变化}
...

## 卷级总览
第一卷·{卷名} | {主线进展} | {关键转折}
```

## Phase 5：全书质量检查（批量模式专用）

所有章节写完后，执行全局一致性检查：

### 1. 禁用词扫描

用 grep 扫描正文目录，对照 `banned-words.md` 一级禁用词列表逐词检查。

### 2. 标题去重检查

扫描所有章节标题，发现同名或明显重复时，按本章核心事件改名。

### 3. 伏笔状态检查

检查追踪/伏笔.md，确认：
- 所有已埋伏笔状态正确
- 无断线伏笔（预计回收章节已过但未回收）
- 伏笔密度合理（每卷15-25条）

### 4. 时间线自洽检查

检查追踪/时间线.md，确认：
- 时间线连续，无跳跃
- 事件时序合理
- 无时间矛盾

### 5. 角色属性一致性检查

检查追踪/角色状态.md，确认：
- 角色属性前后一致
- 关系变化有铺垫
- 无突然的性格转变
