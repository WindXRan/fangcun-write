# 分析任务

## 任务
分析原书，提取事件表、生成故事骨架、制定改编策略。

## 输入
- 原书路径：{原书路径}
- 作者名：{作者名}
- 源书名：{源书名}

## 执行步骤

1. 提取事件表（events.json）
2. 生成故事骨架（story_skeleton.md）
3. 制定改编策略（adaptation_strategy.md）

## 输出文件

```
_cache/
├── events.json              # 事件表
├── story_skeleton.md        # 故事骨架
└── adaptation_strategy.md   # 改编策略
```

## 质量检查

- [ ] 事件表是否完整（每章一行：角色/事件/主线关系/情绪）
- [ ] 故事骨架是否合理（三幕/分集/付费卡点/反转）
- [ ] 改编策略是否可行（8大要点/删减决策/世界观）
