# 简介生成提示词

## 核心原则

**简介要具体，不要抽象。** 别写"风云起天地动"这类华而不实的。

## 最佳方式

截取小说中一个有趣的点（设定或情节），进行具体完整讲述，直接展示风格。

- 喜欢的会留下
- 不喜欢的会离开（不要惋惜）

## 两种类型

### 第一类：场景截取式（推荐）

从书中截取一个有趣的小场景，完整展示：
- 主角是谁
- 面临什么 situation
- 如何应对
- 结果如何

**示例结构：**
```
{主角}发现{异常情况}。
{具体描写这个异常}。
{主角的反应和决定}。
{悬念钩子}。
```

### 第二类：传统介绍式（需极高功底）

先介绍背景开篇，后续留悬念。

## 生成要求

根据以下信息生成简介：
- **主角**：{主角身份和特点}
- **核心设定**：{金手指/特殊设定}
- **开篇事件**：{故事如何开始}
- **核心爽点**：{爽在哪里}
- **情绪基调**：{基调}

## 避免的问题

1. 太抽象（"风云起天地动"）
2. 太长（超过200字）
3. 剧透太多
4. 没有钩子
5. 与正文风格不符

## 输出格式

生成3个备选简介：
1. 场景截取式
2. 悬念钩子式
3. 混合式

每个附带说明为什么好。
