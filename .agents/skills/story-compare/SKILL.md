# story-compare：章对章仿写对比

将仿写新书正文与源文逐章并排，生成 `对比/对比_{start}-{end}.md`。

## 对比维度

| 维度 | 说明 |
|------|------|
| 字数对齐 | 新书每章字数 vs 源文同章，±10% 为合格 |
| 全文对照 | 源文和新书同章并排，方便逐段比对 |
| AI 痕迹 | 路标词、二分对照、协作口吻、高频副词等模式检测 |
| 风格指纹 | 句长分布、标点密度、对话比例、段落节奏 |

## 工具

| 脚本 | 用途 | AI 参与 |
|------|------|---------|
| `compare.py` | 逐章统计对比 + 全文对照 + 风格指纹 | 无 |
| `tools/auto_compare.py` | 调 LLM 评估换皮质量、台词重合、封控风险 | 有（调 API） |

## 用法

```bash
# 纯数据对比（无 AI，快）
python .agents/skills/story-compare/compare.py "{书名}" 1 10 --source {作者名}/{源书名}

# AI 换皮质量分析（调 API，深）
python .agents/skills/story-compare/tools/auto_compare.py --config configs/xxx.json --start 1 --end 10
```

## 对比完成后问 AI

对比报告生成后，把 `对比_{start}-{end}.md` 丢给 AI，问以下问题逐条评估章对章仿写质量：

1. **换皮检验**：剥掉人名地名后，新书和源文能区分吗？有没有段落是"换了名字其他照抄"？
2. **台词重合**：有没有 6 字以上连续匹配的对话？逐章标注重合位置。
3. **冲突类型**：源文每章的核心冲突（身份/利益/信息差/道德）在新书中是否换了类型？
4. **情绪曲线**：同章的情绪起伏是否对齐源文？有没有该爽的地方没爽到？
5. **章节排序**：这 {N} 章里哪三章写得最好、哪三章最差？差的原因是什么（字数不足/人设漂移/台词抄源文/情节空洞）？
6. **改进建议**：按章节给修复方案——该 trim 的 trim、该重写的重写、该微调润色的标注具体段落。

## 输出

`{书名}/对比/对比_{start}-{end}.md`，结构：

1. **统计对比表**（章节汇总：源文字数 vs 新书字数 vs 偏差）
2. **风格指纹**（句长/标点/对话占比/AI 模式检测）
3. **版本A（源文）**（全文连着放）
4. **版本B（仿写）**（全文连着放，可与源文逐段对照）
