# 质量审查管线

**目标：** 独立的质量门禁，不依赖写作流程
**触发：** 任何阶段均可单独调用
**入口：** `run_tool("工具名", {参数}, project_dir)`

---

## 审查清单

```
□ 格式规范检查 — fangcun-format.md
  □ 无机械分段
  □ 段落间无空行
  □ 无「他说」「她道」
  □ 无大段描写堆砌
  □ 无视角跳出
  □ 无语主过密

□ 叙事规范检查 — fangcun-narrative.md
  □ 无章末总结升华
  □ 无「他感到」「他觉得」直述情绪
  □ 无连续 2 段以上纯心理
  □ 无万能比喻
  □ 所有角色说话方式有区分
  □ 每段对话三功能检验（推进情节/制造期待/传递情绪）

□ 一致性检查 — fangcun-consistency.md
  □ 角色属性未冲突
  □ 时间线未矛盾
  □ 伏笔状态一致
  □ 力量体系一致

□ 大纲对齐检查 — fangcun-outline.md
  □ 核心事件对齐章纲
  □ 情绪弧线匹配
  □ 钩子类型正确
```

## 工具调用

```python
# 格式比较（章纲 vs 正文）
run_tool("compare", {
  "chapter_number": N,
  "user_input": "格式规范审查"
}, project_dir)

# 去AI味检查（不修改，只报告）
run_tool("deslop", {
  "chapter_number": N,
  "user_input": "只检查不修改"
}, project_dir)

# 一致性检查（通过 grep 查设定/正文/追踪）
# 需人工执行：grep 关键角色属性/时间线/伏笔
```

## 通过标准

| 级别 | 要求 | 动作 |
|------|------|------|
| P0 阻断 | 格式红线违反 | 必须修复 |
| P1 严重 | 叙事规范违反 | 建议修复 |
| P2 建议 | 一致性/大纲偏离 | 评估后决定 |
