# 逆推法工作流

**目标：** 从一本优质原文中提取通用写作规则，编译进写章prompt，确保产出质量对齐原文。

---

## 前置条件

- 有一本完整的高质量原文（朱雀100%绿）
- 正文已导入项目目录（`正文/正文/第N章.xml`）
- Python环境可用

---

## Step 1: 采样

均匀分布采样50-60章，覆盖全书前中后。

```python
import re, os
from pathlib import Path

d = "projects/{书名}/正文/正文"
files = sorted(os.listdir(d))
valid = [f for f in files if f.endswith('.xml') and re.match(r'第\d+章', f)]
chs = sorted([int(re.match(r'第(\d+)章', f).group(1)) for f in valid])
step = max(1, len(chs) // 55)
sample = chs[::step][:60]  # ~55章均匀分布
```

---

## Step 2: 特征提取

对每章提取以下硬指标。**所有指标必须量化，不能模糊。**

### 基础指标

```
chars = len(body)           # 每章总字数
lines = len(paragraphs)     # 总行数（段落数）
line_lens = [len(l) for l in lines]  # 每行字数列表
```

### 标点指标

```
感叹号数 = body.count('！')
问号数   = body.count('？')
省略号数 = body.count('……')
破折号数 = body.count('——')
分号数   = body.count('；')
双引号对 = body.count('"') // 2  # 对话使用
```

### OS（心声）指标

```
os_blocks = len(re.findall(r'【.*?】', body))   # OS条数
os_chars  = sum(len(m.group()) for m in re.finditer(r'【.*?】', body))  # OS总字数
os_ratio  = os_chars / chars * 100                # OS字数占比%
```

### 行级指标

```
avg_line_len = mean(line_lens)                    # 均行字数
short_line_ratio = count(l < 15) / lines * 100    # 短行(<15字)占比%
long_line_ratio  = count(l > 50) / lines * 100    # 长行(>50字)占比%
```

### 开篇指标

```
first_line = lines[0]  # 首行内容
# 判断首行类型：
is_OS      = '【' in first_line
is_dialog  = '"' in first_line or '"' in first_line
is_narrate = 以上都不是  # 叙述句开场
is_env     = 首行包含天气/风景/时间(需自定义关键词库)
is_protag  = 首行以主角名/人称代词开头
```

### 结尾指标

```
last_200 = body[-200:]  # 末尾200字

# 结尾类型判断：
last_char = body.strip()[-1]
结尾标点 = '！' or '？' or '。'
# 环境收尾：最后80字包含"暮色""月光""夜风""烛火""天色"等
# 时间收尾：最后80字包含"夜深""天亮了""亥时"等
# 升华收尾：最后80字包含"终于明白""这一刻""这一切"等
# 对话收尾：最后50字包含引号
```

### 修辞指标

```
有像字     = '像' in body
有比喻句式 = re.search(r'像.{0,10}(一样|一般|似的|般)', body)
有被动态  = re.search(r'是被.{2,20}的', body)
```

### 对话指标

```
对话轮次 = 连续对话的组数（难自动统计，可抽10章人工验证）
```

## Step 3: 聚合统计

将55章的数据聚合，计算每个指标的范围和分布：

```python
import statistics

stats = {
    'chars':   [2080, 2193, ..., 4668],   # 每章字数列表
    '感叹号':  [3, 23, ..., 45],          # 每章感叹号数列表
    'os_ratio': [0, 0, ..., 26],          # 每章OS占比列表
    # ... 每个指标的完整列表
}

# 对每个指标计算
for name, vals in stats.items():
    min_v = min(vals)                     # 最小值
    max_v = max(vals)                     # 最大值
    med_v = statistics.median(vals)        # 中位数
    avg_v = statistics.mean(vals)          # 均值
    zero_ratio = sum(1 for v in vals if v == 0) / len(vals)  # 为零的比例
```

## Step 4: 分类规则

按以下标准将每个指标分类：

### A — 100%规则（每章都成立，必须遵守）

判定标准：`zero_ratio >= 95%` 或 `非零比例几乎100%`

从《全家偷听心声》55章提取到的：

| 规则 | 原文统计 |
|------|---------|
| 感叹号每章必有 | 100% |
| 无省略号 | 96% |
| 无分号 | 100% |
| 无环境收尾 | 100% |
| 无时间收尾 | 100% |
| 无升华收尾 | 100% |

→ 写入 write-chapter.md 和 fanxie-chapter.md 的"100%规则"区块

### B — 高频特征（大多数章节如此，非强制）

判定标准：大多数章节符合，但有明显例外

从《全家偷听心声》55章提取到的：

| 特征 | 统计 | 用法 |
|------|------|------|
| 叙述句开篇 | 89% | 推荐叙述开篇，但不强制 |
| 通篇无"像"字 | 69% | 尽量少用"像"，但对话中出现可以 |
| OS占比<10% | 75% | OS只在情绪点时出现 |
| 感叹号密度10-40/章 | 中位23 | 参考范围 |

→ 写入 checker.py 的阈值，不作为prompt规则

### C — 章类型相关（不同章节类型不同）

判定标准：分布在特定类型章节中

| 特征 | 分布 | 说明 |
|------|------|------|
| 高OS（>10%） | 仅在情绪章/建立章 | 由章纲的writing_style控制 |
| 长章节（>3000字） | 仅在事件章/战斗章 | 章纲标注了本章功能 |
| 结尾钩子 | 情绪章多，事件章少 | 不强制每章必须有 |

→ 不由prompt控制，由章纲控制

---

## Step 5: 编译规则

### 写入 write-chapter.md

```
## 100%规则
- 感叹号：每章必有
- 省略号：禁止
- 分号：禁止
- 环境收尾：禁止
- 时间收尾：禁止
- 升华收尾：禁止
```

### 写入 checker.py

```
// checker thresholds
{
  "感叹号": {"min": 1, "warn_below": 3},
  "省略号": {"max": 0, "severity": "fail"},
  "分号": {"max": 0, "severity": "fail"},
  "环境收尾": {"match": false, "severity": "fail"},
  "时间收尾": {"match": false, "severity": "fail"},
  "升华收尾": {"match": false, "severity": "fail"},
}
```

### 不写入prompt，也不写入checker

- 开篇类型（89%叙述句，不是100%）
- OS密度（75%<10%，不是100%）
- 结尾钩子（42%，不是100%）
- 对话轮次（未验证）
- 情绪变化密度（未验证）
- 行级指标（范围波动大）

---

## Step 6: 验证闭合

```
原文 → 统计 → 规则 → write-chapter → 输出 → 再次统计
                                              ↓
                                        与原文统计对比
                                              ↓
                                     特征偏差是否在阈值内？
                                       ↓             ↓
                                     ✅ 通过       ❌ 调规则
```

每轮修改只影响一个维度。改完跑一次统计对比，确认偏差在阈值内。

---

## 迭代记录

每次逆推的统计结果应保存，形成多本书的规则基线：

```
_optimize/逆推统计/
├── 全家偷听心声.json    ← 每章指标全量数据
├── Book2.json           ← 下一本书
└── 汇总对照.md          ← 多本书规则对比
```

多本书的统计汇总后，可以提炼出跨书通用的规则（比如省略号/分号/感叹号可能所有网文都一致）。
