# 逆向拆书管线

**目标：** 把源书正文拆成结构化数据（总纲/骨架/文风指纹/改编策略）
**输入：** `projects/{author}/{book}/正文/正文/` 下有章节文件
**产出：** 作品信息/主题/*.xml + 故事骨架.md + 拆文库/文风.md + 改编策略.md
**注意：** 本管线不依赖 events.json。每步工具直接读取源文章节文件。

---

## Step 1: 导入总纲/简介/标签

读取前三章正文 → 生成总纲/简介/标签。

```
run_tool("book-import", {}, project_dir)
```

**变量依赖：**
- `@故事名称` → 从 `project.xml` 读取

**产出：**
- `作品信息/主题/总纲.xml`
- `作品信息/主题/简介.xml`
- `作品信息/主题/标签.xml`

**成功标准：** 三个文件都存在，内容非空

---

## Step 2: 故事骨架

从章节标题+总纲分析故事结构。

```
run_tool("skeleton", {"total_chapters": 总章数}, project_dir)
```

**变量依赖：**
- `@总章数` → 运行时传入
- `@作品信息/主题/总纲` → 上一步产出

**产出：** `故事骨架.md`

---

## Step 3: 文风分析

采样前三章+随机五章，分析句长/对话比例/段落节奏/情绪交替周期/开篇收尾技法。

```
run_tool("style-analysis", {"total_chapters": 总章数}, project_dir)
```

**变量依赖：**
- `@本章正文` → 自动读取

**产出：** `拆文库/文风.md` + `作品信息/dna.json`

---

## Step 4: 改编策略

从总纲+角色清单制定换皮方案。

```
run_tool("adaptation", {"total_chapters": 总章数}, project_dir)
```

**变量依赖：**
- `@作品信息/主题/总纲` → 注入

**产出：** `改编策略.md`
