# 仿写搭建管线

**目标：** 从逆向结果（源书）→ 换皮生成新书的全部框架文件
**前提：** `reverse-engineer` 管线已完成
**产出：** 新书项目文件（总纲/角色/卷纲/章纲）
**入口：** `run_tool("工具名", {参数}, project_dir)`

---

## Step 1: 开书设定

从逆向的源书数据 → 换皮生成新书的总纲/简介/标签/角色。
核心规则：所有人名/地名/势力名与源文完全不同，但故事骨架、爽点内核、角色功能位全部保留。

```
run_tool("open-book", {}, project_dir)
```

**产出：**
- `作品信息/主题/总纲.xml` — 新书总纲（已换皮）
- `作品信息/主题/简介.xml` — 新书简介
- `作品信息/主题/标签.xml` — 新书标签
- `作品信息/设定/角色/*.xml` — 新书角色卡（每个角色含 7 维语言风格）

**成功标准：** 所有文件已产出，且角色名/地名与源文完全不同

---

## Step 2: 卷纲

从总纲生成完整卷纲，自动匹配男频/女频格式。

```
run_tool("volume-outline", {}, project_dir)
```

**变量依赖：**
- `@作品信息/主题/总纲` → 上一步产出
- `@目标频道` → 从 project.xml channel 字段自动识别

**产出：** `正文/卷纲/第X卷.xml`（每卷一个文件）

**成功标准：** 所有卷的文件都已产出，每卷有剧情大纲

---

## Step 3: 章纲

从总纲+卷纲逐章生成章纲（五段式+情节点序列）。

```
# 批量生成前 N 章
for chapter_number in range(1, N+1):
    run_tool("plot-guide", {"chapter_number": chapter_number}, project_dir)
    # 展示→确认→保存
```

**变量依赖：**
- `@作品信息/主题/总纲`
- `@关联章节` → 上一章最后 500 字（自动关联）
- `@前文章纲` → 前 3 章章纲（自动关联）

**产出：** `正文/章纲/第N章.xml`（每章一个文件）
每份章纲包含：`<core_event>` + `<summary>`（五段式）+ `<characters>` + `<scene_plan>`

**建议：** 先批量生成前 10 章章纲，确认节奏后再继续。
按 192 章算，建议分 6 批（对应 6 卷），每批生成后人工确认。

**成功标准：** 至少前 10 章有章纲，后续可边写边补

---
