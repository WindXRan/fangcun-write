# 逆向拆书管线

**目标：** 把源书正文拆成结构化数据（总纲/骨架/设定/角色/文风DNA/拆书报告）
**输入：** `projects/{author}/{book}/正文/正文/` 下有章节文件
**产出：** 作品信息/主题/*.xml + 故事骨架.md + 拆文库/拆文报告.md + dna.json + 设定文件
**注意：** 本管线不依赖 events.json。每步工具直接读取源文章节文件或上游注入的数据。

---

## Step 0: 角色提取 + 生成章节摘要

读取全文章节标题和正文摘要，提取角色清单 + 全书章节摘要。

```
run_tool("character-extract", {"workers": 5}, project_dir)
```

**输入：** 源文章节文件
**产出：**
- `作品信息/设定/角色/*.xml` — 每个角色独立文件，含 7 维语言风格
- `@chapter_summaries` — 全书章节摘要（后续步骤的共享数据底板）

**说明：** 这是全管线的数据底板。后续所有工具都基于摘要而非原始正文分析，避免反复读取正文。

---

## Step 1: 前三章深度拆解

对前三章逐章做微观技法拆解，输出可复用的写作技法清单。

```
run_tool("golden-chapters", {}, project_dir)
```

**输入：** 源文前三章正文（自动读取）
**产出：** `拆文库/第1章_深度拆解.md`、`拆文库/第2章_深度拆解.md`、`拆文库/第3章_深度拆解.md`

**成功标准：** 三个深度拆解文件都存在，内含可复用的技法清单

---

## Step 2: 总纲/简介/标签

基于章节摘要 + 前三章深度拆解，生成总纲/简介/标签。
和旧版不同：此时已有 `@chapter_summaries` 全量数据，book-import 不再需要读原文。

```
run_tool("book-import", {}, project_dir)
```

**变量依赖：**
- `@故事名称` → 从 `project.xml` 读取
- `@chapter_summaries` → Step 0 产出

**注入辅助：**
- `拆文库/第1-3章_深度拆解.md` → 注入前三章的微观技法发现，辅助总纲判断

**产出：**
- `作品信息/主题/总纲.xml`
- `作品信息/主题/简介.xml`
- `作品信息/主题/标签.xml`

**成功标准：** 三个文件都存在，内容非空

---

## Step 3: 故事骨架

从章节标题 + 章节摘要 + 总纲分析故事结构。

```
run_tool("skeleton", {"total_chapters": 总章数}, project_dir)
```

**变量依赖：**
- `@总章数` → 运行时传入
- `@作品信息/主题/总纲` → Step 2 产出
- `@chapter_summaries` → Step 0 产出（新增：骨架分析现在有全量摘要可参考，不限于标题）

**产出：** `故事骨架.md`

**成功标准：** 文件存在且包含三幕结构、冲突升级节奏、情绪弧线、关键转折点

---

## Step 4: 文风DNA

采样前三章 + 随机五章正文，提取可量化的风格参数。**仅输出结构化 DNA，不再输出人工审阅报告。**

```
run_tool("style-analysis", {"total_chapters": 总章数}, project_dir)
```

**变量依赖：**
- `@总章数` → 运行时传入
- `@本章正文` → 自动读取

**产出：** `作品信息/dna.json`  — 结构化文风参数（句长/对话比例/段落节奏/情绪交替周期/章首章尾手法）

**不再产出：** ~~拆文库/文风.md~~（已砍，无下游依赖）

**成功标准：** dna.json 存在且字段完整

---

## Step 5: 设定提取（地点/物品/势力/背景）

从章节摘要提取世界观设定。角色已在 Step 0 提取完成。

```
run_tool("setting-extract", {}, project_dir)
```

**变量依赖：**
- `@chapter_summaries` → Step 0 产出

**产出：**
- `作品信息/设定/地点/*.xml`
- `作品信息/设定/物品/*.xml`
- `作品信息/设定/势力/*.xml`
- `作品信息/设定/背景/*.xml`

**成功标准：** 至少 2 类设定文件存在且非空

---

## Step 6: 关系图谱提取

从章节摘要提取角色关系网络（家族/情感/势力对立/盟友师徒）。

```
run_tool("relationship-extract", {}, project_dir)
```

**变量依赖：**
- `@chapter_summaries` → Step 0 产出

**产出：** `作品信息/设定/关系图谱.xml`

**成功标准：** 文件存在且四类关系至少各 1 条

---

## Step 7: 拆文报告汇聚

汇聚全部拆解产物，生成一份结构化的综合报告，供下游仿写搭建管线一步注入。

```
run_tool("analysis-report", {}, project_dir)
```

**变量依赖：**
- `@故事名称` → 自动读取
- `@总章数` → 运行时传入
- 所有上游文件自动汇入

**产出：** `拆文库/拆文报告.md`

**成功标准：** 文件存在且包含一句话解构、节奏分析、角色体系、可复用技法

---
