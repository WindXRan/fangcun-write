# 逆推法·全书共性规则

## 来源

逆推自《全家偷听心声》全书599章，均匀采样55章统计得出。这些特征在全书范围内稳定，不依赖特定章节。

---

## 一、100%规则（每章必须遵守）

从55章采样中，以下规则**每章都成立**：

| 规则 | 统计证据 | 执行 |
|:----|:--------|:-----|
| 感叹号！至少1个 | 55章100%有感叹号 | 每章必须有感叹号，OS必用感叹号收尾 |
| 禁止分号； | 55章0%使用分号 | 全书零分号，用句号断开 |
| 禁止省略号…… | 55章中仅4%使用省略号 | 基本不用，用句号/逗号代替 |

**只有这三条可以写进"必须遵守"。** 其他特征的统计分布都有波动，不能作为硬规则。

## 二、参考基准（不是规则，是参考）

以下特征有波动，不能要求"必须遵守"，但可作为写作参考：

| 特征 | 55章统计 | 用法 |
|:----|:--------|:-----|
| 感叹号 | 中位23个/章，范围3-45 | 参考值，不要求每章写到23个 |
| 问号 | 中位9个/章，范围3-22 | 正常使用 |
| OS | **75%章节OS=0条**，均值3条/章 | **OS不是必须的**。不要为了有OS而写OS |
| 破折号 | 56%章节使用，中位2次 | 偶尔用可以，不是必须 |
| 字数 | 中位2193，范围2080-4668 | 大部分章2000-2500，但战斗/事件章可到4000+ |
| 均行字 | 中位28.4，范围24.7-36.3 | 参考范围，不强制 |
| 强调短行 | 20%章节有（≤3字） | 少量章用强调行，不是必须 |
| 章尾钩子 | 42%落在！或？上 | 约半数的章尾是情绪收尾，其余是叙述收尾 |

## 三、核心原则

**统计是用来发现规律的，不是用来制造规则的。**

只有100%成立的特征才能写进"必须遵守"。其他特征——哪怕看起来很有规律——也只能作为参考。

正确的使用方式：

```
1. 统计全书 → 找出100%的特征（感叹号/分号/省略号）
2. 这些写进写作红线 → 硬规则
3. 其他统计数据 → checker.py 的阈值（不是prompt的规则）
4. 不需要把中位数当规则写进prompt
```


前几章（1-5章）OS密度高（12-33%），容易误判为"全书风格"。实际全书的OS分布极度不均匀：

```
OS密度 < 10% 的章节：75%
OS密度 10-25% 的章节：22%
OS密度 > 25% 的章节：3%
```

**结论：OS是调味料，不是主菜。** 大部分章节通过叙述和对话推进，OS只在关键情绪点出现。前几章OS多是因为需要快速建立主角人设，后面的章节不需要维持这个密度。

## 四、executable 规则文件

以下规则可以直接写入 write-chapter.md 和 fanxie-chapter.md 的写作红线：

### 标点红线
```
- ❌ 省略号……（全书94%章节零出现）
- ❌ 分号；（全书零出现）
- ✅ 感叹号！（中位23个/章，OS必用）
- ✅ 问号？（中位9个/章，正常使用）
- ⚠️ 破折号——（中位2个/章，偶尔用）
```

### 行级红线
```
- 每行25-35字
- 每章2000-2500字
- 短行率10-20%
- 长行率<10%
```

### OS红线
```
- OS是调味料，不是主菜
- 全书75%的章节OS占比<10%
- OS只在情绪关键点出现：吐槽/感叹/喊话
- 不要为了"有OS"而写OS
```

## 五、统计脚本

```python
# 统计算法书全章节的共性特征
# 放到 _optimize/prompt_regression/ 下
python3 << 'PYEOF'
import re, os, statistics
from pathlib import Path

d = "projects/{书名}/正文/正文"
files = sorted(os.listdir(d))
valid = [f for f in files if f.endswith('.xml') and re.match(r'第\d+章', f)]
chs = sorted([int(re.match(r'第(\d+)章', f).group(1)) for f in valid])
step = max(1, len(chs) // 50)
sample = chs[::step]

stats = {k:[] for k in ['chars','lines','excl','ques','os','ellip','dash','avg_line','short_pct','long_pct']}
for ch in sample[:60]:
    for f in files:
        if not f.endswith('.xml') or not f.startswith(f'第{ch}章') or len(f) < 15: continue
        body = re.search(r'<content>(.*?)</content>', Path(f"{d}/{f}").read_text('utf-8'), re.DOTALL)
        if not body: continue
        lines = [l.strip() for l in body.group(1).split('\n') if l.strip() and not l.strip().startswith('第')]
        full = '\n'.join(lines)
        ll = [len(l) for l in lines]
        stats['chars'].append(len(full)); stats['lines'].append(len(lines))
        stats['excl'].append(full.count('！')); stats['ques'].append(full.count('？'))
        stats['os'].append(len(re.findall(r'【.*?】', full)))
        stats['ellip'].append(len(re.findall(r'……', full))); stats['dash'].append(len(re.findall(r'——', full)))
        stats['avg_line'].append(round(sum(ll)/max(len(ll),1),1))
        stats['short_pct'].append(round(sum(1 for l in ll if l<15)/max(len(ll),1)*100))
        stats['long_pct'].append(round(sum(1 for l in ll if l>50)/max(len(ll),1)*100))
        break

for k,v in stats.items():
    if v: print(f"{k}: min={min(v)} med={statistics.median(v)} avg={round(statistics.mean(v),1)} max={max(v)}")
PYEOF
```

## 六、使用方法

1. 导入新书时，先跑统计脚本
2. 修改 `checker.py` 的检查阈值（字数、行数、标点等）
3. 修改 `write-chapter.md` 的通用规则
4. 如果新书的统计特征和此书一致，规则不用改
5. 如果不同，按新书的统计值重新编译
