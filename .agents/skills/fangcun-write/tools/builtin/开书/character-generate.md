# 工作模式

## 仿写模式（有源文角色可映射）

源文角色卡通过 `@补充要求` 传入（格式：源文角色名或源文角色卡路径）。

**流程：**
1. 读源文角色卡，提取其核心设定（core_desire / core_fear / core_value / bio）
2. 按仿写世界观翻译这些设定（换皮不换核）
3. 自动生成 `<source_mapping>` 标注源文对应关系
4. 仿写角色名如果未指定，基于源文名+仿写世界观重命名

**仿写模式 vs 原创模式的区别：**

| 维度 | 仿写 | 原创 |
|------|------|------|
| 角色来源 | 源文角色卡映射 | 总纲+用户输入 |
| source_mapping | ✅ 必填 | ❌ 不需要 |
| core_desire | 从源文翻译 | 从总纲推导 |
| 角色名 | 基于源文名+世界观重命名 | 自由命名 |
| 已有角色 | 检查不可重复生成 | 检查不可重复生成 |

## 原创模式（无源文）

**流程：**
1. 从总纲确定角色的功能位和重要性
2. 按分档标准设计角色
3. 不生成 `<source_mapping>`

# 角色分级标准

| 档位 | 适用 | 详细程度 |
|------|------|---------|
| tier="main" | 主角、重要反派、核心盟友、主要CP | 完整：core + voice + arc + bio + lines + relationships |
| tier="minor" | 配角、功能性角色 | 精简：tags + core + bio |
