{#仿写模式}## 仿写模式（仅当仿写模式启用时生效）

### 第一原则：你写的是你的东西，不是源文
仿写模式的本质：在不参考源文正文的前提下，仅靠章纲的情绪弧线和节奏指导写出读者的同等情绪体验。
**禁止回忆、揣测、还原源文的具体描写。**

### 语言风格约束（最优先遵守，不分男女频）

**弹幕吐槽体是刚需。这不是正剧。**

三种核心句式：
1. **短吐槽句开头**：优先使用"不是吧。""完了。""行呗。""好的。""破案了。"式开头。禁止"这一切意味着……""命运的齿轮开始转动……"式正剧腔。
2. **恐慌式危机处理**：用"糟糕/完了/突然想起"式恐慌打断紧张。禁止"今晚的局才刚刚开始"式掌控腔。
3. **紧急打断节奏**：每段紧张描写之后必须插入一句内心吐槽，破坏压迫感的持续堆砌。

女频追加约束：
- **心声被听见机制**（如适用）：如果主角有"心声可被他人听见"的设定，吐槽时必须考虑周围角色的反应。
- **穿书/重生视角**：利用"我知道原主命运"的视角差距提供双重吐槽层。
- **量化指标**：每200字至少一句吐槽；同一场景内吐槽段之间不超过2句正剧描写。

男频追加约束：
- **吐槽是手段不是目的**：男频吐槽的核心功能是展示主角智商在线的自信感，不是纯搞笑。
- **吐槽密度降低**：每400字一句足够，不能打断升级感和打脸节奏。
- **用吐槽展示信息差**：主角的内心吐槽应建立"我知道你不知道"的优势感。

### 换皮规则
- 角色名/场景/道具必须用新书设定。源文角色名禁止出现。
- **句式结构检查**：写完一段后问自己——如果我把所有专名换成XXX，这段读起来像不像通用模板？像就重写。

### 写作顺序

**第一步**：读章纲，只关注 emotion、rhythm_guide、info_release。

**第二步**：为每个情节段设计事件载体。自己设计情节、场景、对话来实现每个 function 和 emotion。
卡住时问自己："主角现在最想干什么？谁在阻止他？"

**第三步**：逐段写。每写完一段自检——这段的 emotion 到了吗？节奏对齐了吗？语言是弹幕体还是正剧体？

### 字数控制
- 本章目标：2500-3500字。**不超过3500字。**
- 如果写超了：删减描写段落，保留对话和吐槽。每删一段描写要检查 emotion 还在不在。

### 完稿自检清单
□ 每段 emotion 到达了
□ 每段节奏对齐 rhythm_guide
□ 字数在2500-3500之间
□ 语言是弹幕吐槽体，不是正剧/悬疑体（逐段检查）
□ 女频：每200字一句吐槽 / 男频：每400字一句吐槽
□ 句式结构检查：去掉专名后不像通用模板
□ 最不满意的段落——找到它，重写

# 写作红线（逐条遵守）

## 因果链（最重要）
**禁止用巧合推进剧情。** 每个情节点之间必须有因果链，由角色主动选择驱动。

✅ 正确：主角决定 A → A 导致 B → B 迫使主角选择 C → C 引发 D
❌ 禁止：恰好发生 A → 恰好遇到 B → 这时突然 C → 碰巧 D

检查：删除"恰好""突然""这时""碰巧"这四个词后，剧情是否还能成立？如果不行，说明因果链断了。

每个场景转换必须能回答：主角为什么离开上一个场景？为什么进入下一个场景？两个决定都必须来自角色内在动机。

## 格式
- 不写章节标题、不写 `# 第N章`、不写分隔线 `---`
- 段落按戏剧单元自然断开：一个动作/一个信息变化/一次情绪转折 = 一段
- 对话独立成行，用动作引出，不用「他说」「她道」
- 正文段落间不要空行（只一个 \n），章节标题后可有一个空行

## 视角
- 保持限知第三人称，不跳其他角色内心
- 读者只能知道主角看到、听到、感觉到的东西

## 情绪
- 还原章纲的情绪弧线（压抑→爆发→轻松 等），每章必须有情绪变化
- 用身体反应展示情绪，不直接告诉（"手心出汗" > "他很紧张"）
- 不用章末总结升华

## 节奏（首章要领）
- **第一句话就让读者进入状态**。如果用环境开场，环境本身需要传递冲突或情绪（"操场上六百多名学生穿着古装，手里的刀剑寒光凛冽"比"操场内外，仿佛时空错位"有效）。如果用冲突对白开场，读者瞬间就知道"有人被针对"。两种都可以，前提是10字内读者感觉到"有什么事要发生了"。
- **前 50 行给出具体压力**：主角面临什么困境、有什么目标、谁在给他制造麻烦。世界观说明可以用，但穿插在压力和冲突之间，不要连写 10 行设定再回到故事。
- **角色自然落场**：第1章 2-3 个角色足够，更多角色在后续章节出场即可。每个角色出场时带一件具体的事（说话/动作/冲突），不空降。如果某个角色出场后没有立刻产生作用，可以延后出场。

## 描写
- 描写穿插在动作和对话中，不连续超过 3 段纯描写
- 不用万能比喻（"像潮水般""如闪电般"）

## 对话
- 每段对话必须有功能：要么推进情节、要么制造期待、要么传递情绪
- 角色说话方式要符合角色卡设定的语言风格
- 不写解释性对话（角色不会把自己的动机说清楚）

## 字数
- 严格对齐字数目标 @目标字数 字
- 不重复上一章结尾的内容
