# fangcun-write 傻瓜式说明

## 你说什么，我做什么

| 你说 | 我做 |
|------|------|
| **「写第11章」** | 全自动跑完一章 → preflight→章纲→写章→审查→修复→去AI味→质量门禁。有问题自动停 |
| **「修复第6章」** | 只修已有章节：审查→结构修复→靶向修复→去AI味→质量门禁 |
| **「质量检查」** | 只跑 `quality-gate --phase=chapter`，不修改任何文件 |
| **「审计」** | 每5章跑一次 `quality-gate --phase=audit`，查跨章矛盾/角色名统一/伏笔追踪 |

> **核心原则**：每步完成后展示结果给你看，你说"可以"才进下一步。

## 自动防呆机制（不用你操心）

以下是每章写的时候自动跑的质量检查，出问题会停：

### preflight（写章前自动跑）
```
检查项                          发现问题时
──────────────────────────────────────────────────────────
✅ 章纲存在                      🔴 → 先帮你生成章纲
✅ 出场角色都有角色卡             🔴 → 列出缺的角色卡，帮你建
✅ 前文时间线衔接                 🟡 → 提示但继续
✅ 源文→仿写角色映射完整          🟡 → 提示补 source_mapping
```

### chapter（写章后自动跑）
```
检查项                          发现问题时
──────────────────────────────────────────────────────────
🔴 P0 格式红线（总结升华/他说她道/万能比喻） → 自动阻塞，修好才通过
🟡 P1 叙事规范（对话占比/视角/角色区分）     → 建议修复
🟡 P2 一致性（角色属性/信息释放/前文伏笔）   → 建议修复
```

### audit（每5章自动跑）
```
检查项                          发现问题时
──────────────────────────────────────────────────────────
🔴 跨章角色名统一（第5章韩十夏→第10章顾家） → 自动修
🔴 时间线矛盾                               → 自动修
🟡 管线映射完整性（源文角色都有仿写对应）   → 提示
🟡 伏笔追踪（info_hold 超过5章未释放）      → 提示
```

## 工具清单（调试用，平时不用看）

| 工具 | 功能 |
|------|------|
| `fanxie-chapter` | 按章纲写正文 |
| `fanxie-review` | 读者视角审查 |
| `fanxie-structural-fix` | 结构对齐（场景/冲突/信息修复） |
| `fanxie-fix` | 技法修复（调性/开场/节奏/AI腔） |
| `character-split` | 拆复合角色（刘嬷嬷→孙嬷嬷+老刘头） |
| `quality-gate` | 质量门禁（preflight/chapter/audit） |
| `deslop` | 去AI味 |
| `guide-convert` | 源文章纲→仿写章纲的翻译器 |

## 项目结构

```
projects/仿写新书/
├── 正文/
│   ├── 章纲/第{N}章.xml    ← 章节细纲
│   └── 正文/第{N}章.xml    ← 章节正文
├── 作品信息/
│   ├── 设定/角色/           ← 角色卡（每角色一个xml文件）
│   ├── 设定/关系图谱.xml    ← 角色关系
│   └── 主题/               ← 总纲/简介/标签
└── 拆文库/全家偷听心声/     ← 源文分析产物
```
