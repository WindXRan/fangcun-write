<task>
为《{新书名}》第{N}章生成场景级章纲。换皮不换骨架。

⚠️ **主角名锁定：从 name_map 中查找源文"林哲"对应的新名。**

⚠️ **本章章纲（严格按此执行，不可偏离核心事件）：**
{outline_entry}
章纲规定了：核心事件、开头承接、结尾状态、衔接方式。你的细纲必须严格实现章纲的核心事件和出场角色，不可替换、不可跳步、不可将人物关系戏份替换为纯事业线。

## 第一步：读源文 + 读章纲

1. **读源文**：数事件、记节奏、记信息释放、记基调、提取五段式弧线（同上）
2. **读 events.json**：从源书 `_cache/events.json` 中查本章条目——核心事件、开头承接、结尾状态、衔接、情绪弧线。这些是本章的**硬约束**，不能偏离

⚠️ 以上信息必须在输出时写进「源文结构分析」和「内容概括」字段。

## 第二步：闭卷设计 —— 换皮不换骨

- **事件数必须对齐**：源文写了 3 个场景，你设计了几个场景？多一个少一个都不行
- **角色位必须对齐**：源文本章出场了"羞辱主角的上司""帮腔的同事""看热闹的路人"三个角色位→你必须是同样的三个角色位，用新名，但功能一一对应。源文没出场过的角色，不能在本章凭空出现
- **角色名必须锁定**：所有角色名必须来自 characters.md 中的 name_map，不得自己编造新名。主角名在整个仿写过程中固定不变
- **信息释放对齐**：源文本章引入 2 个新角色→你也是 2 个。源文本章藏着 X 信息→你同样藏着对应信息
- **情绪曲线对齐**：源文前面紧张中间爆发后面轻松→你也是
- **换什么**：人物名字、地点、职业、具体台词、具体情节——这些都是"皮"
- **不换什么**：事件数量、场景节奏、信息释放节奏、情绪曲线——这些是"骨"

## 第三步：输出章纲
按输出格式输出。章纲即写作蓝图，必须包含五段式弧线、情节点序列（带功能标签）、钩子设计——写章 agent 拿到就能照着写，不需要再回读源文。

## 基调一致性（对齐源文）
- 本章的基调（轻松/沉重/温馨/紧张/虐心）必须与**源文的整体基调**一致
- 源文如果是轻松吐槽，本章就不能写成沉重现实
- **同一本书内各章的基调必须稳定**：不能这章荒诞冷幽默，下章温馨生活化。所有章节的"喜感类型"要统一——要么全走荒诞线，要么全走生活化喜剧，不可混合
- 输出时先检查：本章的基调是否偏离了全书已建立的调性？如果偏离，必须修改

## 源文结构分析（内部思考，不输出）

以下字段只用于你对齐源文、确保换皮不换骨，**不要写进最终输出**：

## 输出格式

### 第 N 章：{章名}
- 核心事件：{一句话}
- 字数目标：{源文字数 ±10%}
- 目标情绪：{本章交付什么情绪}
- 章首钩子：{从章首7式中选} — {具体内容}
- 爽点：{本章爽点，如无显性爽点写"无显性爽点，功能是…"}
- 主角名：{来自 name_map 中 role=主角 的新名，本章全文固定使用此名}

### 内容概括（五段式）
- 起因：{本章事件为什么发生}
- 发展：{冲突如何推进}
- 转折：{信息/关系/局势哪里改变}
- 高潮：{本章情绪或动作峰值}
- 结尾：{收束到什么状态}

### 情节安排（多线）
- 主线推进：{本章对主目标的推进}
- 辅线推进：{可写"无"，不能凭空制造}
- 事件线 / 任务线：{外部事件链}
- 感情线 / 关系线：{无显性感情线时写"无显性，但关系变化为…"}
- 逻辑线：原因 → 行动 → 结果 → 后果/新问题

### 人物关系和出场顺序
- 出场顺序：{角色/势力/关键物件按实际出现顺序列出，只需列出本章出场的角色，不必列出全书角色}
- 人物关系变化：{本章前 → 本章后}
- 视角/信息差：{谁知道什么；读者知道什么；主角误判什么}

### 情节细化
- 情节点序列：按字数目标反推数量（约200-300字/个情节点；下限10个；常规2000字章节10个，复杂高潮章可到15个），每个情节点写清"谁做了什么 + 功能标签"
  - 功能标签即目的词：冲突/推进/爽点/打脸/铺垫/反转/过渡/人物塑造/信息揭示
- 代价兑现 / 收益兑现：{谁付出什么代价；谁获得什么收益；是否留下后续账}

### 结尾设定和钩子
- 结尾设定：{收束状态；未解决问题；下一章推动力}
- 章尾钩子：{具体内容，期待度：强/中/弱}

⚠️ 以上所有字段必须全部填写，不得留空或写"无"。本章的「结尾设定」中的"未解决问题"和"下一章推动力"是下章 plot-guide 衔接的关键——下章必须接着这个状态写。
</output_format>
