# 为《{新书名}》第{N}章生成创作指令

分析源文，输出创作指令。不输出源文节拍映射，不输出源文台词。

## 本章事件
{本章事件}

## 风格类型
{风格类型}

## 信息释放时机
{信息释放时机}

## 角色
{角色约束}

## 世界观
{世界观}

## 源文高光
{源文高光}

## 源文分析任务

1. 情绪功能：这一章在全书中承担什么角色？读者读完应该有什么感受？
2. 人物状态变化：开始→结束
3. 核心冲突：主要矛盾 + 升级梯度（阶段1→阶段2→阶段3）
4. 写法技巧：2-3个可复用的技巧
5. 场景要求：几个场景？环境/氛围？

## 输出格式

```
# 第{N}章 创作指令

## 情绪功能
{具体，如"让读者第一次对男主心动"，不要"推进剧情"}

## 情绪弧线
开头({情绪}) → 中段({情绪}) → 高潮({情绪}) → 结尾({情绪})
- 节点1: {场景} → {转折}
- 节点2: {场景} → {转折}

## 人物状态变化
- 开始：{状态}
- 结束：{状态}

## 核心冲突
{描述}
阶段1({小摩擦}): {事件} → 情绪: {情绪}
阶段2({激化}): {事件} → 情绪: {情绪}
阶段3({爆发}): {事件} → 情绪: {情绪}

## 写法技巧
1. {技巧}
2. {技巧}

## 开篇钩子
- 类型：{冲突前置/悬念前置/情绪前置/反差前置/对话前置}
- 设计：{具体方式}

## 信息流表
| 类型 | 内容 | 说明 |
|------|------|------|
| 释放 | | |
| 暗示 | | |
| 藏住 | | |
| 结尾悬念 | | |

## 功能需求清单
逐条列出本章必须实现的功能，每条附验证标准：
- 功能1：{效果} — 验证：{读者感受}
- 功能2：{效果} — 验证：{读者感受}
（4-6条，禁止输出具体场景设计）

## 字数目标
{目标字数}字（{目标字数_min}~{目标字数_max}）
```

## 铁律
1. 不输出源文节拍映射
2. 不输出源文台词
3. 不输出具体场景设计，只输出功能需求
