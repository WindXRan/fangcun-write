# 仿写模式

## 核心原则

**仿写 ≠ 换皮。仿写 = 学习底层逻辑，独立创作。**

- 角色名必须换（与源文完全不同）
- 剧情结构保持一致
- 冲突类型保持一致
- 情绪基调保持一致
- **但情节事件、场景设计、对话内容必须是全新的**

## 仿写方法论

### 第一步：拆解（动笔前必做）

分析源文的底层创作逻辑：
1. **场景功能**：这个场景在全书中承担什么功能？（建立关系/制造冲突/推进剧情/情感转折）
2. **情感节奏**：情绪如何推进？（先紧张后温馨/先压抑后爆发）
3. **人设展现**：角色通过什么方式立起来？（动作/对话/选择/反应）
4. **信息释放**：读者在这章获得了什么信息？这些信息如何推动后续剧情？

### 第二步：重构（基于拆解结果独立创作）

基于拆解出的底层逻辑，设计一个**完全不同的情节事件**来实现相同的场景功能：
1. **核心冲突**的起因、过程和收尾均不得与源文雷同
2. **人物**通过新的细节和动作来展现相同性格底色
3. **关键戏剧性桥段**必须用功能相同但形式完全不同的设计替代

### 第三步：角色行为档案（从人物出发，而非从情节出发）

**角色的行为模式决定情节走向，而非反过来。**

为每个角色建立独立的行为档案：
1. **核心性格关键词**（3个，不可与源文角色重叠）
2. **面对冲突时的本能反应**（1句话，必须与源文角色有明显区别）
3. **独有的语言习惯**（1个标志性特征）

**自检：** 每完成一个场景，问自己——"这个角色在这个情境下会怎么做？"而非"源文角色在这个情节节点做了什么？"

## 仿写禁区（违反=失败）

**禁止情节骨架照搬：**
- 源文中"A事件→B冲突→C收尾"的情节推进顺序，不可在仿写中以相同顺序重现
- 例如：源文是"电话骚扰→下楼拿东西→搬不动→男配帮忙→装病诬陷→母亲通话"，仿写必须设计完全不同的事件链

**禁止标志性道具复用：**
- 源文中承担关键戏剧功能的道具，仿写中不可使用相同或高度相似的道具来实现相同功能

**禁止标志性台词移植：**
- 源文中具有高辨识度的台词设计，仿写中不得出现同义替换版本

**自检：完成后逐条说明你的仿写是如何规避以上禁区的。**

## 角色设定规则

### 必须覆盖所有出场角色

1. 从events.json中提取所有出场角色
2. 按出场次数排序
3. 角色卡必须覆盖所有角色，不能遗漏

### 角色卡格式

```
【角色名】（源文对应：{源文角色名}）
- 功能位：{与源文一致}
- 性格内核：{2-3句话描述性格本质}
- 核心动机：{他/她最想要什么}
- 关系：{与主角/其他角色的关系}
```

### 验证规则

1. 角色卡必须覆盖所有出场角色
2. 每个角色都有性格内核和核心动机
