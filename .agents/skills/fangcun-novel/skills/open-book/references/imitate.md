# 仿写模式

## 核心原则

**仿写 = 换皮不换骨**

- 角色名必须换（与源文完全不同）
- 剧情结构保持一致
- 冲突类型保持一致
- 情绪基调保持一致

## 角色名映射规则

### 必须覆盖所有出场角色

1. 从events.json中提取所有出场角色
2. 按出场次数排序
3. 映射表必须覆盖所有角色，不能遗漏

### 起名铁律

- 新名必须与源文名完全不同（不是加字、不是换一个字、不是换姓）
- 名字像真人，禁用AI审美
- 禁用字：寒琛霆辰砚辞屿淮洲舟泽衍瑾瑜墨熙尧翊宸 / 晚念栀眠清意棠予初浅瑶萱璇颖苒若汐诺涵 / 婳翊珩彧旻昶堇蓁莞芊筱妤淇沄璟
- 合并条目必须拆开（如"林建华/林远征/林兴源"必须分别起3个不同新名）
- 名字区分度：不同角色的名字不能有相同字、不能谐音、姓氏要多样化、长度要混合（单名+双名）

### 映射表格式

```
| 源文名 | 新名 | 性别 | 功能位 | 出场次数 |
|--------|------|------|--------|----------|
| {源文角色} | {新名} | {性别} | {功能位} | {次数} |
```

### 验证规则

1. 映射表必须覆盖所有出场角色
2. 新名必须与源文名完全不同
3. 新名不能重复
4. 新名不能是AI审美
