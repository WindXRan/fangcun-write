# 审改参考

## 核心原则

**审改 = 审查 + 分类 + 修复 + 报告**

## 审查维度

### AI痕迹
- 路标词（首先/然后/最后）过多
- 情绪直说（"她很生气"）
- 比喻堆砌
- 句式均匀
- 代词开头过多

### 角色一致性
- 角色名是否正确
- 角色行为是否与角色卡一致
- 角色能力边界是否一致

### 字数偏差
- 字数是否在目标范围内（±15%）

### 钩子设计
- 开篇是否有钩子
- 章末是否有钩子

### 爽点设计
- 是否有爽点
- 爽点是否足够强烈

## 问题分类

### 机械问题（搜索替换即可）
- AI路标词
- AI痕迹词
- 代词密度

### 风格问题（局部改写）
- 结尾意象
- 对话标签
- 情绪同质化
- 台词雷同

### 结构问题（需要整章重写）
- 人设漂移
- 时间线断裂
- 冲突降级
- 地点错误

## 修复方式

### 机械问题 → 搜索替换
- AI路标词/痕迹词：直接删除
- 代词密度：将部分"她/他"替换为角色名

### 风格问题 → 局部改写
- 结尾意象：只改最后1-2句
- 对话标签：只改标签
- 情绪同质化：只改情绪描写部分

### 结构问题 → 整章重写
- 人设漂移：重写时以characters.md为准
- 时间线断裂：重写时以上一章结尾状态为起点
- 冲突降级：重写时提升冲突强度
