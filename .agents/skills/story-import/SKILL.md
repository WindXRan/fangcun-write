# story-import · LLM 导入

## 功能

用 LLM 读取源文件，自动识别：
- 书名、作者、简介
- 正文章节（第X章）
- 番外章节
- 输出到标准目录结构

## 使用

```
/story-import <txt文件路径>
```

## 执行步骤

1. 读取源文件前 2000 字，识别书名、作者、简介
2. 读取源文件全文，识别章节和番外
3. 拆分章节到 chapters/ 目录
4. 拆分番外到 fanwai/ 目录
5. 生成 _header.txt 和 _toc.txt

## 输出结构

```
projects/{作者}/{书名}/
└── _cache/
    ├── _header.txt          # 书籍信息
    ├── _toc.txt             # 目录
    ├── chapters/
    │   ├── 第001章.txt
    │   └── ...
    └── fanwai/
        ├── 番外1.txt
        └── ...
```

## LLM 提示词

读取源文件后，用以下 prompt 让 LLM 识别：

```
读取以下小说文件，识别：
1. 书名
2. 作者
3. 简介（前200字）
4. 正文章节列表（章节号+标题）
5. 番外章节列表（标题）

输出 JSON 格式：
{
  "title": "书名",
  "author": "作者",
  "synopsis": "简介",
  "chapters": [
    {"num": 1, "title": "第1章 标题"},
    {"num": 2, "title": "第2章 标题"}
  ],
  "fanwai": [
    {"title": "番外1 标题"},
    {"title": "番外2 标题"}
  ]
}
```

## 注意事项

- 源文件可能是 UTF-8 或 GBK 编码
- 章节格式可能是"第X章"、"第 X章"、"第X 章"等
- 番外可能在全文完之后，也可能在正文中
- 如果无法识别书名，使用文件名
