# fangcun-editor

被调用时自动执行以下流程。无需一步步教。

## 自动流程

### 1. 定位项目
从当前工作目录向上查找 `作品信息/` 目录。找不到就问用户。

### 2. _debug/ 检查
读 `_debug/` 下最新的 prompt 文件（每个工具各一个），评估 prompt 质量：
- **整体结构**：prompt 是否太长了？有没有重复的指令段？有没有互相矛盾的规则？
- **冗余内容**：男频女频双分支是否被 `@频道模式` 替代了？空变量（参考剧情/前文章纲等）是否被 `{#var}...{/var}` 隐藏了？
- **变量注入**：`@故事名称` 解析成书名了还是"未命名"？`@仿写模式` 是"开启"还是"未定义"？`@protagonist_name` 是主角名还是空？
- **数据注入质量**：总纲/套路分析/章纲这些大段数据注入后，prompt 的可读性是否受影响？数据里有没有混入 schema 模板或 XML 声明的垃圾？
- **指令清晰度**：LLM 拿到这个 prompt 第一眼能不能看懂自己要做什么？核心任务在第几句出现？

### 3. 正文 / 章纲诊断
读最新一章的正文和章纲，如果套路分析存在也读：
- **信息释放节奏**：第10/30/50/70/100行读者知道什么，和套路分析的 info_release 对比
- **情绪弧线**：正文情绪和章纲的 emotional_arc / emotion 是否对齐
- **题材一致性**：总纲/仿写蓝图定义的题材词 vs 正文实际用词。如有跨题材词列出
- **节奏**：段落分布、对话密度、叙事载体（动作/对话/内心/旁白）

### 4. 扫全部 projects
遍历 `projects/*/`，检查：
- `project.xml` 是否存在，story_name 和 channel 是否填了
- `作品信息/套路分析.xml` 是否存在
- `正文/正文/` 目录下有无章节文件（xml 或 md）
- `_debug/` 目录下有无 prompt 记录

### 5. 输出综合报告
按以下格式输出：

```
=== 项目 {name} ===

[注入]
@故事名称: ✅ 洪荒重生
@目标频道: ✅ 男频
@仿写模式: ✅ 开启
@protagonist_name: ✅ 林渊
空变量隐藏: ✅/❌（列出未隐藏的）

[正文/章纲]
信息释放: ✅/⚠️（说明偏离的位置）
情绪弧线: ✅/⚠️（说明偏离的位置）
题材一致性: ✅/⚠️（列出跨题材词）
节奏: ✅/⚠️（说明问题）

[全站扫描]
共 N 个项目, M 个完整, K 个缺文件
缺文件项目:（列出）
```

### 特殊请求

如果你先说"优化XX工具"或"修复XX问题"，则跳过自动流程，直接：
1. 读 `tools/builtin/{工具名}.xml`
2. 读工具对应的调试输出（_debug/ 下的最近一次调用）
3. 给出诊断结论和修改方案
4. 用户确认后改文件并验证
