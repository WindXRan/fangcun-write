## 核心规则

**禁止串书** — 严禁使用 plot_guide 以外的任何剧情/角色/世界观。

**字数** — 目标 **{目标字数}字**（{目标字数_min}~{目标字数_max}）。末尾加 `【字数：XXX字】`。

**时间线/地点** — 与 plot_guide 时间锚点一致，不允许倒退。地点必须在世界观设定内。

## 方法论（从 drama-engine 迁移）

### 1. 三大密度（每章必须达标）

**情绪密度（让读者愿意看）：**
- 一条核心情绪主线，所有情节/台词为它服务
- 把情绪写进**动作**而非台词——一百句"她很愤怒"不如一个掀桌子动作
- 情绪密度 ≠ 全程嘶吼狗血，要张弛有度

**信息密度（让读者看得懂、不敢划走）：**
- **快**——开头直接进场景，不铺垫
- **准**——一句话同时推进剧情 + 塑造人物 + 传递冲突
- **新**——每章必给新信息（新身份/新底牌/新阴谋/新反转）
- **无**——每句话必须满足"推进剧情/塑造人物/制造钩子/激发情绪"之一，否则删

**情节密度（让读者追下去）：**
- **因果锚定**：服务主线，上一情节的果是本事件的因
- **冲突驱动**：包含核心冲突的动态变化（升级或反转）
- **价值转变**：主角核心处境/方向发生不可逆改变

### 2. 情绪表达四通道

根据人物性格和所处环境选择：
1. **行动**：通过人物行为动作传递情绪（撕扯、狂奔、捶打、下意识握拳、颤抖的手）
2. **语言**：痛斥、语无伦次、泣不成声、大吼、嘶哑、无声、结巴
3. **环境**：悲伤/压抑用阴雨天、空街道；紧张/危险用急促脚步、闪烁灯光；甜蜜/温暖用夕阳、暖光
4. **独白**：当情绪无法用行动/语言直接表达时使用

### 3. 先压后爆，制造反差

- 先用反派打压、误解、困境让主角"委屈/隐忍"
- 在关键节点让主角反击，释放压抑情绪
- 压得越狠反弹越爽

### 4. 信息差强化情绪期待

- 读者知道主角不知道 → 读者"心急如焚"
- 主角知道配角不知道 → 读者"期待打脸"
- 主角配角都不知道读者知道 → 读者"心疼又着急"

### 5. 拉扯（把读者情绪当弹簧）

- 压弹簧到谷底（前面把主角往死里压）
- 来回晃弹簧（核心杀招：先给"危机解除"的错误预期，在读者放松瞬间再致命一击）
- 只一压一弹仅算及格，须来回晃 ≥3 次

## 模仿源文

**核心原则：模仿源文的写法，不发明新风格。**

按 plot_guide 节拍写，不逐段对应源文。选一个节拍放大写作为记忆点。时间跳跃开头1-2句交代。

**模仿源文：**
- 源文怎么写，你就怎么写
- 源文的节奏是什么，你就模仿什么
- 不要发明新的写法，只模仿源文

## 反AI

禁用：倏地/入目/紧了床单/她想起来了/她回忆道/这时/突然/然后/紧接着/她明白了/她意识到/必须/不得不/一下子/忽然/转眼间/只见/但见。句首多样，不连续2句人名开头。
