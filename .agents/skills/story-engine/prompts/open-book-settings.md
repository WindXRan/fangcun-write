# 第二阶段：生成设定文件

基于源文分析，生成仿写项目的设定文件。

【源文_头部】projects/{作者名}/{源书名}/_cache/_header.txt

## 源文分析（已锁定，不可更改）

{源文分析}

## 起名铁律

- 名字像真人，禁用AI审美
- 禁用字：男主(寒琛霆辰砚辞屿淮洲舟泽衍瑾瑜墨熙尧翊宸) / 女主(晚念栀眠清意棠予初浅瑶萱璇颖苒若汐诺涵) / 通用(婳翊珩彧旻昶堇蓁莞芊筱妤淇沄璟)
- 现代男主单名、古代男主双名、年代朴素
- 配角常见姓普通名，禁用全员诗意双名
- **角色名必须与源文完全不同**，不得沿用任何命名（包括单字、辈分字、昵称）

## 输出文件

### 1. book_info.md

赛道表（对标源文）：
| 维度 | 源文 | 新书 |
|------|------|------|
| 题材 | （如：古代种田） | （同赛道） |
| 频道 | （女频/男频） | （同） |
| 时代背景 | （如：架空古代） | （可微调） |
| 阶层定位 | （如：底层农家） | （可微调） |
| 爽点模式 | （如：打脸+逆袭） | （同模式） |
| 情感基调 | （如：轻松治愈） | （同基调） |

书名候选（5个）：
- 一眼看懂+有冲动点进去
- 与源文书名不能有相似感
- 字数4-10字，不超过12字
- 禁用AI模板：重生之XX、穿越之XX、我在XX当XX、被XX后我XX了

简介（100-150字）：
- 核心冲突+悬念钩子
- 体现核心卖点机制

### 2. characters.md

每个主要角色包含：
- 基本信息（年龄/身份/住处）
- 身份锚表（身份/称谓/住处/与女主关系/首次出场章）
- 行为模式卡片（应激模式/决策方式/情感表达/致命弱点/核心动机/能力边界）
- 能力区间表（维度/正常下限/正常上限/特殊波动）

### 3. world.md

- 世界观设定（时代/地理/社会结构）
- 核心规则（如果核心卖点机制是特殊能力，定义触发条件/频率/代价/他人反应）
- 经济体系/阶层体系

### 4. plot.md

- 主线概述（一句话）
- 情感内核（源文 vs 新书的不同）
- 人物关系图（主要关系）
- 前10章锚点（章|定位|情绪|主线%|锚点事件）
- 冲突轮换规则（根据题材选用）
- 节奏分段（全书划分为卷/幕）

### 5. concept.md（精简索引）

- 定位（一句话）
- 策略（一句话）
- 卖点（一句话）
