## 核心规则

| 要素 | 规则 | 做法 |
|------|------|------|
| **赛道** | **源文是什么调性，新书就是什么调性。对标升级，不是换赛道** | 源文擦边→你更擦边；源文甜→你更甜；源文爽→你更爽。方向错了等于白写 |
| **价值** | **每章至少有一个源文没有的亮点** | 更强的钩子/更立体的角色互动/更意外的反转/更精准的情绪打击/更刺激的擦边。选择最惊艳的角度，不惜笔墨 |
| **结构** | 用源文的骨架，但加自己的血肉 | 源文该有的情绪曲线你也要有，但你的抵达方式必须让读者觉得"这才是正宗" |
| **人物** | 类型通用，但细节比源文更鲜活 | 角色说话方式、小动作、应激反应要比源文更让人记住 |
| **冲突** | 源文是A→你用B，但B必须比A更痛 | 源文用身份冲突→你用利益冲突，但要让读者觉得利益冲突比身份冲突更真实、更难解 |

**最重要的一条：每章至少给读者一个"这章有点东西"的瞬间。** 这个瞬间必须是源文那章做不到的。

- 时间线不能跳跃（如前章"早上"→本章"一个月后"而无交代）

- **不要求与源文时间线/场景一致**

## 任务流程

1. 读 settings/characters.md，记住固定角色名
2. 读 settings/plot.md，了解主线和冲突
3. 读 settings/world.md，了解世界观
4. **逐段拆源文结构**：标结构功能+冲突类型+情绪+强度
5. **对标升级**：对每个节拍，先想"源文怎么实现的"，再看"怎么在这个方向上做得更到位"
6. 按节拍字数占比×{目标字数}分配新书字数
7. 识别源文的结构技巧，在新书中对应位置用同类技巧且升级

### 事件设计流程

**第一步：读源文** — 理解该节拍的结构功能+情绪+冲突类型

**第二步：判断"源文做得好不好"**
- 源文强→用不同方式达到同等或更强效果
- 源文弱（套路/平淡）→这是你的升级机会

**第三步：升级设计** — 不看源文具体动作，想"我的版本在同方向上怎么做更到位"
- 更强的钩子？更深的人物揭露？更意外的反转？
- 每章至少一个源文写不出来的亮点

**第四步：验证** — 新书的实现是否真的比源文强，还是只做到"不同"？
