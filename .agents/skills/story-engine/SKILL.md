# story-engine

> 全书规划先行，情绪蓝图处方化，写章按方抓药。

## 文件结构

```
novel-download-authors/{作者名}/
└── {源书名}/
    └── 源文/第N章.txt

仿写/{新书名}/
├── 设定/
│   ├── 新书设定.md（书名+类型+卖点+人设+NPC映射+世界观）
│   ├── 全书弧线.md（情感曲线+角色成长+伏笔+转折点）
│   ├── 简介.md（3种风格）
│   ├── 章节顺序.md
│   └── 情绪蓝图/               # 批处理生成，弧线驱动，处方级强度
│       └── 第N章.md            # 每章鼓点+情绪节拍表+跨章钩子检查
├── 追踪/
│   └── 真相.md（时间线+角色状态+伏笔状态）
└── 正文/第N章.txt
```

## Pipeline

```
开书（弧线+设定） ──→ 情绪蓝图（批处理） ──→ 写章（10 agents × N批） ──→ 人设侵权自检
                             │                        │
                      读章节顺序+全书弧线          读情绪蓝图（处方级强度）
                      逐一读源文章节               读源文（感受情绪曲线）
                      出每章鼓点+情绪节拍表        按节拍表一段一段写
                      跨章钩子连续性检查           生理反应替代形容词
                      保存到 设定/情绪蓝图/               │
                                                 每批写完 → 批后处理（冲突检测+更新真相）
                                                       ↓
                                                 下一批
```

## 开书编排

### Step 0：定位源文

检查用户提供的路径或扫描 `novel-download-authors/{作者名}/{源书名}.txt`（合并文件）。

### Step 1：创建项目目录

```bash
python .agents/skills/story-engine/tools/create_templates.py setup <章节数> <设定目录>
```

### Step 2：开书（2 agents 串行）

| 顺序 | Agent | Task prompt | 输出 |
|------|-------|-------------|------|
| 1 | A1：新书设定 | arc-concept.md | 设定/新书设定.md + 设定/简介.md |
| 2 | A2：全书弧线 | arc-skeleton-core.md | 设定/全书弧线.md |

### Step 3：初始化真相

Task prompt：`prompts/init-truth.md`
输出：仿写/{新书名}/追踪/真相.md

### Step 4：章节映射

Task prompt：`prompts/chapter-mapping.md`
输出：设定/章节顺序.md

### Step 4.5：全书情绪蓝图（批处理）

**思想不受保护，表达才受保护。蓝图只出情绪指令，不出源文情节。**

Task prompt：`prompts/chapter-emotion-blueprint.md`
- 读取章节顺序表→了解映射关系
- 读取全书弧线→提取每章目标情绪类型+强度
- 按顺序逐一读取源文章节→输出每章的鼓点节奏+情绪节拍表
- **情绪强度是处方不是参考：原文压到4级憋屈，蓝图就必须写"堆到4级憋屈"**
- 跨章检查：第N章钩子与第N+1章开头连贯，断层=掉读者
- 保存到：设定/情绪蓝图/第N章.md（每章独立文件）

⛔ **一回出。一次agent调用输出所有章的蓝图，不分批。**

### Step 5：写章（10 agents × N批）

**思想不受保护，表达才受保护。仿故事模式（好心反被讹），不仿具体段落（暴雨被追杀）。**

每批10章并行，每个写章 agent：

1. 读情绪蓝图：`设定/情绪蓝图/第N章.md`（处方级强度，少半档都不行）
2. 读源文第N章（感受情绪曲线，然后忘掉情节）
3. 读弧线+设定+真相
4. 运行 write-chapter.md 写正文第N章
5. 回传：本章状态变更（角色状态/关系/伏笔等）

⛔ **一次出稿。不出中间产物，不跑校验，不重写。**

### Step 5.5：批后处理

Task prompt：`prompts/post-batch.md`
- 汇总本批各章回传的状态变更
- 检测冲突（角色位置/关系阶段/伏笔状态）
- 更新追踪/真相.md

### Step 5.7：人设侵权自检（每批写完后）

**"战神归来"是通用设定，谁都可用。但如果某个角色剥掉所有剧情，读者还能一眼认出是某本书里的特定角色——改到认不出为止。**

每批写完后，检查本批各章正文：
- 角色的身份标签、行为模式、说话风格是否与原书角色存在强对应
- 去掉剧情后，读者是否还能仅凭角色特质定位到源文
- 发现风险→标记并修改正文中的角色描写（只改描写，不重写剧情）

### Step 6：下一批

重复 Step 5 → Step 5.5 → Step 5.7 直到所有章节写完。

### Step 7：导出

```bash
cat 仿写/{新书名}/正文/*.txt > 仿写/{新书名}/{新书名}.txt
```
