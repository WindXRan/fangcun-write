# 润色改写

## 流程

用户说"改/润色/重写"时，先定位问题类型。

## 问题类型

| 用户说 | 问题类型 | 做什么 |
|--------|----------|--------|
| 节奏太慢 | 节奏问题 | 删冗余，加快推进 |
| 对话太假 | 对话问题 | 加潜台词，改口吻 |
| 描写太多 | 描写问题 | 精简，保留关键细节 |
| AI味重 | AI味问题 | 替换禁用词，改句式 |
| 没有爽点 | 爽点问题 | 加冲突/反转/打脸 |
| 没有钩子 | 钩子问题 | 章末加悬念 |

## 输出格式

```
【问题诊断】：XXX
【修改方案】：XXX
【修改后】：XXX
```

## 注意

- 先诊断再改，不要盲目重写
- 改完要对比，让用户看到变化
- 不要改太多，一次改1-2个问题
