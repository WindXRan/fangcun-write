# fangcun-fishbone 鱼骨式源文设定提取

## 分阶段执行

```
主干骨（一次）
─────────────────     ← 你已经在这里了
扫590章章纲
  只产出清单：
  ├─ 角色列表（名字+身份+出场章节）
  ├─ 地点列表
  ├─ 物品列表
  ├─ 势力列表
  └─ 背景概念列表
  不读正文，不做深度
        ↓
小骨头（按需）
─────────────────
对清单中每项：
  读该角色/地点出场章节的正文
  生成结构化设定文件
  例如角色卡含 voice（口癖/OS风格/语气）
```

---

## 用户说

| 你说 | 我做 |
|------|------|
| `/fangcun-fishbone --source 全家偷听心声` | 主干骨：扫全部章纲，产出设定清单 |
| `/fangcun-fishbone --source 全家偷听心声 --deep 角色` | 小骨头：对角色类批量生成完整卡 |
| `/fangcun-fishbone --source 全家偷听心声 --deep 乔忠国` | 小骨头：只补单个角色的详细卡 |

---

## 主干骨（Phase 1）

扫源文项目 `正文/章纲/` 下所有章纲文件，只提取：

- **角色**：`<character name="...">` 和 `<perception char="...">` 中出现的名字，标注首次出场章节
- **地点**：正文中出现的场景名（章纲不包含时从总纲提取）
- **物品**：正文中出现的重要道具名
- **势力**：正文中出现的阵营名
- **背景概念**：总纲中提到的世界观规则名

**主干骨产出**是清单，不是完整文件。
格式示例：
```
角色: 乔娇娇（主角, ch1-594）, 乔忠国（父亲, ch1-530）, 乔天经（大哥, ch1-530）...
地点: 乔府, 皇宫, 护国寺...
```

## 小骨头（Phase 2）

对主干骨清单中的每一项，启动独立 agent 读源文正文生成完整设定文件：

| 类型 | 读什么 | 产出 |
|------|--------|------|
| 角色 | 该角色出场章节的正文 | 角色卡（voice/背景/动机/弧线/口癖/OS风格） |
| 地点 | 提到该地点的章节 | 地点（氛围/布局/功能） |
| 物品 | 使用该物品的章节 | 物品（效果/消耗/限制） |
| 势力 | 涉及该势力的章节 | 势力（成员/目标/资源） |

### 小骨头 agent 执行标准

对每个角色，agent 要做的是：
```
读角色出场的前5章正文
  → 提取口癖（原文原词）
  → 提取OS风格（吐槽/正剧/现代网感）
  → 提取性格气质（2-3个关键词）
  → 提取典型行为模式
  → 输出完整角色卡XML
```

## 输入来源

- 源文章纲：`projects/{source}/正文/章纲/`
- 源文正文：`projects/{source}/正文/正文/`
- 源文总纲：`projects/{source}/作品信息/主题/总纲`

## 输出

- `projects/{source}/作品信息/设定/角色/{名}.xml`
- `projects/{source}/作品信息/设定/地点/{名}.xml`
- `projects/{source}/作品信息/设定/物品/{名}.xml`
- `projects/{source}/作品信息/设定/势力/{名}.xml`
- `projects/{source}/作品信息/设定/背景/{名}.xml`
