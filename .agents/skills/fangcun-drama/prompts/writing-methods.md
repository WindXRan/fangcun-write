# 短剧写作方法论（agent 参考）

本文件是写作技法参考，不是格式约束。agent 在需要时加载，不写进 script.md。

## 三大情绪要点（每集必含至少1个）

| 要点 | 定义 | 作用 |
|------|------|------|
| 爆点 | 令人震惊的事件 | 快速入戏 |
| 虐点 | 让人心痛的事件 | 情感代入 |
| 爽点 | 让人兴奋的高光时刻 | 满足情绪需求 |

- 每集500-800字须覆盖至少一个
- 小情绪累积成大爆发，不可一次宣泄完

## 爽点公式

爽点 = 装 + 打脸 + 震惊 + 收获

## 情绪铺设技巧

- **先压后爆**：先打压主角，关键集反击。压得越狠反弹越爽
- **信息差**：观众知道角色不知道 → 期待感
- **拉扯**：每分钟一次"危机→解除→再危机"

## 开篇规则

- 第1行就入危机，无缓冲期
- 前3集必须见效
- 每集至少1个反转
- 第1集设定主角大目标

## 台词原则

- 精准戳点：针对角色软肋设计
- 贴合性格：遮挡名字仍能判断说话人
- 口语化：禁半文半白
- 单句 ≤20字

## 人物塑造

- 先立标签（1-2个关键词）
- 行动契合人设
- 设定记忆点（口音/动作/怪癖）
- 弧光：初始→变故→转变→最终

## 高频情绪模板

**打压-反击**：嘲讽→变本加厉→亮身份→道歉（逆袭类）
**误会-解开**：造谣→冷战→发现真相→撒糖（甜宠类）
**危机-救赎**：难题→求助无门→贵人出现→温暖（家庭类）

## 场景价值判断示例

**问题场景**：张景辰去市场买菜（猪肉→鱼→橘子→讲价→回来）

**问自己**：这个场景让观众感受到什么？
- 答：买东西过程 → 没有情绪回报 → 废笔

**正确写法**：
```
△张景辰推门进来，车后座绑得满满当当——肉、鱼、橘子。
于兰（愣住）：你…买这么多？
张景辰：你爱吃嘛。
△于兰看着那堆东西，没说话，眼眶红了，赶紧低头假装整理围裙。
```
- 结果展示（车后座绑满）+ 情感反应（愣住、眼眶红）= 4行完成

**另一个例子**：追狍子

**问自己**：这个场景让观众感受到什么？
- 答：他想打猎但追不上 → 为后面搞枪做铺垫 → 有用，但节奏太慢

**正确写法**：
```
△他猫下腰捡起树枝悄悄靠近。刚抡起来，那狍子"呦"一声灵巧跳开，几下跑没了影。
张景辰OS：徒手不行…得搞把枪。
```
- 砍掉"一步一步靠近→三米两米→猛扑→扑进雪里"的慢镜头 → 2行搞定

## 节奏压缩技巧

**原则**：用最少的行数传递最大的信息量。

**技巧**：
1. **跳过过程，直接展示结果**：买东西→他推门进来车后座绑满
2. **合并场景**：买菜+做饭+吃饭 = 一个场景（他推门进来+于兰被感动）
3. **用OS替代描写**：不用写他心里想什么，用一句OS搞定
4. **砍掉过渡**：不用写"他走到门口""他推开门"，直接写结果
5. **合并对话**：把多句废话合并成一句关键台词

**例子**：原来103行的第3集，压缩到37行
- 买菜过程（18行）→ 1行结果展示
- 做饭细节（22行）→ 砍掉
- 追狍子慢镜头（40行）→ 10行核心爽点
