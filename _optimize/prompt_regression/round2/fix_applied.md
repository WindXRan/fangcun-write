# Round 2 修复记录

## 根因判断

审稿分析后，根因判定为 **prompt指令不够（缺少关键约束和优化指令）**，而非结构化数据缺失。

### 判定依据

| 证据 | 结论 |
|------|------|
| 输出199/200行与原文逐字一致 | prompt无"不得逐字复制"约束，模型默认最大化文本相似度 |
| 原文缺陷（开场平淡/infodump/情绪过急）被完整继承 | prompt只有"复刻"指令，没有"分析缺陷并优化"指令 |
| 唯一的主动编辑（删冗余句）降低了微节奏 | prompt缺少"保留有功能的过渡拍"的节奏意识约束 |
| 模式卡数据已在R1补全（含protagonist_voice等7维字段） | 数据面充足，不需要补充结构化数据文件 |

**关键判断**：模型有足够数据做判断，但prompt没有告诉它"什么时候不要复制"和"怎样做得更好"。这是一个纯指令问题。

## 修复操作

### 1. Prompt指令补丁 → `prompt_patch.md`

创建了 `_optimize/prompt_regression/round2/prompt_patch.md`，包含3个补丁：

| 优先级 | 补丁 | 目标 | 目标文件 | 对应审稿差距 |
|--------|------|------|---------|-------------|
| P0 | A: 禁止逐字复制 | 解决"记忆复现vs推理生成"问题 | fanxie-chapter.md | G1 + 修复建议2 |
| P0 | B: 缺陷修复优化 | 解决开场/infodump/情绪过急 | fanxie-chapter.md | G2 + 修复建议F1~F4 |
| P1 | C: 微节奏保留 | 解决自主编辑退化问题 | fanxie-chapter.md | G3 + 修复建议F5 |

**补丁A详细内容**：在fanxie-chapter.md #角色段末尾新增"禁止逐字复制（第一约束）"块，包含：
- 3条禁止项（连续5字相同、句式+关键词、措辞选择）
- 允许项（保留情绪弧线/节奏/调性，换全部措辞）
- 自检方法（文本diff，相似度>60%重写）

**补丁B详细内容**：在补丁A之后新增"仿写即优化"块，包含：
- 5种问题的扫描点和修复原则：
  1. 开场冲击力不足 → 替换抽象描写为具象感知
  2. 叙事性infodump → 碎片化插入进行时，禁止简历式段落
  3. 情绪转折过急 → 加犹豫/怀疑/自问步骤
  4. 情绪表达太直白 → 用身体反应代替直抒
  5. 冗余分解 → 先判断是否构成"收→散"呼吸节奏

**补丁C详细内容**：在Step 4自检清单新增"微节奏检查"项
