# Prompt 修复补丁 — Round 4

## 根因定位

审稿报告指出 **结构化缺数据（开场模型没有参考标杆）是主因**。同时，R1-R3的所有prompt补丁**从未实际应用到 `fanxie-chapter.md`**，导致累积的指令缺口在R4集中爆发。

具体表现为以下差距维度：

| 差距 | R4评分 | 根因层次 | 来源回合 |
|------|--------|---------|---------|
| **开场无画面冲击** — 第1句抽象概念("疼痛")无主语无动作 | 5/10 | 结构化缺数据(主因)+prompt指令不够 | R4新缺口 |
| **信息dump** — 第54-59行整段原著背景说明打断节奏 | 6/10 | prompt指令不够 | R2 补丁B(未应用) |
| **系统展示冗长** — 商城占25行，吐槽重复15行+ | 5/10 | prompt指令不够 | R3 补丁E(未应用) |
| **情感弧线空洞** — 孤儿背景一行带过，无具体画面 | 4/10 | prompt指令不够 | R2 补丁B / R3 补丁D(未应用) |
| **母亲信任过程拖沓** — 46行间隔，两次确认之间填充无效信息 | 6/10 | prompt指令不够 | R1 补丁E(未应用) |
| **OS冗余重复** — 系统展示后连续3段吐槽无新信息 | 5/10 | prompt指令不够 | R1 补丁C / R3 补丁E(未应用) |

**核心结论**：R4输出在关键节点（开场/情感/系统/节奏）全部出现退化，因为之前设计的prompt补丁从未落地。本补丁集一次性应用全部未执行的R1-R3补丁 + 新增R4修正。

## 补丁 P1 — 开场硬约束：前3行必须有具体主语+动作+感官 🔴

**目标文件**: `fanxie-chapter.md`
**对应差距**: 开场引力5/10 — 第1句"疼痛是最先抵达的东西"抽象无画面
**位置**: "Step 1"前，"# 工作流程"之后

**插入内容**:

```
### 🔴 开场硬约束：前3行必须有具体主语+动作+感官（P0）

读 @设定:开场模式参考，确认你的开场类型。第1章**必须使用动作开场**（参见 开场模式参考 > opening_types > action）。

**前3行结构硬约束：**

| 行号 | 必须 | 禁止 |
|------|------|------|
| 第1行 | 主角名 + 具体动作（醒来/睁眼/落地/被托住）。例："沈妙妙从混沌中醒来。" | ❌ 抽象概念（疼痛/黑暗/虚无） ❌ 代词开头（"她"） |
| 第2行 | 感知渲染——用比喻/感官描写建立状态。例："意识像从深水里浮上来。" | ❌ 内心OS ❌ 哲理论述 |
| 第3行 | 触发事件——正在发生的动作/外部刺激。例："屁股上挨了一巴掌，疼得张嘴要哭。" | ❌ 背景回忆 ❌ 设定说明 |

**自检方法**：遮住角色名，读者能"看到"第1行发生了什么吗？如果不能——重写。

**违反后果**：第1行抽象=读者第一眼不知道谁在经历什么，开场冲击力归零。
```

## 补丁 P2 — 禁止单段信息dump 🔴

**目标文件**: `fanxie-chapter.md`
**对应差距**: 节奏6/10 — 第54-59行整段原著背景dump
**位置**: 补丁P1之后

**插入内容**:

```
### 🔴 信息释放：禁止单段纯说明性背景dump（P0）

原著/穿书背景信息的释放方式有硬约束：

- ✅ **允许**：背景信息穿插在角色情感反应中释放（如原文"完了完了完了——穿书前最后一晚，她刚看完一本小说叫《冷面王爷轻点宠》"，情绪和信息捆绑释放）
- ❌ **禁止**：连续4行以上纯说明性段落（"记忆像潮水涌回来"后接整段简历式叙事）
- ❌ **禁止**：用"记忆像潮水涌回来""思绪回到XX前"等句式开启说明段落

**替代方案**：
1. 将原著背景拆成2-3段，穿插在主角的瞬间情感反应中
2. 每个碎片不超过2行，释放1个关键信息点
3. 碎片之间用当前场景的动作/对话/感官打断

**示例修正**：
```
❌ (R4输出第54-59行 dump版)
记忆像潮水涌回来。她死前刚追完一本小说——《冷面王爷轻点宠》。
书里的沈家是太子党铁杆，被大皇子用巫蛊案陷害，满门抄斩。
父亲腰斩，母亲闻讯吐血而亡，两个哥哥流放三千里。
而她——沈妙妙，沈家幼女，三岁夭折。

✅ (穿插式碎片版，参考原文)
穿书前最后一晚，她刚看完一本小说叫《冷面王爷轻点宠》。
开头第一章就是太子太傅沈敬堂一家被诬陷藏匿巫蛊娃娃，满门抄斩——【完了完了完了！】
``` 
（注：后面不需要继续补充完整设定，让读者通过上下文自己脑补。关键信息"灭门"已经释放够了。）
```

## 补丁 P3 — 系统展示压缩 🔴

**目标文件**: `fanxie-chapter.md`
**对应差距**: 节奏6/10 — 商城展示25行，吐槽15行重复
**位置**: 补丁P2之后

**插入内容**:

```
### 🔴 金手指/系统展示压缩约束（P0）

系统/商城/金手指在第1章的展示必须压缩，这不是情绪高潮段，是功能段。

**硬约束：**
- 系统展示从"弹窗"到"关闭"不超过 **10行**
- 商品列表只需扫1-2眼（展示3-4件商品名+价格即可，不用逐一品评）
- 主角吐槽系统**不超过2次**，且两次必须有信息增量差异
- ❌ 禁止连续吐槽：（"绝了""1000？你怎么不去抢""你一无所有"三句说同一件事——删掉后两句）
- ❌ 禁止展示第1章不可见的商品
- ❌ 禁止展示"攒分途径"（如"救人一命得10功德"）

**自检方法**：数一下从"光幕弹出"到"光幕消失"的字数。如果超过150字（约10行），说明太长了。

**示例修正**：
```
❌ (R4输出第94-119行 25行版)
光幕弹出→展示4件商品→吐槽→翻页→看到LV2→再吐槽→又吐槽→光幕消失

✅ (压缩版 ≤10行)
光幕浮现：功德商城。功德值：100。下方几件商品：安神符80、易容丹20……
【绝了，我上辈子见义勇为捐了七个器官，就攒了五张脸？】
光幕底部一行灰字：LV2解锁更多（功德1000+）。
【这破系统聊胜于无。】（光幕消失）

## 补丁 P4 — 情感弧线：孤儿背景必须用具体画面支撑 🔴

**目标文件**: `fanxie-chapter.md`
**对应差距**: 情绪共鸣4/10 — "我前世孤儿院长大，从来不知道"一行带过
**位置**: 补丁P3之后

**插入内容**:

```
### 🔴 情感弧线：前世背景必须有具体画面支撑（P0）

"前世孤儿/凄惨过去"是最重要的情感锚点，不能用一句空洞OS带过。

**硬约束：**
- 第1次情感锚点前必须有**至少3个具体记忆碎片**铺垫
- 碎片必须是"画面感信息"（具体事件/场景/触感），不是评价性语言
- ✅ 好碎片："助学贷款、加班到凌晨、献血、捐骨髓、被歹徒捅了一刀时刀尖刺入皮肉的冰凉"
- ❌ 坏碎片："我前世很苦"、"没有人对我好"、"从没过过好日子"
- 第1次"有娘真好"的OS必须出现在这些碎片**之后**（读者先看到画面，再听到OS，才会被触动）

**工作原理**：读者共情需要"画面→感受→理解"三步骤。跳过画面直接给感受，读者无法建立情感连接。
```

## 补丁 P5 — 母亲信任过程压缩 🔴

**目标文件**: `fanxie-chapter.md`
**对应差距**: 情绪共鸣4/10 — 母亲从听到→确认间隔46行
**位置**: 补丁P4之后

**插入内容**:

```
### 🔴 母亲发现心声：信任过程压缩（P0）

母亲从"听到心声"到"确认并行动"的过程必须压缩，不能拖沓。

**硬约束：**
- 母亲第1次听到心声 → 第2次确认并行动，**不超过20行**
- 最多允许**1次**自我否定/犹豫（"大概是幻听"类，只出现1次，第二次直接确认）
- 两次之间的填充内容必须是：母亲的动作变化/眼神变化/身体反应，**不能是无效日常**
- 确认后的行动序列：震惊→控制表情→低声试探→派人确认→保护行动（参考 @设定:乔夫人）
- ❌ 禁止：母亲自我怀疑超过1次 ❌ 禁止：确认后用大段OS表达母爱

**违反后果**：46行间隔让读者对"母亲是否相信"失去耐心，削弱戏剧张力。
```

**位置**: Step 4 "输出前自检"清单中，新增自检项

**插入内容**:

```
- [ ] **呼吸空间检查**：每个重大情绪转折（穿书认知/被母亲听懂/情绪感动）之后，是否有至少2-3行环境描写或身体感受来让读者消化？检查是否出现了"一个情绪点刚建立就被下一个事件冲走"的情况。
  - 例如：母亲确认女儿心声后→不要马上跳到危机，先写一段母亲抱娃离开、关门、深呼吸的动作，给读者缓冲
  - 例如：穿书认知的震惊后→不要马上弹金手指，先写一段婴儿身体的无力感（手指攥不紧、脖子撑不住）
```

## 补丁 P7 — 内心OS留白控制（续R3 补丁E）

**目标文件**: `fanxie-chapter.md`
**对应差距**: OS重复冗余，情绪被OS戳破无留白
**位置**: 补丁P4之后（或补丁P5之后）

**插入内容**:

```
### 🔴 内心OS留白控制

- **每个情感点只允许1次内心OS表达**，禁止连续2段OS说同一件事
- **OS密度约束**：
  - ✅ 允许：功能性OS（传递情报、推进认知）
  - ❌ 禁止：追认型OS（已经用【】表达了情感，又在旁白里重复一遍）
  - ❌ 禁止：解读型OS（角色把自己的情绪用OS全说出来，不给读者自己感受的空间）
- **示例修正**：
  - 【呜呜呜，我娘真的好美……而且她好像好爱好爱我……前世我是个孤儿……】→ 压缩为【我娘也太好看了吧】
  - 母亲的爱意通过母亲的行动让读者自己发现，不要通过主角OS说出"她好爱我"
```

## 补丁 P8 — 禁止逐字复制（续R2 补丁A）

**目标文件**: `fanxie-chapter.md`
**对应差距**: 输出与原文相似度过高，缺乏泛化
**位置**: "# 角色"段末尾、"# 工作流程"之前

**插入内容**:

```
### 🔴 禁止逐字复制（第一约束）

这是仿写任务**最高优先级约束**，违反则输出作废：

- ❌ **禁止**连续5个字以上与原文完全相同（标点符号除外）
- ❌ **禁止**复制原文的句式结构+同一组关键词（如原文"从混沌中醒来"→仿写不能也写"从混沌中醒来"）
- ❌ **禁止**保持原文的措辞选择——即使情绪和节奏一致，也必须换一套词汇

✅ **允许**：保留情绪弧线、信息释放节奏、调性风格、段落节奏模式（句均字数、对话占比等量化指标）
✅ **必须**：换全部措辞。源文用"混沌"→仿写可以用"黑暗/虚无/迷雾"；源文用"浮上来"→仿写用"散开/褪去/抽离"

**自检方法**：把仿写和原文并列运行文本对比工具（diff），如果相似度 > 60%，说明违反了本约束。必须重写。
```

## 补丁 P10 — 章尾钩子硬约束（续R1 补丁B）

**目标文件**: `fanxie-chapter.md`
**对应差距**: 钩子出现太晚（第147行），前1/3未埋悬念
**位置**: Step 4 自检清单末尾（补丁P9之后）

**插入内容**:

```
### 🔴 章尾钩子硬约束

- 结尾最后3行必须呈现"具体威胁/行动中的悬念/可预见的冲突"
- **禁止以以下方式收束**：
  - ❌ 抒情总结（"阳光洒进来，暖洋洋的"）
  - ❌ 梦境/幻想（"她做了一个梦，梦里……"）
  - ❌ 哲理升华（"那才叫享清福"类）
  - ❌ 时间跳跃到未来（"多年以后……"）
  - ❌ 圆满意象（"一家人围在桌前吃饭"类大结局感）
- ✅ 允许的收束方式：
  - 具象威胁即将发生（"今晚有人要去书房了"）
  - 新人物/新冲突线首次暗示（"有丫头爬上了爹的床"）
  - 角色行动倒计时（"她必须在三天内想出办法"）
  - 对话未完成/提问悬空（"老爷，有件事……"）
```

## 补丁 P11 — 内心OS口癖强制（续R1 补丁A）

**目标文件**: `fanxie-chapter.md`
**对应差距**: MC声音辨识度不足
**位置**: 补丁P8之后（"# 角色"段末尾）

**插入内容**:

```
### 🔴 MC口癖强制对齐（第一优先级）

读 @源文模式 的 tone 和 protagonist_voice 字段，确认源文主角的口语特征：

- **源文有口癖**（如：哟、特么的、哇哇哇、完了完了）→ 仿写主角必须配同等密度和类型的口癖
- **源文有吐槽风格**（如：现代网感OS、正剧叙事、轻松调侃）→ 仿写主角内心OS必须同风格
- **每个内心OS段落必须有身份辨识度**——把角色名遮住，读者能认出是谁在想
- **禁止**：把MC写成"任何人"——没有口癖、没有吐槽风格、用散文化内心独白
```
