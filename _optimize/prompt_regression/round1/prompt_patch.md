# Prompt 修复补丁 — Round 1

## 根因定位

审稿报告指出 **prompt指令不够精细（次要原因）** 具体表现为：MC人设特征被平均化、开篇/结尾约束缺失、信息释放节奏无硬约束、心声发现过程反复过多。

以下补丁面向 `fanxie-chapter.md`（仿写写章的主 prompt 模板）和关联预设。

---

## P0 — 必须改（影响面大）

### 补丁 A：MC内心OS口癖强制约束

**目标文件**: `fanxie-chapter.md`
**位置**: Step 1 调性对齐段之后，新增一段

**插入内容**:

```
### 🔴 MC口癖强制对齐（第一优先级）

读 @源文模式 的 tone 字段，确认源文主角的口语特征：

- **源文有口癖**（如：哟、特么的、哇哇哇、完了完了）→ 仿写主角必须配同等密度和类型的口癖
- **源文有吐槽风格**（如：现代网感OS、正剧叙事、轻松调侃）→ 仿写主角内心OS必须同风格
- **每个内心OS段落必须有身份辨识度**——把角色名遮住，读者能认出是谁在想
- **禁止**：把MC写成"任何人"——没有口癖、没有吐槽风格、用散文化内心独白
```

### 补丁 B：章节结尾硬约束——禁止抒情/梦幻收束

**目标文件**: `fanxie-chapter.md`
**位置**: Step 4 自检清单末尾

**插入内容**:

```
### 🔴 章尾钩子硬约束

- 结尾最后3行必须呈现"具体威胁/行动中的悬念/可预见的冲突"
- **禁止以以下方式收束**：
  - ❌ 抒情总结（"阳光洒进来，暖洋洋的"）
  - ❌ 梦境/幻想（"她做了一个梦，梦里……"）
  - ❌ 哲理升华（"那才叫享清福"类）
  - ❌ 时间跳跃到未来（"多年以后……"）
  - ❌ 圆满意象（"一家人围在桌前吃饭"类大结局感）
- ✅ 允许的收束方式：
  - 具象威胁即将发生（"今晚有人要去书房了"）
  - 新人物/新冲突线首次暗示（"有丫头爬上了爹的床"）
  - 角色行动倒计时（"她必须在三天内想出办法"）
  - 对话未完成/提问悬空（"老爷，有件事……"）
```

### 补丁 C：重复咆哮限制

**目标文件**: `fanxie-chapter.md`
**位置**: 补丁B之后

**插入内容**:

```
### 🔴 情绪表达密度约束

- 同一情绪爆发（"完了完了完了""啊啊啊啊啊""老阎王"等）全文不超过2次
- 第二次出现必须在情绪层级上比第一次更强烈，或附带新信息增量
- **禁止**连续2段以上的重复咆哮——如果内心OS超过2段还没有释放新信息，删除其中一段
```

## P1 — 建议改（持续优化）

### 补丁 D：信息释放节奏三段约束

**目标文件**: `fanxie-chapter.md`
**目标**: 替换现有的"信息释放节奏对齐"表格，增加更硬的行数约束

**替换**现有的表格（第62-67行表格）：

```
| 位置 | 源文释放了什么 | 仿写必须释放同等信息量 | 行数硬限 |
|------|--------------|---------------------|---------|
| 前50行(前25%) | 例：主角投胎、环境设定、初始感受 | 只写出生/环境/初步感知，不提前揭金手指/世界观 | ❗最多200字，不可超过 |
| 50~120行(25-60%) | 例：穿书真相、金手指上线 | 才揭示世界观设定和主角特殊能力 | 中间段，无额外硬限 |
| 后80行(60-100%) | 例：家庭互动、能力被亲人发现 | 才写家人互动和能力暴露 | 危机必须放在最后20% |
| 最后3行(钩子) | 例：危机预警、悬念钩子 | 具体行动悬念，禁止抒情 | 最后3行必须是具象悬念 |
```

### 补丁 E：心声格式硬约束

**目标文件**: `fanxie-chapter.md`
**位置**: 补丁C之后

**插入内容**:

```
### 🔴 心声格式约束

- 心声内容必须用【】括起来（源文格式），禁止散文化描述
- 母亲听到心声的情节必须在全章前1/3出现，不能拖到中后段
- 母亲从"听到→确认→接受"最多允许1次犹豫/否定，不能超过2次心理反复
- 母亲确认心声的节奏：听到→震惊（最多1次自我否定）→确认来源→行动（派人查看）
```

### 补丁 F：全章网文基调控

**目标文件**: `fanxie-chapter.md`
**位置**: 补丁E之后

**插入内容**:

```
### 🔴 全章网文基调约束

- 全章保持"轻快+节奏感"的网文基调
- 控制文学化/抒情化描写比例在20%以内
- 每200字必须出现至少1句口语/吐槽/心理OS（如果连续200字没有——说明调性漂了）
- 禁止：散文化内心独白（"有一种陌生又跳脱的语调"类）、文学化环境描写、无信息增量的心理活动
```

## 补丁 G（补充）— fanxie-pattern.md 模式卡新增字段

**目标文件**: `fanxie-pattern.md`
**目标**: 模式卡增加 MC 口癖特征提取，让后续步骤有基准可参考

在 `tone` 对象中加入：

```json
"protagonist_voice": {
  "has_speech_pattern": true/false,
  "speech_patterns": ["哟", "特么的", "哇", "完了完了", "啊啊啊"],
  "internal_os_style": "吐槽/现代网感/严肃/文艺/其他",
  "humor_type": "自嘲/毒舌/冷吐槽/夸张/无",
  "identity_anchor": "角色最突出的辨识度标签（如：孤儿→对家庭温暖极度渴望）"
}
```

## 补丁 H — 总纲金手指字段扩展

**目标文件**: `story.schema.xml`（总纲模板）
**目标**: golden_finger 增加 `initial_points` 和 `causal_chain` 字段，确保数据文件中有结构化的积分和因果链信息

将 golden_finger 段扩展为：

```xml
<golden_finger>
  <name>金手指名称</name>
  <effect>能力描述</effect>
  <cost>代价/限制</cost>
  <initial_points>初始积分/资源值（如100）</initial_points>
  <causal_chain>因果链：因为做了什么→获得什么→引发什么后果</causal_chain>
  <activation_condition>激活条件</activation_condition>
</golden_finger>
```
