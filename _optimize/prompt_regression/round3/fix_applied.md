# Round 3 修复记录

## 根因判断

审稿分析后，根因判定为 **混合型：结构化缺数据（primary）+ prompt指令不够（secondary）**。

### 判定依据

| 证据 | 根因类型 | 结论 |
|------|---------|------|
| 父亲角色从输出中完全删除，但角色卡 `沈敬堂.xml` 存在 | 结构化缺数据 | 章纲 `第1章.xml` 未在 characters 和 beats 中包含父亲，导致模型不知需写出场 |
| 母亲人设扁平化为"好妈妈"符号，丢失智谋感 | prompt指令不够 | 角色卡有丰富数据，但 prompt 无"智谋>感性"约束，模型走捷径选母性煽情 |
| 金手指系统价格被篡改（40/20 + "救一命得10功德"） | 结构化缺数据 | 总纲 `golden_finger` 和功德商城缺少结构化定价数值，模型按模板填充乐观系统 |
| 节奏急乱赶，情绪被内心OS戳破 | prompt指令不够 | prompt 无呼吸空间约束和OS留白约束 |

**关键判断**：R1/R2 已补了角色卡语言7维和口癖约束，但章纲和总纲的数据面仍有缺口。同时 prompt 缺少4条关键的写人/写系统约束。这是一个数据驱动问题（缺少结构化数据引导模型行为）和指令问题（prompt未约束关键维度）的混合体，但**数据缺失是更根本的原因**——没有章纲父亲字段，模型不知道父亲该出场；没有总纲定价数据，模型自己编。
