# Prompt 修复补丁 — Round 3

## 根因定位

审稿报告指出 **混合型根因：结构化缺数据（章纲缺父亲角色 + 总纲缺功德数值）+ prompt指令不够（母亲人设/金手指/呼吸节奏）**。

具体表现为四个层次：

| 层次 | 审稿差距 | 根因 |
|------|---------|------|
| G1 (P0) | 父亲角色被完全删除（沈敬堂），家庭氛围腰斩 | 章纲 `第1章.xml` 未在 characters 和 beats 中包含父亲；prompt 无"所有核心家庭成员必须有戏份"约束 |
| G2 (P0) | 母亲人设扁平化为"好妈妈"符号，丢失当家主母智谋感 | prompt 无"沈夫人性格特征：当家主母的果断和智谋优先于母性煽情"约束 |
| G3 (P0) | 金手指数值被篡改（40/20价格 + "救一命得10功德"乐观提示） | 总纲 `golden_finger` 和功德商城缺少结构化定价数据；prompt 无"严格遵循数据文件数值"约束 |
| G4 (P1) | 节奏急乱赶，无呼吸空间，情绪直接OS戳破 | prompt 缺少"重大转折后留呼吸空间"和"OS留白"约束 |

**关键判定**：之前R1/R2已补了角色卡和OS口癖约束，但章纲和总纲的"数据面"仍存在缺口（章纲没写父亲出场，总纲没写功德数值）。同时 prompt 层面缺少4条关键的写人/写系统约束。

---

## 修复目标

| 补丁 | 目标 | 对应审稿差距 |
|------|------|-------------|
| **A (P0)** | 父亲第1章必须出场，至少4-6行戏份 | G1 |
| **B (P0)** | 母亲人设：智谋>感性，行动完成信任，非宣告 | G2 |
| **C (P0)** | 金手指数值严格遵循结构化数据，禁止篡改 | G3 |
| **D (P1)** | 重大转折后加呼吸空间（环境/身体感受2-3行） | G4 |
| **E (P1)** | 删冗余OS，情绪留白给读者自己发现 | G4 |

## 补丁 A — 父亲第1章必有戏份 🔴

**目标文件**: `fanxie-chapter.md`
**对应差距**: G1（父亲角色完全删除）
**位置**: "# 角色"段末尾、"# 工作流程"之前，与R1/R2补丁并列

**插入内容**:

```
### 🔴 核心家庭成员必须全部出场（P0约束）

第1章必须让所有核心家庭成员在物理上出现，不能只提名字。

- ✅ **父亲**：在"母亲出场"之后、"穿书认知"之前，加入父亲抱女儿的段落（至少4-6行）
  - 父亲的反应：笨拙、欣喜、和妻子有温暖对话（"像你"类夫妻温情）
  - 父亲的角色意义：与母亲形成"父爱笨拙+母爱敏锐"的互补对照
  - 不能只通过丫鬟台词提一句"老爷出去了"，必须写父亲登场抱娃
- ✅ **父亲的长相/气质**：依照 @关联角色 中父亲角色卡 bio 和 tags 描写
- ✅ **父亲离去后**：由父亲离开书房去处理公务自然过渡到穿书认知段

**违反后果**：缺少父亲=家庭温馨感腰斩，读者无法建立"这个家值得救"的情感基础。
```

## 补丁 B — 母亲人设：智谋优先于煽情 🔴

**目标文件**: `fanxie-chapter.md`
**对应差距**: G2（母亲人设扁平化为"好妈妈"符号）
**位置**: 补丁A之后

**插入内容**:

```
### 🔴 沈夫人人设精准约束：当家主母优先于母性煽情

读父亲和母亲的角色卡，确认以下人设优先级：

1. **沈夫人的核心是「能干、有手腕、不动声色」**
   - 她听到女儿心声后的反应：震惊→控制表情→低声试探→派人确认动作（行动序列），不是直接宣告"娘护你"
   - **禁止**：一听到心声就泪流满面表忠心
   
2. **信任在行动中完成，不在宣告中**
   - 原文母亲信女儿的方式：眼神变化→低声试探"谁在说话"→派小丫头去书房盯梢→吻女儿额头→出门调度
   - 仿写必须复刻这个「行动序列」，每一步展示母亲的判断力
   - **禁止**：直接写"我不管你是什么来历，娘护你"
   
3. **母亲的语言风格**
   - 对丫鬟：温和但有距离感（当家主母的威仪）
   - 对女儿：话少但举动暖（用行动而不是用台词表达母爱）
   - 对危机：不露声色，先确认再行动
   - 参考角色卡中 `language_style` 的「信息偏好：直觉先行，信任后再找证据」
```

## 补丁 C — 金手指数值严格遵循结构化数据 🔴

**目标文件**: `fanxie-chapter.md`
**对应差距**: G3（金手指数值被篡改）
**位置**: "Step 1: 读模式卡"段中，或补丁B之后

**插入内容**:

```
### 🔴 结构化数值严格遵循（P0约束）

读 @作品信息/主题/总纲 的 golden_finger > pricing_rules 字段，逐条执行：

- **初始功德点**：必须用总纲中的 initial_points 值（100），不得自行修改
- **商品价格**：必须用总纲 pricing_rules 中的价格，不得自行编造
- **可用商品列表**：第1章只显示总纲中标注"第1章可见"的商品，LV2商品保持锁定
- **绝望基调**：总纲 tone_rule 指定"第1章系统基调=绝望感"
  - ❌ 禁止出现"救一命得10功德"等攒分提示
  - ❌ 禁止显示乐观/鼓励型系统文案
  - ✅ 系统应表现为"聊胜于无""什么都买不了"的绝境感
- **商品定价逻辑**：最低价商品（如安神符80）让主角感到"连最便宜的都买不起"

**自检方法**：写完商城段落，逐项对照 total_gang.xml golden_finger 的 pricing_rules 字段，确认所有数值一致。
```

## 补丁 D — 呼吸空间约束（重大转折后加缓冲）

**目标文件**: `fanxie-chapter.md`
**对应差距**: G4（节奏急乱赶）
**位置**: Step 4 "输出前自检"清单中，新增自检项

**插入内容**:

```
- [ ] **呼吸空间检查**：每个重大情绪转折（穿书认知/被母亲听懂/情绪感动）之后，是否有至少2-3行环境描写或身体感受来让读者消化？检查是否出现了"一个情绪点刚建立就被下一个事件冲走"的情况。
  - 例如：母亲确认女儿心声后→不要马上跳到危机，先写一段母亲抱娃离开、关门、深呼吸的动作，给读者缓冲
  - 例如：穿书认知的震惊后→不要马上弹金手指，先写一段婴儿身体的无力感（手指攥不紧、脖子撑不住）
```

## 补丁 E — 内心OS留白控制

**目标文件**: `fanxie-chapter.md`
**对应差距**: G4（情绪被内心OS戳破，无留白）
**位置**: 补丁D之后

**插入内容**:

```
### 🔴 内心OS留白控制

- **每个情感点只允许1次内心OS表达**，禁止连续2段OS说同一件事
- **OS密度约束**：
  - ✅ 允许：功能性OS（传递情报、推进认知）
  - ❌ 禁止：追认型OS（已经用【】表达了情感，又在旁白里重复一遍）
  - ❌ 禁止：解读型OS（角色把自己的情绪用OS全说出来，不给读者自己感受的空间）
- **示例修正**：
  - 【呜呜呜，我娘真的好美……而且她好像好爱好爱我……前世我是个孤儿……】→ 压缩为【我娘也太好看了吧】
  - 母亲的爱意通过母亲的行动让读者自己发现，不要通过主角OS说出"她好爱我"
```
