# 异常模式库 — 跨轮次积累

**用途**：记录每次逆推法迭代中发现的问题模式，追踪修复状态，避免同一问题反复修。

**原则**：
- 每条模式**首次记录**后，后续轮次无论是否复现都追加时间戳
- 如果某模式**连续 2 轮未复现** → 标记 `dormant`（休眠）
- 如果某模式**休眠后复现** → 标记 `reactivated`（复发），启动升版方案

## 模式详情

### 001 — 开场缺乏画面冲击

| 字段 | 内容 |
|------|------|
| **症状** | 开场第一句话抽象/散文化/"像……一样"比喻句，不是具体动作/OS/对话 |
| **典型表现** | "疼痛是最先抵达的东西""意识从深水中浮上来" vs 原文"乔娇娇投胎了""她被分娩出来，奋力睁大眼睛" |
| **触发条件** | 原文开场是具体动作/吐槽，仿写在同样位置变成抽象描写 |
| **模式寿命** | R1 → R5 全部命中，从未彻底修复 |

**修复历史：**

| 轮次 | 方案 | 效果 | 备注 |
|:----:|------|:----:|------|
| R1 | 未专门修复 | 持续 | R1的补丁针对MC口癖和结尾，未覆盖开场 |
| R2 | 禁止逐字复制约束 | 暂有效(R2 7.5分) | 但R2分数因记忆复现嫌疑不可信 |
| R3 | 未修复 | 恶化4.0 | 退步 |
| R4 | 开场硬约束(前3行)+开场模式参考.xml | 部分有效(R4 5.0) | 从4.0→5.0有改善但未根治 |
| R5 | 未新增修复 | 复发4.0 | R4补丁未正确加载或约束力不够 |

**当前状态**：🔴 active — 连续5轮命中，2次修复均未彻底解决

**未尝试的方向**：
- 在前200字内加入"每句检测"：如果第1句不是具体动作/OS/对话→强制重写
- 将开场模式参考.xml 直接嵌入 prompt（而非外部引用）

**复发后的升版方案**：
```
第7轮再复发后 → 在 fanxie-chapter.md 中加硬约束：
"写完开场后自检：原文第1句是'乔娇娇投胎了'——具体动作+口语。
你的第1句如果是抽象描写、比喻句、被动语态、环境氛围 → 删掉重写。"
```

### 002 — 信息释放节奏失控

| 字段 | 内容 |
|------|------|
| **症状** | 前70-100行塞入过多新信息（8+个设定点），违背"每个 beat ≤2 条新事实"的约束 |
| **典型表现** | R5前70行塞了"出生+投胎+系统+功德+穿书+人物关系+母亲反应+前世"共8个信息点 |
| **触发条件** | 信息密度超过2条/beat，或前25%篇幅释放了全部世界观的50%以上 |
| **模式寿命** | R1 → R5 全部命中，最顽固的模式 |

**修复历史：**

| 轮次 | 方案 | 效果 | 备注 |
|:----:|------|:----:|------|
| R1 | 信息释放节奏三段约束（补丁D）+ 约束表格 | 有效但不够硬 | 表格约束被LLM选择性忽略 |
| R2 | 微节奏保留（补丁C） | 暂有效 | 但同样因记忆复现嫌疑不可信 |
| R3 | 未修复 | 恶化 | 消防水管式信息倾倒 |
| R4 | 禁止单段dump（补丁P2） | 部分有效 | 改善但未根治 |
| R5 | 未新增修复 | 恶化 | 前70行塞8个设定点 |

**当前状态**：🔴 active — 连续5轮命中，3次修复均未彻底解决

**未尝试的方向**：
- 行数硬限：前X行内最多释放Y个信息点，超了自动截断
- 信息释放的"时间锁"：某些信息在特定行数之前禁止揭示
- 多段投放检查：如果连续3段都在释放新信息→强制插入1段纯反应/动作

**复发后的升版方案**：
```
第7轮再复发后 → 在 fanxie-chapter.md 新增行数硬限：
"前200字：最多释放1个纯叙事信息（出生/环境/初始状态）。
200-500字：最多释放2个叙事信息。
500-800字：最多释放1个叙事信息 + 1个情绪/关系信息。
800字以后：不再受行数限，但禁止连续3段均释放新信息。
写完统计信息点：总数 ≤ beat数×2，超了重写。"
```
