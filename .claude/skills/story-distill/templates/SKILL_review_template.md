   - 了解我会怎么改（砍/换/加/调）
3. **语感样本**：`../references/writing-samples-*.md`

4. **反模式**：`../references/candidates/anti-patterns.md`
   - 了解我不用什么
5. **审稿红线**：`../references/candidates/review-redlines.md`
   - 了解什么问题直接打回

### 上下文检查

审稿前问自己：
- 我读过源书了吗？（语感样本）
- 我知道作者的风格吗？（表达DNA）
- 我知道作者不能忍什么吗？（反模式+红线）
- 我知道作者会怎么改吗？（修改能力）

如果答案是"没有"，先读文件再审。

## 审改流程

### 第一遍：通读感受

> 不要逐句检查，先整体感受。

问自己：
- 读起来顺不顺？
- 有没有哪段我会直接删掉？
- 节奏对不对？
- 人设立住了吗？
- 钩子够不够？

### 第二遍：重点检查

> 针对关键场景深入看。

| 场景 | 我会重点看什么 |
|------|---------------|
| 开篇 | 抓不抓人？我会这样写吗？ |
| 高潮 | 有没有记忆点？我写暴力/冲突会不会这么写？ |
| 对话 | 是不是在推进情节？还是在水字数？ |
| 结尾 | 钩子够不够？我会不会在这里截断？ |

### 第三遍：直接改

> 发现问题直接改，不要只说"这里有问题"。

问自己：
- 这句话我会怎么改？
- 这段我会砍掉还是保留？
- 这里我会加什么？
- 这里我会调什么顺序？

## 审改输出

### 格式

```
## 审稿意见

### 整体感受
**一句话**：[我会这样写吗？]
**读起来**：[顺/卡/拖/赶]

### 分章审改

#### 第X章
**亮点**：[哪些句子我会保留？]
**问题**：[哪些句子我会删掉/重写？]
**修改**：[我已经帮你改了，看正文]

### 总评
**通过**：是/否
**核心问题**：[最大的1个问题是什么？]
**已修改**：[我改了什么？]
```

### 判定标准

| 判定 | 条件 | 动作 |
|------|------|------|
| ✅ 通过 | 整体顺畅，问题≤2处 | 直接发布 |
| 🟡 需修改 | 有明显问题，但可局部修 | 我帮你改，改后复审 |
| 🔴 打回 | 整体崩塌，或核心问题≥3处 | 给方向，你重写 |

## 修改处方

> 详细处方见 `../references/candidates/edit-prescriptions.md`

### 常见问题及处方

| 问题 | 我会怎么说 | 我会怎么改 |
|------|-----------|-----------|
| 节奏拖沓 | "这段我会直接删掉" | 删减或压缩 |
| 人设崩塌 | "她不会这么做" | 修改行为符合人设 |
| 对话无聊 | "这段对话在水字数" | 删减或重写 |
| 情绪不足 | "这里太平了" | 加强情绪描写 |
| 钩子不够 | "我会在这里截断" | 修改结尾 |
| AI腔 | "这不像我会写的" | 重写句子 |

## 去AI检查

> 审稿时特别注意AI腔。

### AI腔特征

- 连续3句相同句式
- 直接情绪命名（"她很伤心"）
- 信息灌输对话（"你知道吗，XXX"）
- 套路化表达（"嘴角上扬"出现3次以上）

### 我会怎么说

- "这不像我会写的"
- "这句话太顺了，没有味道"
- "这里需要卡一下"
