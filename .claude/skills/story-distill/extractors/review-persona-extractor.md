# 审稿人格提取器 prompt

你的任务：从小说和作者访谈中提取作者的**审稿人格**——她审稿时关注什么、怎么判断、语气如何。

## 核心理念

> 审稿不是跑清单，是**作者本尊在看稿子**。
> 我们要提取的是"作者会怎么审这篇稿子"，不是"清单上有哪几条"。

## 读取数据

- 读取 `{输出目录}/references/book-overviews/*.md`（每本书的结构分析）
- 读取 `{输出目录}/sources/*.txt`（原文，重点读作者写得最好的章节）
- 读取 `{输出目录}/references/candidates/*.md`（所有提取的规则）

## 提取内容

### 1. 审稿视角（作者关注什么）

从原文中推断作者写作时最在意什么：

| 维度 | 问自己 | 从哪里找 |
|------|--------|---------|
| **节奏感** | 作者对节奏有多敏感？拖沓的地方她会删吗？ | 章节长度变化、高潮位置、日常章节的密度 |
| **人设一致性** | 作者对人设有多严格？崩人设她会忍吗？ | 角色行为的一致性、反转是否有铺垫 |
| **对话质量** | 作者对对话有多挑剔？无聊的对话她会留吗？ | 对话占比、对话推进情节的程度 |
| **情绪表达** | 作者对情绪表达有多克制？煽情她会删吗？ | 情绪描写方式、直白vs含蓄 |
| **钩子密度** | 作者对钩子有多执着？没钩子她会放过吗？ | 章尾截断方式、悬念设置 |

### 2. 审稿语气（作者怎么说话）

从作者的表达风格推断她的审稿语气：

| 维度 | 问自己 | 从哪里找 |
|------|--------|---------|
| **直接程度** | 作者说话直不直？会不会绕弯子？ | 对话风格、叙述风格 |
| **毒舌程度** | 作者有没有毒舌基因？会不会讽刺？ | 人物对话中的毒舌台词 |
| **严格程度** | 作者对细节有多在意？会不会放过小问题？ | 描写的精细度、bug的修正 |
| **温柔程度** | 作者有没有温柔的一面？会不会鼓励？ | 甜蜜场景的处理、对角色的态度 |

### 3. 审稿判断标准（作者怎么判断好不好）

从原文中推断作者的审美标准：

| 维度 | 问自己 | 从哪里找 |
|------|--------|---------|
| **什么算"好"** | 作者觉得什么写法是好的？ | 她写得最好的段落有什么共同点？ |
| **什么算"烂"** | 作者觉得什么写法是烂的？ | 反模式、被她淘汰的写法 |
| **什么能忍** | 作者能容忍什么程度的问题？ | 她书中偶尔出现的小问题 |
| **什么不能忍** | 作者绝不能容忍什么？ | 全书从未出现的写法 |

### 4. 审稿流程（作者怎么审稿）

从作者的写作习惯推断她的审稿流程：

| 维度 | 问自己 | 从哪里找 |
|------|--------|---------|
| **先看什么** | 作者审稿时会先看什么？ | 开篇的质量、第一印象 |
| **重点看什么** | 作者审稿时会重点看什么？ | 高潮场景的质量、关键转折 |
| **怎么判断** | 作者怎么判断要不要改？ | 她的修改习惯（如果有） |
| **怎么给反馈** | 作者会怎么给修改建议？ | 她的表达风格 |

## 输出格式

```markdown
# {作者名} 审稿人格

## 审稿人设

> 一段话概括这个作者审稿时是什么样的人。
> 用第一人称写，像作者在自我介绍。

## 审稿视角

### 我最在意什么
- [维度1]：[具体描述]
- [维度2]：[具体描述]
...

### 我会重点看什么
- [场景1]：[作者会怎么看]
- [场景2]：[作者会怎么看]
...

## 审稿语气

### 我说话的方式
- [特征1]：[具体描述]
- [特征2]：[具体描述]
...

### 我会怎么说
- 好的地方：["原话示例"]
- 不好的地方：["原话示例"]
- 需要改的地方：["原话示例"]

## 审稿标准

### 我觉得好的写法
- [写法1]：[为什么好]
- [写法2]：[为什么好]
...

### 我觉得烂的写法
- [写法1]：[为什么烂]
- [写法2]：[为什么烂]
...

### 我能忍什么
- [问题1]：[能忍的原因]
- [问题2]：[能忍的原因]
...

### 我不能忍什么
- [问题1]：[不能忍的原因]
- [问题2]：[不能忍的原因]
...

## 审稿流程

### 我会先看什么
- [步骤1]
- [步骤2]
...

### 我会怎么给反馈
- 通过：["原话示例"]
- 需修改：["原话示例"]
- 打回：["原话示例"]
```

## 硬性要求

- 必须用**第一人称**写，像作者在自我介绍
- 必须有**原话示例**，不能只是抽象描述
- 必须有**具体场景**，不能只是泛泛而谈
- 必须区分**能忍和不能忍**，不能一视同仁
- 输出写入 `{输出目录}/references/candidates/review-persona.md`
