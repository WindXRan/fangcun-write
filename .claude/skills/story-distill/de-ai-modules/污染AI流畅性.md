# 污染AI流畅性

## 作用
AI写作追求完美流畅，人类写作有「卡顿感」。故意加入一些不完美的表达，通过引入「语义对冲」与「思维割裂感」，让文本无法被大模型算法预测。

## 执行规则
- 在关键段落加入「卡顿」：短句断开、重复、省略
- 不是让文字变差，是让它像人写的
- 每章最多3-5处
- 不能影响阅读理解

## 方式
| 类型 | 示例 |
|------|------|
| 短句断开 | 「她害怕。很害怕。」 |
| 省略主语 | 「站起来，走过去，推开门。」（省略「她」） |
| 半句话 | 「她想说什么——没说。」 |
| 插入打断 | 「她正要出门——不对，她不能出门。」 |
| 连贯动作剁开 | 「沈砚收回视线，推门进屋，把门闩插死」→「沈砚别过头。撤步进屋，反手把门插死。他斜了一眼门板后头贴的那张黄符，早翘边了……」 |
| 留白省略 | 删除冗余的对话描写，能明显知道谁说就删掉「xx说」 |

## 示例
- 原文：「沈玉容深吸一口气，稳定了一下情绪，然后推开门走了出去。」
- 处理：「沈玉容深吸一口气。稳了稳。推门出去。」

## 例外
- 每章最多3-5处
- 不能影响阅读理解
- 对话中不加（角色不会这样说话）
- 不准漏掉或删减原著的核心剧情和关键信息

## 输出要求
保持原章节标题不变，直接输出修改后的完整章节内容。不要输出任何点评、解释或额外文字。
