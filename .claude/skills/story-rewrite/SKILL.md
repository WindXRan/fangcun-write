# story-rewrite：仿写流程引擎

**职责：调度 Agent、管理状态、校验质量。不包含任何写作规则。**

---

## 流程总览

```
Phase 1（主线程）    读源文本 → 提取框架 + 设定锁 + 生成简介
                      ↑ 目录分析与源文本基础层并行
Phase 1.5（测试）    前3章 → 用户确认（可 --skip-test 跳过）
Phase 2（全书写作）  章纲生成 ↘
                      ↘ 正文写作 → 循环至完本（章纲与正文流水线）
Phase 3（收尾）      去AI + 一致性检查 + 字数校验（并行）
```

## 核心原则

1. **分层架构**：主线程调度+校验（子 agent：story-architect 章纲 / narrative-writer 正文）。子 agent 单一任务、read 读取、直接写文件不返回正文。
2. **状态管理**：关键信息存文件，agent 通过 read 读取。
3. **并行优先**：多 agent 必须在单条消息中同时发起所有 task tool call。
4. **风格优先**：风格规则绝对优先于通用规则。prompt 注入语感样本+风格锚点路径，让 agent 自然模仿。
5. **职责分离**：rewrite 定义"何时检查"和"触发条件"，style 定义"检查什么"和"怎么判定"。
6. **资源限制**：单批 ≤10 agent，全书累计 >100 暂停确认。超时 5 分钟标记跳过，同批最多重试 2 次，同一章最多重写 1 次，连续 3 批失败暂停。

### 状态文件

| 文件 | 内容 |
|------|------|
| `{书名}/设定/设定锁.md` | 核心地名、关键物件、人物关系 |
| `{书名}/设定/反调色盘.md` | 源书DNA + 新书反调设计 |
| `{书名}/简介.md` | 简介正文 |
| `{书名}/大纲/章纲_*.md` | 分批章纲 |
| `{书名}/大纲/章纲摘要.md` | 总章数、卷结构、关键转折点 |
| `{书名}/追踪/上下文.md` | 当前写作上下文（滚动窗口，≤3000字） |
| `{书名}/追踪/禁止词.md` | 反模式词汇（生成一次复用） |
| `{书名}/追踪/变异指令.md` | 当前批次的风格变异参数 |
| `{书名}/追踪/进度.md` | 已完成章号 + 待补写队列 |
| `{书名}/追踪/风格锚点.md` | 风格锚点句（从 style 表达DNA 提取，3句） |
| `{书名}/追踪/用户反馈.md` | 用户反馈记录 |

并发安全：进度.md 写入前读取最新状态；上下文.md 只由主线程更新。

## Style 加载

`--style` 指定风格，不指定则询问。

**加载顺序**：先读通用规则，再读风格规则。风格规则绝对优先于通用规则，字段级覆盖。

### 参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--style` | 询问 | 必须指定 |
| `--parallel` | 5 | 每批并行 agent 数，1-10 |
| `--per-agent` | 3 | 每 agent 写几章，1-3 |
| `--skip-test` | false | 跳过 Phase 1.5，直接进 Phase 2 |
| `--timeout` | 5 | 单 agent 超时（分钟） |
| `--resume` | 自动检测 | 断点续写 |

### prompt 模板

```
你是小说写手。请根据以下材料写第 {N} 章。

【风格规则】请先读取：{style_path}
【设定锁】请先读取：{设定锁_path}
【风格锚点】请先读取：{风格锚点_path}

【语感样本】（模仿此语感写作）
{2-3段源书语感样本，≤500字}

【上章结尾】（必须衔接）
{上章最后500字}
上章情绪：{紧张/舒缓/悬念}
本章开头应：{衔接/转折/升级}

【本章章纲】
{当前章章纲}

【输出要求】
- 输出到：{书名}/正文/第{N}章.txt
- 字数：1800-2600 字（风格可覆盖）
- 不返回正文内容，直接写文件
```

**风格锚点文件**（`{书名}/追踪/风格锚点.md`）：Phase 1 完成后，从 style SKILL.md「表达DNA」提取 3 句代表性句子写入此文件。agent 读取后自然获得风格锚点。

---

## Phase 1：框架提取

### 源文本要求

txt 或 md，UTF-8，每章以 `第X章` 开头，≥3章。

### 提取内容

1. **目录分析 + 源文本基础层**（并行）：章名风格、章节密度、第1-3章+最后3章
2. **源文本扩展层**（依赖①）：1/4、1/2、3/4处各±1章 + 情绪高点 + 转折点
   - 输出：全书章数、卷结构、3-5段语感样本（≤500字）+ 叙述者风格
   - **核心原则：模仿 > 定义。** 语感样本嵌入写作 prompt
3. **新书方案**（依赖②，全自动）：书名、人物名、核心设定、章数、题材、**核心冲突**、**爽点类型**
   - **强制**：人物名/地名必须新创。输出：`{书名}/设定/新书方案.md`
4. **简介 + 设定锁 + 风格锚点**（依赖③，三者并行）：
   - 简介：读取 style「书名与简介」，输出 `{书名}/简介.md`
   - 设定锁：一级锁定+二级锁定，输出 `{书名}/设定/设定锁.md`
   - 风格锚点：从 style「表达DNA」提取 3 句，输出 `{书名}/追踪/风格锚点.md`
5. **大纲设计**（依赖③）：读取 style「章纲模板」，输出 `{书名}/大纲/大纲.md`

### 反抄袭机制

触发规则（rewrite 负责）：每章写完自动触发 | Phase 1.5 后批量检查 | Phase 2 每批抽检1章 | Phase 3 终检。具体方法委托 style「反抄袭检查」节。

反调色盘：`{书名}/设定/反调色盘.md`，供 style 检查流程参考。

---

## Phase 1.5：前三章测试

1. 生成前3章 mini-outline（主线程直接生成）
2. 写前3章正文（注入 mini-outline + 设定锁 + style 路径 + 语感样本 + 风格锚点）
3. 字数校验 + 抄袭检查（并行）

**反馈循环**（最多2轮）：满意→Phase 2 | 不满意→重写 | 2轮后仍不满意→回 Phase 1 调整设定/风格后重试。

`--skip-test` 跳过此阶段。

---

## Phase 2：全书写作

### 断点续写

正文目录存在时：扫描最大章号 N → 读取进度/上下文/反馈 → 从 N+1 章续写。

### Step 1：章纲生成

1. 章纲生成（story-architect，≤50章/agent，必须并行）
2. 完整性校验 + 生成章纲摘要 + 提取禁止词 + 初始化进度
3. 容错：全部失败→降级重试；部分失败→只重跑失败 agent；超时→跳过后补写
4. 章纲局部修改：只重新生成被修改章，检查前后章衔接

### Step 2：写作循环

M=5 并行，K=3 章/agent。推荐 M=5/K=3，K≤3，M≤10。

**每批流程**：
1. spawn M 个 narrative-writer（注入设定锁+章纲+style 路径+上章结尾+语感样本+风格锚点路径+变异指令路径）
2. 字数校验 + 质量检查（并行执行）：
   - 字数：pass（1800-2600）| 软边界标记 | 硬边界重写1次
   - 质量：pass/fail，fail→重写1次。同一章最多重写1次
3. 抽查1章反抄袭 + 更新进度 + 上下文滚动（>3000字只保留最近2批摘要）
4. 剩余章纲不足2批时补充生成

### 风格变异

每3批执行：从 style「表达DNA」选取不同句式/节奏组合，写入变异指令.md。

### 用户反馈

修改某章→spawn 修复 agent | 修改章纲→局部修改 | 暂停→保存断点。反馈写入用户反馈.md。

---

## Phase 3：全书写完后

1. **全书去AI**：按 style「去AI策略」节，调用 tools/ 脚本（可并行）
2. **一致性终检**：角色属性/时间线/伏笔状态（可与去AI并行）
3. **字数总校验**：按通用规则 7.1 标准

200+ 章并行 spawn 3-5 agent 处理。
