# story-rewrite：仿写流程引擎

**职责：调度 Agent、管理状态、校验质量。不包含任何写作规则。**

---

## 流程总览

```
Phase 1（主线程）    读源文本 → 提取框架 + 设定锁 + 生成简介
Phase 1.5（测试）    前3章 mini-outline → 写前3章 → 用户确认
Phase 2（全书写作）  章纲生成 → M×K 正文写作 → 循环至完本
Phase 3（收尾）      去AI + 一致性检查 + 字数校验
```

**默认直接写全书，不试水。**

---

## 核心原则

1. **分层架构**：主线程调度+校验，子 agent 单一任务
2. **状态管理**：关键信息存文件，agent 通过 read 读取
3. **并行优先**：多 agent 必须在单条消息中同时发起所有 task tool call
4. **摘要机制**：每步结束生成摘要，下一步用摘要

### 分层架构

```
主线程（调度+校验）
├── story-architect（章纲生成）
└── narrative-writer（正文写作）
```

**主线程职责**：读源文本、提取框架、生成新书方案、创建目录、调度 agent、校验输出、管理状态

**子 agent 职责**：单一任务、read 读取信息、直接写文件不返回正文

### 状态文件

| 文件 | 内容 |
|------|------|
| `{书名}/设定/设定锁.md` | 核心地名、关键物件、人物关系、背景事件 |
| `{书名}/设定/反调色盘.md` | 源书DNA + 新书反调设计 |
| `{书名}/简介.md` | 简介正文 |
| `{书名}/大纲/章纲_*.md` | 分批章纲 |
| `{书名}/大纲/章纲摘要.md` | 总章数、卷结构、关键转折点 |
| `{书名}/追踪/上下文.md` | 当前写作上下文（滚动窗口，≤3000字） |
| `{书名}/追踪/禁止词.md` | 从 style 提取的反模式词汇（生成一次复用） |
| `{书名}/追踪/变异指令.md` | 当前批次的风格变异参数 |
| `{书名}/追踪/进度.md` | 已完成章号列表 + 待补写队列 |
| `{书名}/追踪/伏笔.md` | 伏笔状态 |
| `{书名}/追踪/角色状态.md` | 角色状态 |

## Style 加载

`--style` 指定风格。不指定则询问。风格由 `story-style` 管理。

**prompt 只放路径，不注入内容。** 动态数据（上章结尾、当前章纲）必须注入。

```
/story-rewrite --style=空留                        # 默认参数
/story-rewrite --style=空留 --parallel=8 --per-agent=3
```

- `--parallel`：每批并行的 agent 数（默认 5，最大 10）
- `--per-agent`：每个 agent 写几章（默认 3，最大 3）

### 规则继承

**两层结构**：通用规则（基础层） + 风格规则（风格层）

| 层级 | 文件 | 内容 |
|------|------|------|
| 通用规则 | `.claude/skills/story-style/通用规则.md` | 所有风格继承的基础规则 |
| 风格规则 | `.claude/skills/story-style/{作者名}/SKILL.md` | 该风格的独特规则 |

**加载顺序**：先读通用规则，再读风格规则。风格规则覆盖通用规则中的同类项。

**示例**：
- 通用规则：「章尾钩子」（基础要求）
- 闻栖风格：「钩子轮换制：悬念/甜蜜/反转三选一」（特色变体）
- 最终效果：使用闻栖的钩子轮换制，而非通用的章尾钩子

## Phase 1：框架提取

### 源文本要求

txt 或 md，UTF-8，每章以 `第X章` 开头，≥3章。

### 提取内容

1. **目录分析**（必做）：章名风格、章节密度、情绪曲线、精读章节选择
2. **源文本分析**：
   - 基础层：第1-3章 + 最后3章
   - 扩展层：1/4、1/2、3/4处各±1章
   - 专家层：情绪高点 + 转折点
   - 输出：全书章数、卷结构、3-5段语感样本 + 1章完整样本、叙述者风格
   - **核心原则：模仿 > 定义。** 给 agent 语感样本让它自然模仿
3. **自动生成新书方案**（全自动，不询问）：书名、男女主名、核心设定、章数（对齐源书）、题材
   - **强制要求**：人物名/地名必须新创，不能使用源文本中已有的名字
   - **输出文件**：`{书名}/设定/新书方案.md`（含人物名/地名/核心设定）
4. **生成简介**：主线程直接生成，读取 style SKILL.md「书名与简介」部分
   - **反抄袭原则**：金句必须原创，不照搬源书原句
   - **输出**：`{书名}/简介.md`
5. **设定锁**：一级锁定（核心地名、关键物件、人物关系、背景事件）+ 二级锁定（动态补充）
   - **强制检查**：设定锁必须使用新书方案中的人物名字，不能直接复制源文本的人物名字
   - **一致性校验**：写入前比对新书方案与设定锁的人物名/地名/关键物件名，不一致则修正
6. **大纲设计**：读取 style SKILL.md「章纲模板」部分，输出到 `{书名}/大纲/大纲.md`

### 反抄袭机制（写作时硬约束）

**问题**：事后检查无效，写的时候会直接照搬源书句式。

**解决方案**：在每章写作prompt中注入以下硬约束，写作时必须遵守。

#### 写作时检查清单（注入到每章prompt）

```
【反抄袭硬约束】
写每一句时自查：
1. 这句话是不是源书原句？→ 必须改写，不能只换人名
2. 这个对话是不是源书原话？→ 必须用不同句式表达相同意思
3. 这段描写是不是源书模板？→ 必须用不同比喻/不同角度

允许：借用情节框架、事件顺序、人物关系
禁止：照搬句子、照搬对话、照搬描写模板

具体规则：
- 对话必须重写，不能只换人名（如"走一步看一步"→必须换成别的表达）
- 描写必须换角度（如"手肘撑桌"→换成其他疲惫表现）
- 成语/俗语可以用，但连续3个以上相同组合不行
- 标志性金句（如"把你眼珠子挖出来"）绝对禁止
```

#### 反调色盘（Phase 1 输出）

输出 `{书名}/设定/反调色盘.md`，包含：
- 源书DNA表（8维度）
- 红/黄/绿判定
- 禁用设定清单（只禁人名/地名）

**注意**：反调色盘是参考文件，不是检查工具。真正起作用的是每章prompt中的硬约束。

## Phase 1.5：前三章测试（快速出效果）

Phase 1 完成后，先写前三章让用户快速看到效果。此时全书章纲尚未生成，前3章章纲由主线程直接生成。

1. **生成前3章 mini-outline**（主线程直接生成，参考源文本分析 + 设定锁 + style 章纲模板）
2. **写前3章正文**（主线程直接写，注入 mini-outline + 设定锁 + style 路径）
   - **反抄袭**：写每句时自查——是不是源书原句？只换人名不行，必须改句式/换角度
   - **输出**：`{书名}/正文/第1章.txt`、`第2章.txt`、`第3章.txt`
3. **字数校验**：每章 1800-2600 字
4. **抄袭检查**：写完后与源书前3章逐句对比，发现照搬的句子必须改写

用户确认效果OK后继续 Phase 2。

---

## Phase 2：全书写作

Phase 1.5 完成后继续全书。如果用户确认效果OK，则继续；如果需要调整，先修改再继续。

### 断点续写

`{书名}/正文/` 已存在时：扫描目录找最大章号 N → 读取章纲 → 检查追踪文件 → 从第 N+1 章续写。

### Step 1：章纲生成

目录已在 Phase 1 创建，此处不再重复。

1. 章纲生成（强制 story-architect subagent，≤50章/agent）：
   - 章数硬性对齐源文本
   - 每 agent 知道：总章数、结局章范围、本批次是否是结局
   - **必须并行**：在单条消息中同时发起所有 task tool call
2. 完整性校验：grep 统计章数，对比源文本
3. 生成章纲摘要
4. **提取禁止词**：从 style SKILL.md 提取反模式词汇，写入 `{书名}/追踪/禁止词.md`
5. **初始化进度**：创建 `{书名}/追踪/进度.md`，列出所有待写章号
6. 容错：
   - 全部失败→降级 K=1, M=3 重试
   - 部分失败→只重新 spawn 失败的 agent，成功的不重跑
   - 单 agent 超时→标记该批次，跳过后补写

### Step 2：写作循环

每 agent 写 K=2-3 章，M 个并行。推荐 M=5/K=3，K≤3，M≤10。

**每批流程**：
1. spawn M 个 narrative-writer，注入设定锁 + K 章章纲 + style 路径 + 上章结尾内容
   - **必须并行**：在单条消息中同时发起所有 task tool call
   - 上章结尾内容必须注入（动态数据，不能靠 agent 自己读）
   - 本章开头必须与上章结尾完全一致（时间、地点、人物状态不矛盾）
   - **注入前校验**：设定锁中的人物名/地名必须与新书方案一致，不一致则先修正再注入
   - **反抄袭注入**：每章prompt必须包含「反抄袭硬约束」，写每句时自查
2. 字数校验：目标 2200 字，软边界 1600-3600，硬边界 1200-4200
   - **重试限制**：字数不达标时最多重试1次，仍不达标则直接接受并标记「字数偏差」
   - **禁止循环**：同一章不得重写超过2次
   - **故事连续性优先**：不要为了凑字数破坏节奏
3. 更新进度：写入 `{书名}/追踪/进度.md`（已完成章号、待补写队列）
4. 上下文滚动：更新 `{书名}/追踪/上下文.md`，**超过 3000 字时只保留最近 2 批的摘要**
5. 剩余章纲不足2批时，补充生成新一批章纲（spawn story-architect）
6. 字数不足章节处理：
   - <1200 字：标记为「字数偏差」，加入补写队列（最多补写1次）
   - 1200-1600 字：接受当前版本，标记「字数偏低」
   - **故事连续性优先**：不要为了凑字数破坏情节节奏

### 章纲局部修改

用户要求修改某几章章纲时：
1. 只重新生成被修改的章（spawn 1个 agent）
2. 自动检查前后章衔接（修改章的前一章结尾 vs 后一章开头）
3. 更新章纲摘要

### 质量衰减防护

每5批执行一次风格变异：从 style SKILL.md「表达DNA」中选取不同的句式/节奏组合，写入 `{书名}/追踪/变异指令.md`。包含：本轮选用的句式、节奏模式、避免的重复模式。

### OOC 自动检测

每批完成后，主线程检查：
- 角色行为是否符合设定锁中的人物性格
- 对话风格是否与前文一致
- 发现 OOC → 标记该章，spawn 修复 agent 重写

## Phase 3：全书写完后

1. **全书去AI**：按风格 SKILL.md「去AI策略」节的模块组合，调用 `tools/` 下的脚本执行
2. **一致性终检**：角色属性/时间线/伏笔状态
3. **字数总校验**：每章 >4200 自动截断

200+ 章必须并行 spawn 3-5 agent 处理。

---

## 字数标准

**所有字数统计使用 `tools/word_count.py`**

| 类型 | 规则 |
|------|------|
| 中文字符 | 1字/字 |
| 中文标点 | 1字/个 |
| 省略号/破折号 | 1字/组 |
| 英文单词 | 1字/词 |
| 数字 | 1字/位 |

| 指标 | 值 |
|------|-----|
| 目标 | 2200 字/章 |
| 软边界 | 1600-3600（正常范围） |
| 硬边界 | 1200-4200（超出则截断/标记） |

**重试限制**：
- 字数不达标时，最多重试1次
- 重试仍不达标，直接接受当前版本，标记为「字数偏差」
- 禁止无限循环重写
- 故事连续性优先于字数达标

---

## 章纲质量标准

1. 章数完整，覆盖第1章到第N章
2. 每章5字段：章名（5-8字口语化）、钩子类型、核心事件、情绪基调、章尾钩子
3. 同批内同类型钩子不超过2个
4. 情节点写清"谁做了什么"，不空泛
5. 每5章内至少1个情绪转折
