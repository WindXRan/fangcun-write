# story-rewrite：仿写引擎

## 核心原理

**仿写 = 学习成功模式 + 创新具体内容**

```
源文 → 成功因子提取 → 新书
     → 文风蒸馏    → 新书
     → 结构映射    → 新书
```

**不是照搬情节，而是学习源文为什么火，然后用新的人物和场景重新演绎。**

---

## 设计原则

### 分层分析，逐层深入

```
Phase 1（快速扫描）        Phase 2（深度分析）
├── 全书统计指纹           ├── 本区间10章逐章分析
├── 章节标题→卷结构        ├── 本区间统计指纹
└── 输出：新书概念         └── 输出：章纲+正文
```

**为什么这样分**：
- LLM 处理10章左右的内容是靠谱的
- 全文分析会导致 token 爆炸、深度不够、遗漏关键信息
- 每个区间独立蒸馏，保证文风准确性

---

## 速查表

```
输入：源文 txt
输出：完整小说

Phase 1：初始化（轻量化）
  1.0 创建目录 + 拆章 + 快速扫描
  1.1 新书概念（含设定+简介）
  1.2 初始化真相文件

Phase 2：写作（每区间10章，循环执行）
  2.1 提取源文 + 深度分析（逐章分析+蒸馏）
  2.2 章纲细化
  2.3 写新章（3步管线）
  2.4 状态沉淀 → 下一区间

Phase 3：收尾
  3.1 全书去AI
  3.2 一致性终检
  3.3 导出
```

---

## Phase 1：初始化（轻量化）

### 1.0 创建目录 + 拆章 + 快速扫描

**Step 1：创建目录**

```bash
mkdir -p {书名}/设定
mkdir -p {书名}/大纲
mkdir -p {书名}/追踪
mkdir -p {书名}/正文
mkdir -p {书名}/真相文件
```

**Step 2：源文全书拆章**

```bash
python source_chapter_splitter.py split <源文.txt> {书名}/追踪/源文章节/
```

**Step 3：快速扫描（⚠️ 不是深度分析）**

```bash
python style_analyzer.py <源文.txt> --name "源文全书"
```

扫描章节标题，识别：
- 卷/区间划分（通常每30-50章一个转折）
- 关键转折点位置（标题含"真相""分手""恢复记忆"等）
- 高潮区间（标题含"大结局""最终""决定"等）

输出：
- `{书名}/追踪/统计指纹.md`
- `{书名}/追踪/卷结构.md`

**血崩检查点**：
- [ ] style_analyzer.py 已运行，有量化数据
- [ ] 卷结构已识别，有明确的转折点位置

---

### 1.1 新书概念（含设定+简介）

基于快速扫描结果，设计新书：

**新书概念**（`{书名}/设定/新书概念.md`）：
- 书名候选（3个）
- 类型定位
- 核心卖点（保留哪些成功因子）
- 人物设定（新角色，不照搬源文）
- 故事弧线（三幕结构，几句话）
- 关键转折点（位置+事件，一行一个）
- 与源文的差异化

**世界观设定**（`{书名}/设定/story_bible.md`）：
- 世界规则
- 角色设定
- 地点设定
- 关键道具

**写作规则**（`{书名}/设定/book_rules.md`）：
- 文风规则
- 人物规则
- 剧情规则
- 禁忌规则

**简介**（`{书名}/简介.md`）：
- 3个版本（标准版/悬念版/甜宠版）

---

### 1.2 初始化真相文件

**⚠️ 真相文件用 Markdown 格式，不是 JSON**

```
{书名}/真相文件/
├── current_state.md        # 当前状态
├── pending_hooks.md        # 伏笔池
├── chapter_summaries.md    # 章节摘要
├── character_matrix.md     # 角色关系矩阵
└── emotional_arcs.md       # 情绪弧线
```

初始化时只填框架，具体内容在 Phase 2 逐步填充。

---

## Phase 2：写作（每区间10章）

### 执行流程

```
┌─────────────────────────────────────────────────────┐
│  循环执行，每次处理10章                                │
│                                                       │
│  2.1 提取源文 + 深度分析（逐章分析+蒸馏）              │
│      ├── 逐章分析（核心事件/情绪弧线/钩子）             │
│      ├── 蒸馏 Layer A：统计指纹                       │
│      └── 蒸馏 Layer B：LLM 8维度分析                  │
│      ↓                                                │
│  2.2 章纲细化（基于深度分析，含源文对应）               │
│      ↓                                                │
│  2.3 写新章（3步管线）                                │
│      ↓                                                │
│  2.4 状态沉淀（更新真相文件）→ 下一区间                │
└─────────────────────────────────────────────────────┘
```

---

### 2.1 提取源文 + 深度分析

**⚠️ 每个区间必须重新分析，不能复用其他区间**

**Step 1：提取本区间源文**

```bash
python source_chapter_splitter.py extract <源文.txt> <start> <end> <{书名}/追踪/源文_{start}-{end}.txt>
```

**Step 2：逐章分析（情节结构）**

使用 `prompts/chapter-analyzer.md` 分析本区间每章：
- 核心事件类型
- 情绪弧线
- 钩子类型
- 爽点来源
- 关键细节

输出：`{书名}/追踪/源文分析_{start}-{end}.md`

**Step 3：蒸馏 Layer A（统计指纹）**

```bash
python style_analyzer.py {书名}/追踪/源文_{start}-{end}.txt
```

**⚠️ 输入必须是本区间的10章，不是全书！**

**Step 4：蒸馏 Layer B（LLM 8维度分析）**

使用 `prompts/style-analysis.md` 分析本区间的：
1. 叙事声音与语气
2. 对话风格
3. 场景描写特征
4. 转折与衔接手法
5. 节奏特征
6. 词汇偏好
7. 情绪表达方式
8. 独特习惯

**⚠️ Step 4 必须引用 Step 3 的统计数据，禁止引用全书指标！**

输出：`{书名}/追踪/蒸馏_{start}-{end}.md`

**血崩检查点**：
- [ ] 逐章分析完成，每章都有分析
- [ ] style_analyzer.py 输入是本区间文件，不是全书
- [ ] 蒸馏文件引用本区间统计数据，不是全书指标

---

### 2.2 章纲细化（基于深度分析）

**⚠️ 章纲必须基于 2.1 的深度分析，不能自创剧情**

**章纲模板**：

```markdown
## 第{N}章 [新书章名]

### 源文对应
- 源文第{M}章：[源文章名]
- 核心事件类型：[类型]
- 情绪弧线：[情绪流动]
- 钩子类型：[钩子]

### 新书情节
- 核心事件：[本章要发生什么]
- 情绪弧线：[情绪流动]
- 钩子设计：[章末钩子]
- 爽点设计：[爽点来源]

### 关键细节
- 场景：[在哪里发生]
- 角色动态：[谁做了什么，关系变化]
- 信息揭示：[本章揭示什么新信息]
- 转折点：[关键转折是什么]

### 字数目标
- 2000-2500字
```

输出：`{书名}/大纲/章纲_{start}-{end}.md`

---

### 2.3 写新章（⚠️ 3步管线）

**Step 1：Writer（创作）**

读取：
- 章纲
- 真相文件（真相文件目录下所有 .md 文件）
- 蒸馏_{start}-{end}.md（文风指南）
- story_bible.md
- book_rules.md

输出：第N章初稿（2000-2500字）

**Step 2：Auditor（质量审计）**

读取：第N章初稿

使用 `prompts/auditor-system.md` 进行5维度审计：
1. 字数合规（2000-2500字）
2. 情绪弧线
3. 钩子设计
4. 人设一致性
5. AI痕迹

输出：审计报告（Markdown格式）

**如果不通过**：根据审计报告修复后重新提交

**Step 3：写入正文**

输出：`{书名}/正文/第{N}章.txt`

**并行执行**：
- 每区间10章，启动10个并行管线
- 每个管线独立运行

---

### 2.4 状态沉淀

**⚠️ 每区间更新一次真相文件（10章写完后）**

使用 `prompts/state-updater.md` 更新真相文件：
- current_state.md（当前状态）
- pending_hooks.md（伏笔池）
- chapter_summaries.md（章节摘要）
- character_matrix.md（角色矩阵）
- emotional_arcs.md（情绪弧线）

更新后进入下一区间循环。

---

## Phase 3：收尾

### 3.1 全书去AI

**目标**：清除 AI 写作痕迹，让文字更自然。

**检查清单**：
- [ ] 删除"首先"、"其次"、"总之"等路标词
- [ ] 减少"然而"、"但是"、"因此"的过度使用
- [ ] 删除"仿佛"、"似乎"、"好像"的连续使用
- [ ] 增加口语化表达和个性化细节
- [ ] 检查句式是否过于规整

**方法**：
1. 逐章扫描，标记疑似 AI 痕迹
2. 人工或 LLM 修复
3. 重新检查

---

### 3.2 一致性终检

**检查维度**：

| 维度 | 检查内容 |
|------|----------|
| 角色一致性 | 角色言行是否符合人设，有无崩塌 |
| 世界观一致性 | 是否与 story_bible.md 设定一致 |
| 时间线一致性 | 时间线是否正确，有无矛盾 |
| 伏笔回收 | pending_hooks.md 中的伏笔是否都已回收 |
| 关系一致性 | character_matrix.md 中的关系是否正确发展 |

**方法**：
1. 读取所有真相文件
2. 逐章检查一致性
3. 标记问题并修复

---

### 3.3 导出

**导出格式**：
```
{书名}/{书名}.txt
```

**导出内容**：
- 按章节顺序合并所有正文
- 添加书名和简介
- 添加目录（可选）

**导出命令**：
```bash
# 合并所有章节
cat {书名}/正文/*.txt > {书名}/{书名}.txt
```

| 检查点 | 检查内容 | 失败处理 |
|--------|----------|----------|
| 1.0 快速扫描 | style_analyzer.py 已运行 | 强制运行 |
| 1.0 快速扫描 | 卷结构已识别 | 重新扫描 |
| 2.1 深度分析 | 逐章分析完成，每章都有分析 | 补充分析 |
| 2.1 深度分析 | style_analyzer.py 输入是本区间文件 | 重新运行 |
| 2.1 深度分析 | 蒸馏引用本区间统计数据，不是全书指标 | 重新蒸馏 |
| 2.2 章纲细化 | 基于深度分析，不是自创剧情 | 重新生成 |
| 2.3 写新章 | 执行了3步管线 | 重新执行 |
| 2.4 状态沉淀 | 真相文件已更新 | 强制更新 |

## 常见错误

❌ **错误1**：Phase 1 深度分析全文
✅ **正确**：Phase 1 只做快速扫描（统计指纹+章节标题→卷结构）

❌ **错误2**：Phase 2 用全书统计指标
✅ **正确**：Phase 2 必须用本区间10章的统计指标

❌ **错误3**：章纲自创剧情
✅ **正确**：章纲必须基于深度分析

❌ **错误4**：跳过深度分析
✅ **正确**：每个区间必须重新做完整分析（逐章分析+蒸馏）

❌ **错误5**：不更新真相文件
✅ **正确**：每章必须更新真相文件

❌ **错误6**：照搬源文情节
✅ **正确**：保留情绪弧线，创新具体情节

❌ **错误7**：真相文件用 JSON 格式
✅ **正确**：真相文件用 Markdown 格式
