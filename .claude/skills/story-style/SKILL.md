## 一、开篇规则

- 主角出场必须处于具体威胁（失去生存基础/身份暴露/被迫分离），不是"处境不佳"
- 用冲突事件切入，不铺垫背景

## 二、节奏规则

- 情感高潮前 3-5 步层层递进（区分里程碑事件 1-2 步 / 情感爆发点 3-5 步）
- 信息差持续时间：误会类 1-3 章，悬念类 5-10 章
- 章尾必有钩子（悬念/甜蜜/反转），日常章可用弱钩子

## 三、角色规则

- 性格通过具体动作展示，不直接描述
- 情绪→身体反应：紧张→攥衣角/咬唇；恐惧→瞳孔收缩/后退半步；愤怒→握拳/咬牙

## 四、冲突规则

- 冲突升级路径：物理→社交→情感→价值观
- 真相必须在关键场合、关键人物面前揭露

## 五、表达规则

- 禁止直接命名情绪（不写"她很愤怒"），用行为展示
- 微物传情：物件越小越日常，情感重量越大；必须有前文铺垫
- 反差设计：身份×行为反差制造记忆点

## 六、反模式

| # | 反模式 | 替代做法 |
|---|--------|---------|
| 1 | 傻白甜女主 | 有独立意志和决策能力 |
| 2 | 误会拖剧情超过2章 | 1-3章内解开 |
| 3 | 女主哭诉推动剧情 | 用行动推动 |
| 4 | "说道"做对话标签 | 用动作beat替代 |
| 5 | 纯回忆/闪回章节 | 穿插回忆片段 |
| 6 | 工具人配角 | 配角需有弧光 |
| 7 | 直接命名情绪 | 用行为展示 |
| 8 | 无限拖延信息差 | 控制持续时间 |

## 七、章纲模板

### 7.1 章均字数

| 指标 | 值 | 说明 |
|------|-----|------|
| 目标 | 2200 字/章 | 通用默认值 |
| pass 范围 | 1800-2600 字 | 正常接受 |
| 软边界 | 1600-1800 / 2600-3000 | 标记，接受不重写 |
| 硬边界 | <1600 / >3000 | <1600 重写1次；>3000 截断 |

风格 SKILL.md 可覆盖此范围。

### 7.2 核心事件密度

- 每章 2-3 个核心事件
- 每 5 章内至少 1 个情绪转折

### 7.3 节奏周期

- 3-5 章小循环（解决一个小冲突）
- 15-20 章大循环（解决一个大冲突）

## 八、书名与简介

- 信息顺序：困境→反转→补充
- 结尾钩子：问句或感叹句，不用句号
- 字数 120-200 字，第三人称
- 使用金句级对话

## 九、去AI策略

- 情绪激动用 `？！`，犹豫用省略号
- 避免过度使用感叹号

## 十、反抄袭检查

**所有风格继承，风格可追加但不得削弱。**

| 类型 | 规则 |
|------|------|
| 禁止 | 照搬句子、照搬对话、照搬描写模板、标志性金句照搬 |
| 允许 | 借用情节框架/事件顺序/人物关系、成语俗语单独使用 |
| 限制 | 连续3个以上相同成语/俗语组合禁止 |
| 结构 | 单章与源书同序章事件重合度 ≤60%；连续3章不得完全复制情节走向 |

## 十一、质量检查

**所有风格继承，风格可追加更严格的标准。**

### 章纲质量

| 检查项 | 标准 |
|--------|------|
| 章数完整性 | 覆盖第1章到第N章，无遗漏 |
| 字段完整性 | 每章5字段：章名、钩子类型、核心事件、情绪基调、章尾钩子 |
| 钩子多样性 | 同批内同类型钩子不超过2个 |
| 事件具体性 | 情节点写清"谁做了什么" |

### 正文质量

| 检查项 | 标准 |
|--------|------|
| 字数 | 见 7.1（风格可覆盖） |
| 反模式 | 不触发第六节清单 |
| 设定一致性 | 人物名/地名/物件与设定锁一致 |
| 上下文衔接 | 本章开头与上章结尾无矛盾 |

### OOC 判定

| 检查项 | 标准 |
|--------|------|
| 对话风格 | 与设定锁人物档案一致 |
| 行为逻辑 | 符合当前情境下的人设 |
| 关系互动 | 符合当前关系状态 |

判定：1项不符→疑似OOC；2项以上不符→OOC。

## 附录：蒸馏黑名单 {#blacklist}

> **供 story-distill 使用，不是写作规则。** 蒸馏时不要写入 SKILL.md：
> 1. 每章要有钩子
> 2. 对话要有潜台词
> 3. 角色要有动机
> 4. 冲突要有升级

**使用说明**：所有风格继承此规则。风格 SKILL.md 只定义该风格的独特规则。通用规则中的特色变体写入风格 SKILL.md。
