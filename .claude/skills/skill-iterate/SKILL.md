# skill-iterate：A-B 协作优化 skill

## 起点判断

用户输入决定 A 的起点：
- **有方案**：用户贴了具体方案 → A 直接优化
- **有方向**：用户说了改进方向（如「触发词太乱」「结构不清晰」）→ A 先分析问题，再出方案
- **只有路径**：用户只给了 skill 路径 → A 先全面诊断，再出方案

## 流程

### 阶段 1：准备
1. 读取用户指定的 SKILL.md（贴全文）
2. 判断起点（见上方）

### 阶段 2：A-B 循环（最多 3 轮）

每轮：
1. **A 出方案**：启动 Task(subagent_type="general")
   - 指令模板：「你是 skill 优化执行者。{起点指令}。当前 SKILL.md：{全文}。{B 的反馈}」
   - 起点指令：
     - 有方案：「用户已提供方案。基于此方案优化 SKILL.md：{方案}」
     - 有方向：「用户给了改进方向：{方向}。分析问题并出方案。」
     - 只有路径：「先诊断 SKILL.md 的问题（结构、职责、冗余），再出优化方案。」
2. **B 审查**：启动 Task(subagent_type="general")，指令包含 SKILL.md 全文 + A 的方案
3. **判断**：
   - B 回复「通过」→ 执行修改
   - B 回复「打回重做」或结构性缺陷 → A 重出方案（计入轮数）
   - B 给出可增量建议 → 下一轮
   - 3 轮未通过 → 输出最佳方案 + 未解决建议，由用户决定

### 阶段 3：执行
用 edit 工具按最终方案修改 SKILL.md，输出改了什么、为什么改

## 评估维度

B 审查时参考：
1. **职责单一**：skill 是否只做一件事
2. **结构清晰**：层级是否合理、是否有冗余
3. **无矛盾**：各部分是否自洽
4. **可执行**：指令是否明确、agent 能否直接执行
5. **边界清晰**：是否越界（如注入不相关的功能）

## 注意事项

- A 和 B 都是 general agent，全部分析和推理工作由 A 承担
- 每轮 A 的 prompt 要包含 B 上一轮的反馈
- 修改必须保持 skill 原有的职责边界，不越界
- A/B 返回空 → 重试 1 次，仍为空则终止（A 空）或跳过审查（B 空）
