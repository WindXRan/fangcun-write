# story-rewrite：结构仿写

提取源文本的深层叙事"骨架"。用全新角色、背景、情节填充。生成逻辑自洽的新书。

**核心信念：只仿"骨相"，不仿"皮相"。**

---

## 核心原则

### 原则 1：骨架先行，血肉后填

先理解源文本的叙事引擎。引擎是可迁移的，燃料（具体情节）必须是新的。

**案例**：
- ✅ 复刻"压抑3章→爽点爆发"的节奏，换成全新角色和情节
- ❌ 只换了名字，情节高度相似

### 原则 2：节奏是灵魂，细节是皮囊

读者追的是"压抑-积累-爆发"的情绪节奏。复刻节奏，替换细节。

**案例**：
- ✅ 原文第3章是爽点，仿写第3章也是爽点（节奏对齐）
- ❌ 原文第3章是爽点，仿写第3章还在铺垫（节奏错位）

### 原则 3：逻辑自洽是底线

新故事不能因套用结构而出现逻辑漏洞。

### 原则 4：青出于蓝

仿写不是降级复制。保持结构优势，尝试微创新。"熟悉但更好"。

### 原则 5：题材适配

**不同题材有不同的核心驱动力。仿写时必须识别源文本的题材，使用对应的规则。**

| 题材 | 核心驱动力 | 详见 |
|------|------------|------|
| 神豪/系统 | 金手指+消费+打脸 | `references/genre-specific-rules.md` |
| 言情/总裁 | 感情线+身份冲突+误会 | `references/genre-specific-rules.md` |
| 玄幻/修仙 | 修炼+升级+战斗 | `references/genre-specific-rules.md` |
| 悬疑/推理 | 线索+推理+反转 | `references/genre-specific-rules.md` |

### 原则 6：爽点节奏对齐

仿写的爽点节奏必须与原文完全一致：
- 原文第几章有爽点，仿写就必须第几章有爽点
- 原文爽点是什么类型，仿写就必须是什么类型

## ⚠️ 原创性红线（最高优先级）

**仿"情绪功能"，不仿"具体场景"。**

### 绝对禁止

| 禁止行为 | 说明 | 错误示例 |
|----------|------|----------|
| 场景复制 | 原文有X场景，仿写也有X场景 | 原文"父子喝酒"，仿写"父子喝酒" |
| 句式复制 | 只换名词/数字，句式相同 | "是不是在深圳混不下去了？" |
| 对话直接复制 | 对话内容完全相同 | "帅哥，到点了，你看加个钟不？" |
| 金句直接复制 | 原文标志性金句不能照搬 | "花不完，根本花不完" |
| 系统出场复制 | 用与原文相同的出场方式 | 原文"屏幕黑了+金色字"，仿写也用"屏幕黑了+文字" |
| 验证方式复制 | 用与原文相同的验证序列 | 原文"转账失败→充值失败→网购失败"，仿写也用相同序列 |
| 对话逻辑复制 | 只换AI名字，对话结构相同 | 原文问豆包，仿写问Kimi |
| 信息获取路径复制 | 用相同方式获取经济信息 | 原文"翻政府报告"，仿写也"翻政府报告" |
| 题材错配 | 给言情小说加系统，给悬疑小说加金手指 | 言情小说出现"每天到账100万" |

### 正确做法

**核心原则：复刻情绪功能，替换具体场景/句式/角色。**

**重要：以下示例仅为说明"情绪功能"的概念，不是具体句式。仿写时必须创造全新的表达。**

| 情绪功能 | 说明 | 仿写方向 |
|----------|------|----------|
| 家人质疑 | 家人对主角回乡/失业/失败的质疑 | 创造新的对话场景，不能用任何原文句式 |
| 系统出场 | 金手指/转折点的首次出现 | 用完全不同的媒介和方式 |
| 信息获取 | 主角了解环境/经济/人际关系 | 用不同的场景和角色 |
| 经济分析 | 主角分析当地情况/制定计划 | 用不同的信息来源和思考方式 |
| 消费体验 | 主角第一次使用金手指 | 用不同的消费场景 |
| 余额展示 | 主角意识到自己有钱了 | 用不同的表达方式，不能照搬原文金句 |

### 检查方法

逐句对比原文，问自己：
1. 这句话的**功能**是什么？（传递信息/制造冲突/展示性格）
2. 我用的**场景/句式**和原文一样吗？
3. 能不能用**完全不同的方式**实现同样的功能？

**特别注意**：
- 对话不能直接复制（即使换了名字）
- 场景不能直接复制（即使换了地点）
- 如果发现"只换了名词/数字"，必须重写

## ⚠️ 去AI味预防（写作时直接应用）

**核心：AI味 = 过度圆滑、工整、解释充分。目标是增加口语、停顿、跳跃和具体动作。**

### Gate A：禁用词禁止使用

| 禁用词/句式 | 替代方案 |
|-------------|----------|
| 深吸一口气 | 胸口起伏了一下 / 直接删掉 |
| 眼中闪过一丝... | 他垂下眼 / 眯起眼 |
| 嘴角勾起一抹... | 笑了一下，没到眼底 / 乐了 |
| 仿佛/犹如/宛若 | 像... / 直接白描 |
| 不禁... | 直接写动作 |
| 缓缓开口 | 说 / 用动作引出对话 |

### Gate B：句式去套路

| 禁止句式 | 问题 | 替代方案 |
|----------|------|----------|
| "不是A，而是B" | 最毒AI句式 | 直接写B |
| "...，带着..." | 万能状语 | 用独立短句或动作描写 |
| 连续3+排比 | 过于工整 | 保留1-2个，删掉其余 |

### Gate C：心理描写外化

| AI味写法 | 自然写法 |
|----------|----------|
| 他很紧张 | 他的手在抖 |
| 她很愤怒 | 她一把掀翻了桌子 |
| 他很害怕 | 他的腿在发抖，几乎站不稳 |

### Gate D：节奏打碎

- 段落长短交错（不要每段都3-5行）
- 长句拆短句
- 偶尔用不完整句（口语感）

### Gate E：对话去腔调

- 加入口语化表达（"嗯""哦""行吧""嘶""靠"）
- 适当答非所问
- 用动作穿插对话

### Gate F：结尾去升华（重点）

**AI味结尾的特征**：
- 总结性语句（"他终于明白了..."）
- 升华感慨（"能力越大，责任越大"）
- 哲学反思（"他突然觉得，这个镇子也没那么差"）
- 感叹号结尾（"他一定要改变这一切！"）
- 回忆对比（"想起在深圳的日子..."）

**自然结尾方式**：
1. **动作结尾**：角色做一个具体动作就结束
   - ✅ "他掏出手机，打开备忘录。"
   - ✅ "她转身往家走。"
   - ❌ "他深吸一口气，决定改变这一切。"

2. **对话结尾**：以一句对话结束
   - ✅ "妈，我想买辆车。"
   - ✅ "明天见。"
   - ❌ "他对母亲说：'我一定会让这个家过上好日子。'"

3. **悬念结尾**：抛出一个具体问题或选择
   - ✅ "门外站着一个穿黑色西装的中年男人。"
   - ✅ "手机响了。"
   - ❌ "他知道，真正的挑战才刚刚开始。"

4. **场景结尾**：一个画面定格
   - ✅ "夕阳把他的影子拉得很长。"
   - ✅ "窗外下起了雨。"
   - ❌ "夕阳西下，他突然觉得，人生充满了希望。"

**禁止结尾句式**：
- "他/她知道..."
- "这一刻..."
- "他/她终于..."
- "原来..."
- "从此以后..."
- "他/她暗暗发誓..."
- "能力越大，责任越大"
- "一切都是最好的安排"
- "他突然觉得..."
- "他深吸一口气，决定..."
- "他一定要..."
- "他相信，只要..."
- "这个镇子/城市/世界..."
- "人生/命运/未来..."

**结尾自检清单**：
- [ ] 最后一句话是否以动作/对话/场景结尾？
- [ ] 是否有总结性/升华性/哲学性语句？
- [ ] 是否有"他终于明白了""这一刻他意识到"等句式？
- [ ] 是否以感叹号结尾？
- [ ] 是否有回忆对比（"想起在XX的日子"）？

## 工作流程

### 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 试写章数 | 3 | 默认只写前3章 |
| 并行子agent数 | 1 | 可选1-4，多agent并行生成不同版本取最优 |
| 模式 | fast | fast=跳过框架确认直接生成，safe=先确认框架再生成 |

### Phase 1：接收源文本

用户只需提供**源文本**（文件路径、粘贴文本、或 `拆文库/` 引用）。

**AI 自动决策**：新主角设定、背景等全部由 AI 自动选择最优方向。

**源文本处理**：
- 文件路径 → 读取文件内容
- 对话粘贴 → 直接使用
- 拆文库引用 → 读取拆解报告
- 书名+平台 → 提示用户先用 `/story-long-analyze` 拆解

### Phase 2：结构提取（内部过程）

**此阶段在 AI 内部完成，不输出给用户。**

**多章源文本处理**：
- ≤10章：全量分析
- >10章：取前10章 + 最后1章
- >50章：取前5章 + 中间3章 + 最后2章

**第一步：题材识别**

| 题材 | 关键词 | 核心驱动力 |
|------|--------|------------|
| 神豪/系统 | 系统、金手指、每天到账、消费 | 金钱+消费+打脸 |
| 言情/总裁 | 穿书、替身、总裁、失忆、甜宠 | 感情+身份+误会 |
| 玄幻/修仙 | 修炼、境界、功法、宗门 | 升级+战斗+势力 |
| 悬疑/推理 | 案件、线索、凶手、密室 | 推理+反转+真相 |
| 都市/职场 | 辞职、创业、职场、逆袭 | 事业+人脉+逆袭 |

**第二步：结构提取**

| 要素 | 提取内容 |
|------|----------|
| 节奏地图 | 每章情绪基调、压抑段落长度、爽点频率 |
| 转折点序列 | 关键转折位置、类型、铺垫手法 |
| 场景切换模式 | 切换顺序、节奏、功能分配 |
| 情绪拉扯技巧 | 延迟满足、危机制造、反转手法 |
| 描写手法分布 | 动作/心理/对话/叙述比例 |
| 核心机制分析 | 金手指/感情线/修炼体系/案件线 |
| 角色塑造分析 | 主角性格、配角功能、关系图谱 |

**第三步：结构提取验证**

提取完成后，生成摘要让用户确认方向：
```
## 结构提取摘要
- 源文本：[书名]，[章数]章，[字数]万字
- 题材：[如"神豪/系统"或"言情/总裁"]
- 核心节奏：[如"压抑3章→爽点爆发"]
- 核心驱动力：[如"金钱+消费+打脸"或"感情+身份+误会"]
- 主角类型：[如"逆袭型"或"穿越型"]

方向正确？继续生成仿写框架？
```

### Phase 3：结构映射与确认

**模式决定**：
- **fast模式（默认）**：跳过框架确认，直接生成仿写内容
- **safe模式**：输出框架给用户确认后再生成

#### safe模式输出框架

```

- 书名：[AI生成]
- 主角：[姓名、年龄、性格关键词]
- 核心冲突：[一句话描述]

### 题材适配
- 源文本题材：[如"神豪/系统"]
- 仿写题材：[如"神豪/系统"]
- 核心驱动力：[如"金钱+消费+打脸"]

### 节奏对齐
| 章节 | 源结构功能 | 新剧情方向 |
|------|------------|------------|
| 第1章 | [功能] | [新情节] |
| 第2章 | [功能] | [新情节] |
| ... | ... | ... |

### 核心差异
- 主角：[源] → [新]
- 背景：[源] → [新]

```

> 题材特定规则参考 `references/genre-specific-rules.md`。

### Phase 4：试水仿写（前3章）

**默认只写前3章**。快速试水验证结构可行性。

#### 字数要求

- **字数定义**：含标点符号的总字符数（省略号"……"算2个字，破折号"——"算2个字）
- 先统计源文本每章字数
- 仿写每章字数误差不超过 ±15%
- 网文标准章节约 2000 字/章
- **每章必须达到 2000 字左右**（1700-2300字），不能过短
- **核心章节可适当加长**：第1章和第3章可达2500字
- **禁止章节字数不均衡**：每章字数差异不超过 25%
- **字数不足时的扩充方法**：
  - 增加细节描写（环境、动作、心理）
  - 增加对话内容（金句、信息传递）
  - 增加思考过程（主角的内心独白）
  - 增加场景切换（多个场景串联）

#### 原创性要求

**详见上方"⚠️ 原创性红线"章节。核心要点：仿情绪功能，不仿具体场景。**

#### 输出方式

**单个 txt 文件，存放在仿写试水库**：

#### 写作要求（通用）

1. **严格复刻节奏**：源文本此处是压抑，新文本此处也必须压抑
2. **场景切换对齐**：保持相同的场景顺序
3. **情绪拉扯同步**：相同的手法（信息差、对比等）
4. **细节完全原创**：台词、道具、环境描写全部原创
5. **角色性格鲜明**：主角有2-3个鲜明特点，通过场景/对话/行为展现
6. **节奏对齐检查**：每章写完后对照原文检查节奏
7. **语言完全原创**：禁止模仿原文句式、表达方式、修辞手法
8. **主角性格展现**：通过具体行为、对话、决策展现性格，而非直接描述
9. **禁止章节内重复段落**：同一章节内，任何段落的核心信息不能出现两次
10. **复刻原文幽默感**：如果原文有幽默元素，仿写必须在相同位置保留幽默感
11. **禁止过度内心独白**：反思/哲学段落不得超过连续200字
12. **章尾钩子必须有冲突**：章尾不能以纯哲学反思结尾

#### 技法复刻要求（通用）

13. **开头必须有钩子**：禁止线性叙事开头
    - 悬念式：抛出反常识问题
    - 结果式：先展示结果，再回溯过程
    - 冲突式：直接进入矛盾场景
14. **细节密度要求**：用具体数字堆砌真实感
    - 必须包含：具体金额、具体物品、具体数据
    - 禁止用概述替代细节
15. **对话必须有金句**：
    - 每章至少2句生活化、可传播的台词
    - 禁止平淡对话
    - 对话要有人物性格，不能是纯信息传递
16. **场景分隔用省略号**：场景切换时用"……"分隔
17. **必须通过对话获取信息**：禁止用概述交代背景

#### 题材特定写作要求

**根据Phase 2识别的题材，参考 `references/genre-specific-rules.md` 中的对应规则。**

例如：
- 神豪/系统题材：参考"神豪题材规则"章节
- 言情/总裁题材：参考"言情题材规则"章节
- 玄幻/修仙题材：参考"玄幻题材规则"章节
- 悬疑/推理题材：参考"悬疑题材规则"章节

### Phase 5：质量检查

仿写完成后执行检查：

**评估权重**：
| 维度 | 权重 | 说明 |
|------|------|------|
| 结构一致性 | 35% | 节奏、转折、场景切换 |
| 原创性 | 30% | 无场景/句式/对话复制 |
| 逻辑自洽 | 20% | 动机合理、转折有铺垫 |
| 语言表达 | 15% | 去AI味、口语化、自然 |

#### 结构一致性（35%）
- [ ] 节奏曲线与源文本一致
- [ ] 转折点位置对齐
- [ ] 场景切换顺序相同
- [ ] 情绪拉扯手法复刻

#### 原创性（30%，详见"⚠️ 原创性红线"章节）
- [ ] 逐句对比原文，无场景复制
- [ ] 逐句对比原文，无句式复制
- [ ] 系统出场方式与原文不同（如有系统）- 必须用完全不同的媒介和方式
- [ ] 验证方式与原文不同（如有验证）- 必须用不同的测试序列
- [ ] 家人互动场景与原文不同
- [ ] 角色姓名、设定完全不同
- [ ] 每章字数达到 2000 字左右（1700-2300字）

#### 逻辑自洽（20%）
- [ ] 新故事内部逻辑通顺
- [ ] 角色动机合理
- [ ] 转折有铺垫支撑

#### 语言表达（15%，详见"⚠️ 去AI味预防"章节）

## 异常处理

### 源文本质量差

**表现**：源文本逻辑混乱、节奏失控

**处理**：
1. 提示用户源文本存在问题
2. 建议先用 `/story-long-analyze` 深度拆解
3. 如果用户坚持，提取可用部分，标记问题区域

### 源文本太短

**表现**：源文本不足3章，无法完整提取节奏

**处理**：
1. 提示用户源文本太短
2. 建议补充更多章节
3. 如果用户坚持，基于现有内容提取，标记缺失部分

### 仿写逻辑漏洞

**表现**：仿写过程中出现逻辑不自洽

**处理**：
1. 暂停仿写
2. 指出逻辑漏洞位置
3. 提供2-3个修复方案
4. 用户选择后继续

### 节奏不对齐

**表现**：仿写节奏与原文不一致

**处理**：
1. 对比原文和仿写的节奏曲线
2. 指出不对齐的位置
3. 提供调整建议

## 常见错误

### 错误1：仿"皮相"不仿"骨相"

**表现**：只换了名字，情节高度相似

**正确做法**：只复刻节奏和结构，具体情节必须原创

### 错误2：场景复制

**表现**：原文有"父子喝酒对话"，仿写也有"父子喝酒对话"

**正确做法**：复刻的是"家人交流"这个情绪功能，不是"喝酒"这个场景

### 错误3：句式复制

**表现**：原文"是不是在外头混不下去了？"，仿写"是不是在深圳混不下去了？"

**正确做法**：创造新的对话逻辑

### 错误4：题材错配

**表现**：给言情小说加系统，给悬疑小说加金手指

**正确做法**：识别源文本题材，使用对应的规则

### 错误5：节奏不对齐

**表现**：原文第3章是爽点，仿写第3章还在铺垫

**正确做法**：逐章对照原文，确保节奏对齐

### 错误6：细节抄袭

**表现**：台词、道具、环境描写与原文雷同

**正确做法**：所有细节必须原创

### 错误7：角色性格不鲜明

**表现**：主角没有特点，配角功能不清

**正确做法**：主角2-3个关键词，配角明确功能定位
