# story-rewrite：结构仿写

提取源文本的深层叙事"骨架"。用全新角色、背景、情节填充。生成逻辑自洽的新书。

**核心信念：只仿"骨相"，不仿"皮相"。**

---

## 核心原则

### 原则 1：骨架先行，血肉后填

先理解源文本的叙事引擎。引擎是可迁移的，燃料（具体情节）必须是新的。

**案例**：
- ✅ 复刻"压抑3章→爽点爆发"的节奏，换成全新角色和情节
- ❌ 只换了名字，情节高度相似

### 原则 2：节奏是灵魂，细节是皮囊

读者追的是"压抑-积累-爆发"的情绪节奏。复刻节奏，替换细节。

**案例**：
- ✅ 原文第3章是爽点，仿写第3章也是爽点（节奏对齐）
- ❌ 原文第3章是爽点，仿写第3章还在铺垫（节奏错位）

### 原则 3：逻辑自洽是底线

新故事不能因套用结构而出现逻辑漏洞。

**案例**：
- ✅ 原文靠"系统限制"制造冲突，仿写靠"地域限制"制造冲突（逻辑自洽）
- ❌ 原文靠"系统限制"，仿写硬套但没有合理解释（逻辑漏洞）

### 原则 4：青出于蓝

仿写不是降级复制。保持结构优势，尝试微创新。"熟悉但更好"。

**案例**：
- ✅ 仿写版在保持节奏的同时，增加了更接地气的细节
- ❌ 仿写版只是原文的降级复制

### 原则 5：金手指设计三要素

金手指必须同时满足：
1. **数字大**：金额足够大，制造爽感（每天至少百万级别）
2. **限制简单**：一句话说清限制，读者理解成本低
3. **场景丰富**：能衍生多种消费/投资场景，支撑长篇剧情

**限制条件的核心作用是制造戏剧冲突，不是限制故事发展。**

**设计公式**：
```
金手指 = 大额资金 + 简单限制 + 丰富场景
```

**案例**：
- ✅ 每天6000万，只能在县城花（数字大、限制简单、场景丰富）
- ✅ 每天500万，只能投资产业（数字大、限制简单、场景丰富）
- ❌ 每天5000万，只能在直播间花（场景单一，无法支撑长篇）
- ❌ 每天100万，只能投资新能源（数字太小，爽感不足）

**检查清单**：
- [ ] 数字足够大（每天至少百万级别）
- [ ] 限制条件一句话能说清
- [ ] 能自然衍生冲突（触动别人利益）
- [ ] 有清晰的升级路径

### 原则 6：爽点节奏对齐

仿写的爽点节奏必须与原文完全一致：
- 原文第几章有爽点，仿写就必须第几章有爽点
- 原文爽点是什么类型，仿写就必须是什么类型
- 原文爽点释放多长时间，仿写就必须释放多长时间

**案例**：
- ✅ 原文第4章"买迈巴赫，刷卡打脸"，仿写第4章"签约付款，转账打脸"
- ❌ 原文第4章是爽点，仿写第4章还在谈判（节奏错位）

## ⚠️ 原创性红线（最高优先级）

**仿"情绪功能"，不仿"具体场景"。**

### 绝对禁止

| 禁止行为 | 说明 | 错误示例 |
|----------|------|----------|
| 场景复制 | 原文有X场景，仿写也有X场景 | 原文"父子喝酒"，仿写"父子喝酒" |
| 句式复制 | 只换名词/数字，句式相同 | "是不是在深圳混不下去了？" |
| 对话直接复制 | 对话内容完全相同 | "帅哥，到点了，你看加个钟不？" |
| 金句直接复制 | 原文标志性金句不能照搬 | "花不完，根本花不完" |
| 系统出场复制 | 用与原文相同的出场方式 | 原文"屏幕黑了+金色字"，仿写也用 |
| 对话逻辑复制 | 只换AI名字，对话结构相同 | 原文问豆包，仿写问Kimi |
| 信息获取路径复制 | 用相同方式获取经济信息 | 原文"翻政府报告"，仿写也"翻政府报告" |

### 正确做法

**核心原则：复刻情绪功能，替换具体场景/句式/角色。**

| 情绪功能 | 原文实现场景 | 仿写可选场景 | 禁止 |
|----------|-------------|-------------|------|
| 家人质疑 | "是不是在外头混不下去了？" | "你怎么突然回来了？"/"出什么事了？"/"工作不顺心？" | "是不是在深圳混不下去了？" |
| 系统出场 | 屏幕黑了+金色字 | 身体发热/耳边声音/梦境惊醒/发现异常物品 | 屏幕黑了+金色字 |
| 信息获取 | 技师说"二十多岁谁留在这儿" | 店主抱怨/邻居闲聊/网上帖子 | 技师说"二十多岁谁留在这儿" |
| 经济分析 | 看政府报告+面馆思考 | 逛市场+与商贩对话/查资料+实地走访 | 看政府报告+面馆思考 |
| 消费体验 | 足疗店洗脚 | 理发店/小饭馆/超市购物 | 足疗店洗脚 |
| 余额展示 | "花不完，根本花不完" | "这钱怎么花？"/"每天都有？"/"做梦吧" | "花不完，根本花不完" |

**系统出场创新方式**（不能与原文相同）：
- 身体异感：突然发热/心跳加速/眼前发黑
- 声音提示：耳边低语/手机铃声/奇怪的提示音
- 环境变化：灯光闪烁/温度变化/气味异常
- 意外发现：捡到东西/收到神秘快递/看到奇怪广告
- 梦境：梦中获得信息/醒来发现变化
- 物件触发：祖传玉佩发光/旧手机震动/奇怪的APP自动安装

**章尾钩子类型**（至少包含一种）：
- 冲突型：主角面临选择或挑战
- 悬念型：抛出未解之谜
- 信息型：揭示关键信息
- 行动型：主角决定做某事

### 检查方法

逐句对比原文，问自己：
1. 这句话的**功能**是什么？（传递信息/制造冲突/展示性格）
2. 我用的**场景/句式**和原文一样吗？
3. 能不能用**完全不同的方式**实现同样的功能？

**特别注意**：
- 对话不能直接复制（即使换了名字）
- 场景不能直接复制（即使换了地点）
- 如果发现"只换了名词/数字"，必须重写

## ⚠️ 去AI味预防（写作时直接应用）

**核心：AI味 = 过度圆滑、工整、解释充分。目标是增加口语、停顿、跳跃和具体动作。**

### Gate A：禁用词禁止使用

| 禁用词/句式 | 替代方案 |
|-------------|----------|
| 深吸一口气 | 胸口起伏了一下 / 直接删掉 |
| 眼中闪过一丝... | 他垂下眼 / 眯起眼 |
| 嘴角勾起一抹... | 笑了一下，没到眼底 / 乐了 |
| 仿佛/犹如/宛若 | 像... / 直接白描 |
| 不禁... | 直接写动作 |
| 缓缓开口 | 说 / 用动作引出对话 |
| 声音不大，却带着... | 直接写声音特征 |
| 不容置疑/显而易见 | 用具体事实说话 |

### Gate B：句式去套路

| 禁止句式 | 问题 | 替代方案 |
|----------|------|----------|
| "不是A，而是B" | 最毒AI句式 | 直接写B |
| "...，带着..." | 万能状语 | 用独立短句或动作描写 |
| 连续3+排比 | 过于工整 | 保留1-2个，删掉其余 |
| "他/她知道..." | 直接告诉 | 用行为展示认知 |

### Gate C：心理描写外化（展示而非告诉）

| AI味写法 | 自然写法 |
|----------|----------|
| 他很紧张 | 他的手在抖 |
| 她很愤怒 | 她一把掀翻了桌子 |
| 他很害怕 | 他的腿在发抖，几乎站不稳 |
| 她很伤心 | 她转过身去，肩膀微微颤动 |
| 他感到一丝失落 | 他愣了一下，把手机放回口袋 |

### Gate D：节奏打碎

- 段落长短交错（不要每段都3-5行）
- 长句拆短句
- 偶尔用不完整句（口语感）
- 打断连续排比句

### Gate E：对话去腔调

- 加入口语化表达（"嗯""哦""行吧""嘶""靠"）
- 适当答非所问
- 用动作穿插对话（"她喝了口水。'然后呢？'")
- 删掉解释性对话（角色不会把自己动机说清楚）
- 60%+对话无标签，用动作替代"说"

### Gate F：结尾去升华

- 删掉总结性语句
- 用动作/场景收尾，不要用感慨收尾
- 禁止"他知道...""这一刻..."结尾

### 自然文本基准

| 维度 | 自然文本 | AI味文本 |
|------|----------|----------|
| 段落长度 | 1-3句为主 | 每段4-6句，整齐均匀 |
| 对话标签 | 60%+无标签 | 几乎每句都有"说道/问道" |
| 情绪表达 | 动作展示（"手在抖"） | 直接告诉（"很紧张"） |
| 比喻 | 生活化（"像哈士奇护食"） | 文学化（"如寒冰般"） |
| 语气词 | "嘤""嘶""靠""行吧" | 几乎没有 |
| 省略 | 大量省略，读者脑补 | 面面俱到 |
| 结尾 | 动作/对话收尾 | 总结/升华/感慨收尾 |

## 工作流程

### 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 试写章数 | 3 | 默认只写前3章 |
| 并行子agent数 | 1 | 可选1-4，多agent并行生成不同版本取最优 |
| 模式 | fast | fast=跳过框架确认直接生成，safe=先确认框架再生成 |

### Phase 1：接收源文本

用户只需提供**源文本**（文件路径、粘贴文本、或 `拆文库/` 引用）。

**AI 自动决策**：新主角设定、背景、金手指等全部由 AI 自动选择最优方向。

**源文本处理**：
- 文件路径 → 读取文件内容
- 对话粘贴 → 直接使用
- 拆文库引用 → 读取拆解报告
- 书名+平台 → 提示用户先用 `/story-long-analyze` 拆解

### Phase 2：结构提取（内部过程）

**此阶段在 AI 内部完成，不输出给用户。**

**多章源文本处理**：
- ≤10章：全量分析
- >10章：取前10章 + 最后1章
- >50章：取前5章 + 中间3章 + 最后2章

提取以下要素：

| 要素 | 提取内容 |
|------|----------|
| 节奏地图 | 每章情绪基调、压抑段落长度、爽点频率 |
| 转折点序列 | 关键转折位置、类型、铺垫手法 |
| 场景切换模式 | 切换顺序、节奏、功能分配 |
| 情绪拉扯技巧 | 延迟满足、危机制造、反转手法 |
| 描写手法分布 | 动作/心理/对话/叙述比例 |
| 金手指分析 | 核心机制、限制条件、升级路径 |
| 角色塑造分析 | 主角性格、配角功能、关系图谱 |

**结构提取验证**：提取完成后，生成摘要让用户确认方向：
```
## 结构提取摘要
- 源文本：[书名]，[章数]章，[字数]万字
- 核心节奏：[如"压抑3章→爽点爆发→压抑2章→爽点爆发"]
- 金手指：[一句话描述]
- 主角类型：[如"逆袭型/成长型/系统型"]
- 题材标签：[如"都市|系统|神豪"]

方向正确？继续生成仿写框架？
```

### Phase 3：结构映射与确认

**模式决定**：
- **fast模式（默认）**：跳过框架确认，直接生成仿写内容
- **safe模式**：输出框架给用户确认后再生成

#### safe模式输出框架

```
## 仿写框架

### 新故事设定
- 书名：[AI生成]
- 主角：[姓名、年龄、性格关键词]
- 金手指：[一句话描述]
- 核心冲突：[一句话描述]

### 节奏对齐
| 章节 | 源结构功能 | 新剧情方向 |
|------|------------|------------|
| 第1章 | [功能] | [新情节] |
| 第2章 | [功能] | [新情节] |
| ... | ... | ... |

### 核心差异
- 主角：[源] → [新]
- 背景：[源] → [新]
- 金手指：[源] → [新]

确认无误？有需要调整的地方吗？
```

> 详细的角色设计、金手指分析等参考 `references/` 目录下的文档。

### Phase 4：试水仿写（前3章）

**默认只写前3章**。快速试水验证结构可行性。

#### 字数要求

- **字数定义**：含标点符号的总字符数（省略号"……"算2个字，破折号"——"算2个字）
- 先统计源文本每章字数
- 仿写每章字数误差不超过 ±15%
- 网文标准章节约 2000 字/章
- **每章必须达到 2000 字左右**（1700-2300字），不能过短
- **核心章节可适当加长**：第1章（系统出场）和第3章（深度思考）可达2500字
- **禁止章节字数不均衡**：每章字数差异不超过 25%
  - 其他标点符号算1个字
- **字数不足时的扩充方法**：
  - 增加细节描写（环境、动作、心理）
  - 增加对话内容（金句、信息传递）
  - 增加思考过程（主角的内心独白）
  - 增加场景切换（多个场景串联）

#### 原创性要求

**详见上方"⚠️ 原创性红线"章节。核心要点：仿情绪功能，不仿具体场景。**

#### 输出方式

**单个 txt 文件，存放在仿写试水库**：
```
仿写试水库/{新书名}_试水.txt
```

#### 写作要求

1. **严格复刻节奏**：源文本此处是压抑，新文本此处也必须压抑
2. **场景切换对齐**：保持相同的场景顺序
3. **情绪拉扯同步**：相同的手法（信息差、对比等）
4. **细节完全原创**：台词、道具、环境描写全部原创
5. **黄金三章钩子**：前3章完成「世界观建立 → 金手指展示 → 第一个爽点」
6. **金手指合理性**：符合"数字大+限制简单+场景丰富"三要素
7. **角色性格鲜明**：主角有2-3个鲜明特点，通过场景/对话/行为展现
8. **节奏对齐检查**：每章写完后对照原文检查节奏
9. **语言完全原创**：禁止模仿原文句式、表达方式、修辞手法。必须用全新的语言风格
10. **主角性格展现**：通过具体行为、对话、决策展现性格，而非直接描述
11. **禁止章节内重复段落**：同一章节内，任何段落的核心信息不能出现两次。如果发现重复，必须删除或改写为新信息
12. **复刻原文幽默感**：如果原文有幽默元素（如与AI对话、自嘲），仿写必须在相同位置保留幽默感
13. **禁止过度内心独白**：反思/哲学段落不得超过连续200字，必须用行动、对话或场景切换打断
14. **章尾钩子必须有冲突**：章尾不能以纯哲学反思结尾，必须有具体冲突、悬念或行动承诺

#### 技法复刻要求

15. **开头必须有钩子**：禁止线性叙事开头
    - 悬念式：抛出反常识问题
    - 结果式：先展示结果，再回溯过程
    - 冲突式：直接进入矛盾场景
16. **细节密度要求**：用具体数字堆砌真实感
    - 必须包含：具体金额、具体物品、具体数据
    - 禁止用概述替代细节
17. **系统/金手指出场要有仪式感**：
    - 必须有视觉冲击，但方式必须与原文不同
    - 必须有情绪反应：主角震惊、怀疑、验证的过程
    - 必须有数字冲击：具体金额的视觉呈现
    - **禁止使用与原文相同的出场方式**（如原文用"屏幕黑了+金色字"，仿写必须用其他方式）
    - **创新示例**：声音提示、身体异感、环境变化、梦境、意外发现等
18. **对话必须有金句**：
    - 每章至少2句生活化、可传播的台词
    - **示例**："混不下去就回来，天塌不了"
    - **示例**："二十多岁谁留在这儿啊？去南方进厂，一个月包吃住还能挣四五千"
    - **示例**："是不是在外头混不下去了？"
    - **示例**："长期熬夜，肾气不足啊小伙子"
    - 禁止平淡对话（❌"你怎么回来了？" ✅"是不是在外头混不下去了？"）
    - **技巧**：对话要有人物性格，不能是纯信息传递
19. **场景分隔用省略号**：场景切换时用"……"分隔
20. **必须有系统验证过程**：主角测试金手指边界（成功+失败案例）
21. **必须有物价展示**：
    - 用具体物价展示环境特点，制造代入感
    - **根据场景选择物价类型**：
      - 城镇/县城：日常消费（牛肉面八块）、房价（四千一平）、对比（"一天能买一百套房"）
      - 古镇/乡村：特产价格、住宿价格、手工艺品价格
      - 异域/特殊：根据世界观自定义物价体系
    - **更多示例**：
      - 早餐：素粉加茶叶蛋四块
      - 交通：出租车起步价五块，摩的八块
      - 娱乐：足疗128元，KTV包厢88元
      - 购物：安踏特步是最高档品牌
    - 禁止空泛描述（❌"物价很低" ✅"一碗牛肉面八块，早餐五块搞定"）
22. **必须通过对话获取信息**：禁止用概述交代背景
23. **必须有消费场景**：
    - 主角获得金手指后，必须有具体的消费体验
    - **必须包含**：去实体店消费、体验服务、与服务人员互动
    - **消费场景要接地气**：洗脚、吃饭、买日用品，不要上来就买豪车豪宅
    - **消费场景要展示环境特点**：县城最高档的足疗198元
    - **更多消费场景**：
      - 吃饭：路边摊、小饭馆、奶茶店
      - 购物：超市、服装店、五金店
      - 娱乐：足疗、KTV、网吧
      - 服务：理发、修车、看病
24. **考察阶段至少2章**：第1章获得金手指，第2-3章考察环境
25. **第3章必须有深度思考**：
    - 主角对当地经济结构的分析
    - 主角对发展方向的规划
    - 主角对自身优势的思考
    - 章尾必须有明确的下一步计划

### Phase 5：质量检查

仿写完成后执行检查：

**评估权重**：
| 维度 | 权重 | 说明 |
|------|------|------|
| 结构一致性 | 35% | 节奏、转折、场景切换 |
| 原创性 | 30% | 无场景/句式/对话复制 |
| 逻辑自洽 | 20% | 动机合理、转折有铺垫 |
| 语言表达 | 15% | 去AI味、口语化、自然 |

#### 结构一致性（35%）
- [ ] 节奏曲线与源文本一致
- [ ] 转折点位置对齐
- [ ] 场景切换顺序相同
- [ ] 情绪拉扯手法复刻

#### 原创性（30%，详见"⚠️ 原创性红线"章节）
- [ ] 逐句对比原文，无场景复制
- [ ] 逐句对比原文，无句式复制
- [ ] 系统出场方式与原文不同
- [ ] 家人互动场景与原文不同
- [ ] 角色姓名、设定完全不同
- [ ] 每章字数达到 2000 字左右（1800-2200字）

#### 逻辑自洽（20%）
- [ ] 新故事内部逻辑通顺
- [ ] 角色动机合理
- [ ] 转折有铺垫支撑

#### 语言表达（15%，详见"⚠️ 去AI味预防"章节）
- [ ] 无禁用词（深吸一口气、眼中闪过等）
- [ ] 无套路句式（不是A而是B、带着...等）
- [ ] 心理描写外化（用动作展示情绪）
- [ ] 对话口语化（有语气词、答非所问）
- [ ] 段落长短交错（不工整排比）

#### 金手指合理性
- [ ] 符合"数字大+限制简单+场景丰富"三要素
- [ ] 限制条件一句话能说清
- [ ] 能自然衍生冲突
- [ ] 有清晰的升级路径

#### 角色塑造
- [ ] 主角有2-3个鲜明的性格特点
- [ ] 性格通过场景/对话/行为展现
- [ ] 配角有明确的功能定位（帮手/对手/导师/情感线/信息源）
- [ ] 角色关系图谱清晰
- [ ] 配角性格与主角形成对比或互补

#### 爽点节奏
- [ ] 爽点位置与原文对齐
- [ ] 爽点类型与原文一致
- [ ] 爽点释放时间与原文相当

## 异常处理

### 源文本质量差

**表现**：源文本逻辑混乱、节奏失控、金手指设计不合理

**处理**：
1. 提示用户源文本存在问题
2. 建议先用 `/story-long-analyze` 深度拆解
3. 如果用户坚持，提取可用部分，标记问题区域

### 源文本太短

**表现**：源文本不足3章，无法完整提取节奏

**处理**：
1. 提示用户源文本太短
2. 建议补充更多章节
3. 如果用户坚持，基于现有内容提取，标记缺失部分

### 仿写逻辑漏洞

**表现**：仿写过程中出现逻辑不自洽

**处理**：
1. 暂停仿写
2. 指出逻辑漏洞位置
3. 提供2-3个修复方案
4. 用户选择后继续

### 金手指设计失败

**表现**：金手指不符合三要素

**处理**：
1. 提示金手指设计问题
2. 提供修改建议
3. 用户确认后继续

### 节奏不对齐

**表现**：仿写节奏与原文不一致

**处理**：
1. 对比原文和仿写的节奏曲线
2. 指出不对齐的位置
3. 提供调整建议

## 常见错误

### 错误1：仿"皮相"不仿"骨相"

**表现**：只换了名字，情节高度相似

**正确做法**：只复刻节奏和结构，具体情节必须原创

### 错误2：场景复制

**表现**：原文有"父子喝酒对话"，仿写也有"父子喝酒对话"，只是换了地点和职业

**正确做法**：复刻的是"家人交流"这个情绪功能，不是"喝酒"这个场景。可以用"吃饭聊天"、"一起干活"、"电话沟通"等不同场景

### 错误3：句式复制

**表现**：原文"是不是在外头混不下去了？"，仿写"是不是在深圳混不下去了？"

**正确做法**：创造新的对话逻辑。比如"妈，我想回来了" "回来也好，正好你爸腰不好，帮帮他"

### 错误4：系统出场复制

**表现**：原文"手机屏幕突然黑了，金色的字"，仿写也用相同方式

**正确做法**：用完全不同的方式出场，如身体异感、声音提示、环境变化、意外发现等

### 错误5：金手指设计不合理

**表现**：数字太小、限制太复杂、场景单一

**正确做法**：严格遵循"数字大+限制简单+场景丰富"三要素

### 错误6：节奏不对齐

**表现**：原文第3章是爽点，仿写第3章还在铺垫

**正确做法**：逐章对照原文，确保节奏对齐

### 错误7：细节抄袭

**表现**：台词、道具、环境描写与原文雷同

**正确做法**：所有细节必须原创，不能有任何雷同

### 错误8：角色性格不鲜明

**表现**：主角没有特点，配角功能不清

**正确做法**：主角2-3个关键词，配角明确功能定位
